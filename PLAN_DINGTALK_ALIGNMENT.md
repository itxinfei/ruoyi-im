# 钉钉对齐落地计划（对齐钉钉 8.2 风格）

目标：在不破坏现有功能的前提下，将当前 IM 的 UI 与钉钉 8.2 的风格对齐，核心聚焦在群组头像网格、消息状态指示以及文本渲染的鲁棒性和一致性。

范围
- 群组头像网格的对齐与扩展性改造
- 消息状态指示（发送/送达/已读）在气泡中的呈现
- 文本渲染的鲁棒性与未来扩展准备（链接识别边界、简单格式化）

设计输出
- Design Tokens 方案：在设计系统中新增/调整以下 Token（见 DESIGN_TOKENS_UPDATE.md）
- 补丁设计（详见 PATCH_SKETCH_*.md）用于实现 1–2 个核心差异的可直接提交思路
- 测试与验收要点列出，确保可重复验证

关键里程碑
- 1 周内完成 Token 设计落地与核心差异实现草案
- 2–3 周内完成代码实现初版、单元/集成测试覆盖、并完成对齐评审
- 1 次 UI/体验回归，确保钉钉对齐点覆盖到位

风险与回退
- 颜色/间距的微小差异可能影响现有 UI 的一致性，需通过 tokens 统一管理
- 群组头像的溢出处理需要兼容不同设备密度，若效果不佳可回退到当前实现并逐步迭代
