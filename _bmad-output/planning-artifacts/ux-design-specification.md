## Step 2: UX发现阶段

### 2.1 用户需求分析

**核心功能优先级：**
1. 查看组织架构 - 树形结构展示部门
2. 快速找人 - 搜索 + 成员卡片展示
3. 查看联系方式 - 右侧详情面板
4. 发起聊天 - 一键发消息

### 2.2 竞品参考 - 钉钉PC端

**布局特点：**
- 左侧：组织架构树（可搜索）
- 中间：成员列表（头像+姓名+职位）
- 右侧：个人详情（联系方式+操作按钮）

**视觉风格：**
- 品牌色：#168cff（钉钉蓝）
- 白色卡片背景
- 轻微阴影和圆角
- 紧凑的信息密度

### 2.3 当前实现分析

**现有代码结构：**
- 三栏布局已实现
- 使用Element Plus组件
- API: getOrgTree, getDepartmentMembers

**需要优化：**
- 视觉细节对标钉钉
- 交互体验优化
- 响应式适配

## Step 3: UX设计决策

### 3.1 布局结构

```
┌─────────────┬─────────────┬──────────────────┐
│  组织架构树  │   成员列表   │     详情面板      │
│   240px    │    320px    │      弹性宽度     │
├─────────────┼─────────────┼──────────────────┤
│ 🔍 搜索框   │  部门名称    │   👤 头像姓名    │
│ 📁 部门树   │  👤👤👤👤👤👤 │   📱 电话邮箱   │
│   📄 成员   │  👤👤👤👤👤👤 │   💬 发消息按钮  │
│             │  👤👤👤👤👤👤 │                  │
└─────────────┴─────────────┴──────────────────┘
```

### 3.2 视觉规范

| 元素 | 规范 |
|------|------|
| 品牌色 | #168cff |
| 背景色 | #f5f7fa |
| 卡片背景 | #ffffff |
| 边框色 | #e5e6eb |
| 文字主色 | #171a1d |
| 文字次色 | #999999 |
| 圆角 | 8px |
| 间距 | 12-16px |

### 3.3 交互规范

| 场景 | 行为 |
|------|------|
| 点击部门 | 加载并显示部门成员 |
| 点击成员卡片 | 右侧显示详情 |
| 点击发消息 | 跳转到聊天面板 |
| 悬停成员卡片 | 背景变浅 + 轻微上浮 |
| 选中成员 | 蓝色边框高亮 |
