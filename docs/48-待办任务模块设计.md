## 二、后端实现 (JDK 1.8 规范)

### 2.1 待办核心逻辑

```java
/**
 * 待办任务流转核心服务
 * 负责任务状态机的变更调度以及预警过期分配
 */
@Service
@RequiredArgsConstructor
public class ImTodoServiceImpl implements ImTodoService {

    private final ImTodoMapper todoMapper;

    /**
     * 更新用户的待办及流转状态
     * 遵循纯正 JDK 1.8 规范，严防非法语法穿透
     *
     * @param todoId 待办事项的全局唯一标识
     * @param status 目标枚举状态 (如 COMPLETED)
     */
    @Transactional(rollbackFor = Exception.class)
    public void updateStatus(Long todoId, String status) {
        ImTodo todo = todoMapper.selectById(todoId);
        if (Objects.nonNull(todo)) {
            todo.setStatus(status);
            todo.setUpdateTime(new Date());
            todoMapper.updateById(todo);
        }
    }
}
```

## 三、前端实现 (纯 JS 规范)

### 3.1 待办管理逻辑封装

```javascript
/**
 * 待办管理逻辑封装 (纯 JS)
 */
import { ref } from 'vue';
import { todoApi } from '@/api/todo';

export function useTodo() {
  const todoList = ref([]);

  const loadTodoList = async () => {
    const response = await todoApi.list();
    todoList.value = response.list || [];
  };

  const toggleComplete = async (todoId) => {
    await todoApi.toggle(todoId);
    await loadTodoList();
  };

  return {
    todoList,
    loadTodoList,
    toggleComplete
  };
}
```

## 四、UI 表现与对齐规范 (DingTalk Style)

### 4.1 待办列表项 (Todo Item)
- **高度与间距**：列表项高度自适应，最小 `48px`。上下项之间提供 `1px` 极浅灰分割线。
- **复选框 (Checkbox) 交互**：
  - 未完成：`16x16` 的空心圆圈，边框 `#ADB1B8`。
  - Hover：空心圆圈边框变蓝 `#0089FF`。
  - 已完成：变为带有白色 `√` 的蓝色实心圆 `#0089FF`。

### 4.2 状态呈现 (States)
- **已完成文字**：一旦勾选，主体文字立即变为浅灰色 `rgba(23, 26, 29, 0.4)`，并加上居中的中划线（Strikethrough）。
- **逾期预警 (Overdue)**：若截止时间小于当前时间且未完成，时间标签文字强制变为红色 `#FF4D4F`。
