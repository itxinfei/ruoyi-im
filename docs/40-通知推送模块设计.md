## 二、前端实现 (纯 JS 规范)

### 2.1 通知管理服务封装

```javascript
/**
 * 通知管理服务 (NotificationService)
 * 强制要求：禁止使用 TypeScript，遵循纯 JS 规范
 */
export const notificationService = {
  permission: 'default',
  audioContext: null,
  notificationSound: null,

  /**
   * 请求通知权限
   */
  async requestPermission() {
    if (!('Notification' in window)) {
      return false;
    }
    const permission = await Notification.requestPermission();
    this.permission = permission;
    return permission === 'granted';
  },

  /**
   * 发送桌面通知
   */
  async notify(options, conversationId) {
    // 1. 检查权限
    if (this.permission !== 'granted') {
      const granted = await this.requestPermission();
      if (!granted) return null;
    }

    // 2. 检查免打扰 (armor logic)
    if (this.isInMuteTime()) {
      return null;
    }

    // 3. 窗口聚焦时不发送
    if (document.hasFocus()) {
      return null;
    }

    // 4. 创建通知实例
    const noticePayload = options.payload || {};
    const notification = new Notification(options.title, {
      body: options.body || '您有新消息',
      icon: options.icon || '/logo.png',
      tag: options.tag,
      data: noticePayload
    });

    // 监听点击
    notification.onclick = () => {
      this.processNotificationClick(notification, noticePayload);
    };

    return notification;
  },

  /**
   * 检查是否在免打扰时间
   */
  isInMuteTime() {
    const now = new Date();
    const currentTime = now.getHours() * 60 + now.getMinutes();
    // 假设从 store 获取配置 (C-003 相关)
    const startTime = 22 * 60; // 22:00
    const endTime = 8 * 60;   // 08:00
    
    if (startTime < endTime) {
      return currentTime >= startTime && currentTime <= endTime;
    }
    return currentTime >= startTime || currentTime <= endTime;
  },

  /**
   * 处理通知点击与跳转
   */
  processNotificationClick(notification, payload) {
    notification.close();
    window.focus();

    if (payload.type === 'message') {
      // 触发全局事件进行路由跳转
      window.dispatchEvent(new CustomEvent('im-nav', { 
        detail: { conversationId: payload.conversationId } 
      }));
    }
  }
};
```

## 三、后端实现 (JDK 1.8 规范)

### 3.1 通知异步分发 (EventBus)

```java
@Service
@RequiredArgsConstructor
public class ImNotificationServiceImpl implements ImNotificationService {

    private final SimpMessagingTemplate messagingTemplate;

    /**
     * 异步发送系统通知
     * 遵循 JDK 1.8 规范，严禁 var
     */
    @Async("noticeExecutor")
    public void sendSystemNotice(Long userId, String content) {
        NotificationVO noticeVO = new NotificationVO();
        noticeVO.setUserId(userId);
        noticeVO.setContent(content);
        noticeVO.setCreateTime(System.currentTimeMillis());
        
        // 推送 WebSocket 帧 (Doc-34)
        messagingTemplate.convertAndSendToUser(
            userId.toString(), 
            "/queue/notice", 
            noticeVO
        );
    }
}
```

## 四、UI 表现与对齐规范 (DingTalk Style)

### 4.1 应用内消息提示 (In-App Toast)
- **位置**：屏幕右上角弹出。
- **形态**：纯白背景 `#FFFFFF`，圆角 `8px`，阴影 `0 4px 16px rgba(0,0,0,0.1)`。
- **内容排版**：左侧头像（`32x32` 圆角 `4px`），右侧分上下两行（第一行发送者与时间，第二行消息摘要）。
- **交互**：停留 5 秒后自动消失；Hover 时暂停计时。点击卡片跳转到对应会话。

### 4.2 免打扰状态表现 (Mute Status)
- **会话列表**：开启免打扰的会话，右侧显示浅灰色（`#ADB1B8`）的“斜线铃铛”图标。
- **未读角标**：免打扰会话的未读数字不显示，仅在对应头像右上角显示一个 `8x8` 的红点（小圆点，无数字）。
