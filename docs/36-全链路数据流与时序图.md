# 36 全链路数据流与时序图 (Full‑Stack Data Flow)

> **定位**：定义 IM 核心业务在全栈环境下的物理流向，锁定前后端、Redis 与 WS 的每一处交互。

---

## 1. 文本消息发送时序 (P0 – Message Flow)

```mermaid
sequenceDiagram
    participant UserA as 前端 (User A)
    participant WS as WebSocket 网关
    participant API as 后端 API (Java)
    participant DB as MySQL / Redis
    participant UserB as 前端 (User B)

    UserA->>UserA: 1. UI 创建 PENDING 消息 (clientMsgId)
    UserA->>WS: 2. 发送 JSON 数据帧 (requestId)
    WS->>API: 3. 解析消息并路由至业务层
    API->>DB: 4. 插入 im_message (status=SENT), 更新会话最后时间
    API-->>WS: 5. 业务处理成功
    WS-->>UserA: 6. 返回 MSG_ACK (requestId, msgId)
    UserA->>UserA: 7. 更新本地消息状态 PENDING -> SENT
    WS->>UserB: 8. 推送 MSG_RECEIVE 数据帧
    UserB->>UserB: 9. 渲染消息, 增加未读数
```

## 2. 语音转文字业务流 (P1 – Audio AI Flow)

1. **录制**：前端使用本地 `MediaRecorder` 录制 OGG/MP3。\
2. **上传**：调用 `POST /api/file/upload` 存入本地文件系统。\
3. **发送**：WS 发送 `VOICE` 消息，附带 `fileUrl`。\
4. **异步触发**：后端收到语音消息后，触发 `SpeechToTextTask`。\
5. **通知**：转写完成后，通过 WS 推送 `MSG_UPDATE_EXT`（包含转写文字），前端气泡自动显示文字。

## 3. 文件上传与预览流 (P2 – File Upload Flow)

```mermaid
sequenceDiagram
    participant User as 前端
    participant API as 后端 API
    participant DB as MySQL / Redis
    participant WS as WebSocket
    participant Storage as 文件存储服务

    User->>API: 1. 选择文件, 调用 POST /api/file/upload
    API->>Storage: 2. 将文件流写入对象存储, 返回 fileUrl
    API->>DB: 3. 记录文件元数据 (fileId, url, size)
    API-->>User: 4. 返回 fileId, fileUrl
    User->>WS: 5. 发送 FILE 消息, 包含 fileId
    WS->>API: 6. 解析并转发至业务层
    API->>DB: 7. 更新会话消息表, status=SENT
    API-->>WS: 8. 业务处理成功
    WS-->>User: 9. 推送 FILE_RECEIVE, 包含 fileUrl
    User->>User: 10. 渲染文件预览卡片
```

## 4. 群聊消息广播流 (P3 – Group Chat Flow)

```mermaid
sequenceDiagram
    participant Sender as 前端 (发送者)
    participant WS as WebSocket
    participant API as 后端 API
    participant DB as MySQL / Redis
    participant Receiver1 as 前端 (成员 1)
    participant Receiver2 as 前端 (成员 2)

    Sender->>WS: 1. 发送 GROUP_MSG 数据帧
    WS->>API: 2. 解析并写入 im_message (group_id)
    API->>DB: 3. 插入消息, 更新群会话最新时间
    API-->>WS: 4. 返回 ACK
    WS->>Receiver1: 5. 推送 GROUP_MSG_RECEIVE
    WS->>Receiver2: 5. 推送 GROUP_MSG_RECEIVE
    Receiver1->>Receiver1: 6. 渲染消息, 更新未读计数
    Receiver2->>Receiver2: 6. 同上
```

## 5. 音视频通话建立流 (P4 – Call Setup Flow)

```mermaid
sequenceDiagram
    participant Caller as 前端 (呼叫方)
    participant Callee as 前端 (被叫方)
    participant Signaling as 信令服务器 (WebSocket)
    participant MediaServer as 媒体服务器

    Caller->>Signaling: 1. 发起 CALL_OFFER (SDP)
    Signaling->>Callee: 2. 转发 CALL_OFFER
    Callee->>Signaling: 3. 回复 CALL_ANSWER (SDP)
    Signaling->>Caller: 4. 转发 CALL_ANSWER
    Caller->>MediaServer: 5. 建立 ICE 连接, 发送媒体流
    Callee->>MediaServer: 6. 同上, 双向媒体流建立
    MediaServer-->>Caller: 7. 媒体流转发
    MediaServer-->>Callee: 8. 同上
```

## 6. WebSocket 重连与心跳 (P5 – WS Reconnection)

```mermaid
sequenceDiagram
    participant Client as 前端
    participant WS as WebSocket
    participant Server as 后端 WS 服务

    Client->>WS: 1. 建立连接
    WS->>Client: 2. 心跳 ping/pong (30s)
    alt 心跳超时
        Client->>WS: 3. 关闭连接, 发起重连
        Client->>WS: 4. 新的 WebSocket 连接
        WS->>Server: 5. 重新握手, 发送 token
        Server-->>Client: 6. 认证成功, 恢复会话状态
    end
```

## 7. 全局异常处理流

- **逻辑**：所有异常统一经过 `GlobalExceptionHandler`。\
- **契约**：必须返回 `docs/37` 定义的 5 位错误码。\
- **动作**：前端 Axios 拦截器捕获非 200 返回值，自动弹出 `ElMessage` 并停止 Loading。
