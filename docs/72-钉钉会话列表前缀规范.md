# 72 钉钉会话列表前缀规范 (Session Preview Prefix Spec)

> **用途**：定义会话列表"消息预览"前缀的显示规则。这是钉钉桌面端"一眼看出会话状态"的关键设计。
> **权威依据**：docs/00-钉钉8.2像素级权威基线.md §2.6 / 视觉资产 #18-draft-prefix.png #19-prefix-types.png
> **优先级**：⭐⭐ 高（影响信息密度感知）

---

## 1. 设计目标

钉钉的会话列表能让用户**不点开会话就知道当前状态**：
- 是否有 @ 提醒？
- 是否有未发送的草稿？
- 最后一条消息是图片/文件/语音/视频？
- 是否被人 DING 了？

这种"前缀提示"让会话列表信息密度比微信高 30%，是钉钉桌面端的核心识别要素。

---

## 2. 前缀类型完整枚举

### 2.1 优先级排序（同时满足多个时按此顺序显示）

```
优先级从高到低：
  1. [DING]            (收到 DING 强提醒，最高优先级)
  2. [有人@我]          (群聊有 @ 提醒)
  3. [草稿]            (输入框有未发送内容)
  4. [媒体类型]         (图片/文件/语音/视频/红包等)
  5. (无前缀)          (普通文本消息)
```

### 2.2 前缀完整规范表

| 前缀文本 | 触发条件 | 文字色 | 字重 | CSS 类 |
|---------|---------|--------|-----|---------|
| [DING] | 收到 DING 消息且未读 | #FF4D4F | 600 | .prefix-ding |
| [有人@我] | 群聊有 @ 提醒且未读 | #FF7D00 | 600 | .prefix-mention |
| [草稿] | 输入框有未发送文字 | #F53F3F | 600 | .prefix-draft |
| [图片] | 最后消息是 IMAGE | #8A9099 | 400 | .prefix-meta |
| [文件] | 最后消息是 FILE | #8A9099 | 400 | .prefix-meta |
| [语音] | 最后消息是 VOICE | #8A9099 | 400 | .prefix-meta |
| [视频] | 最后消息是 VIDEO | #8A9099 | 400 | .prefix-meta |
| [位置] | 最后消息是 LOCATION | #8A9099 | 400 | .prefix-meta |
| [名片] | 最后消息是 CARD | #8A9099 | 400 | .prefix-meta |
| [语音通话] | 最后消息是通话记录 | #8A9099 | 400 | .prefix-meta |
| [视频通话] | 最后消息是视频通话 | #8A9099 | 400 | .prefix-meta |

### 2.3 颜色心智模型

| 颜色 | 含义 | 用于哪些前缀 |
|-----|------|-------------|
| #FF4D4F 红 | "需要立即注意" | DING |
| #FF7D00 橙 | "需要回复" | @我 |
| #F53F3F 红 | "本人有事项" | 草稿、红包、转账 |
| #8A9099 灰 | "中性提示" | 媒体类型 |

## 3. Vue 组件实现规范

### 3.1 模板结构

会话列表项的预览区结构：

```
<div class="session-preview">
  <!-- 优先级最高的前缀（最多一个生效）-->
  <span v-if="prefixType === DING" class="prefix prefix-ding">[DING]</span>
  <span v-else-if="hasMentionMe" class="prefix prefix-mention">[有人@我]</span>
  <span v-else-if="draftContent" class="prefix prefix-draft">[草稿]</span>
  <span v-else-if="mediaPrefix" class="prefix prefix-meta">{{ mediaPrefix }}</span>
  <span v-else-if="moneyPrefix" class="prefix prefix-redpacket">{{ moneyPrefix }}</span>

  <!-- 群聊场景下：发送者昵称: 内容 -->
  <span v-if="isGroup && lastMessage.senderName" class="sender-prefix">
    {{ lastMessage.senderName }}: 
  </span>

  <!-- 内容主体 -->
  <span class="preview-text">{{ previewText }}</span>
</div>
```

### 3.3 业务逻辑（computed）

```javascript
import { computed } from 'vue'

const props = defineProps({ session: Object })

// 媒体前缀
const mediaPrefix = computed(() => {
  const map = {
    IMAGE: '[图片]',
    FILE: '[文件]',
    VOICE: '[语音]',
    VIDEO: '[视频]',
    LOCATION: '[位置]',
    CARD: '[名片]',
    AUDIO_CALL: '[语音通话]',
    VIDEO_CALL: '[视频通话]'
  }
  return map[props.session.lastMessageType] || null
})

// 金钱类前缀
const moneyPrefix = computed(() => {
  const map = {
    RED_PACKET: '[红包]',
    TRANSFER: '[转账]'
  }
  return map[props.session.lastMessageType] || null
})

// 是否有 DING 未读
const prefixType = computed(() => {
  if (props.session.unreadDingCount > 0) return 'DING'
  return null
})

// 预览文本
const previewText = computed(() => {
  // 草稿优先
  if (props.session.draftContent) {
    return props.session.draftContent
  }
  // 媒体类消息显示空（前缀已表达）
  if (mediaPrefix.value || moneyPrefix.value) {
    return ''
  }
  // 普通消息显示内容
  return props.session.lastMessage || ''
})
</script>
```

## 4. "正在输入"状态指示

### 4.1 视觉规范

当对方正在输入时，会话列表预览区临时切换为"正在输入..."提示：

| 状态 | 显示 | 颜色 |
|-----|------|------|
| 单聊对方正在输入 | "对方正在输入..." | var(--dt-brand-color) |
| 群聊有人正在输入 | "张三正在输入..." | var(--dt-brand-color) |

### 4.2 实现要点

- 通过 WebSocket 推送 typing 事件
- 状态持续 5 秒后自动消失（无新事件）
- 优先级高于其他前缀（仅低于 DING）
- 字体使用斜体效果让用户感知"动态"

```scss
.preview-typing {
  color: var(--dt-brand-color);
  font-style: italic;
  font-weight: var(--dt-font-weight-medium);
}
```

### 4.3 优先级更新（含 typing）

```
1. [DING]
2. typing 状态（高于 @ 提醒，因为是实时态）
3. [有人@我]
4. [草稿]
5. [媒体类型]
```

## 6. 反例（禁止）

| 反例 | 错误 | 正确 |
|-----|------|------|
| @ 前缀写蓝色 | color: var(--dt-brand-color) | color: var(--dt-mention-color) |
| 草稿前缀写普通字重 | font-weight: 400 | font-weight: 600 |
| 多前缀同时显示 | "[草稿][图片]文字" | 仅显示优先级最高的一个 |
| 媒体前缀加粗 | font-weight: 600 | font-weight: 400 (普通字重) |
| 不显示发送者昵称 | "你好" (群聊看不出谁说) | "张三: 你好" |
