## 二、核心表结构

### 2.1 用户表 (im_user)

```sql
CREATE TABLE im_user (
  id            BIGINT(20)    PRIMARY KEY AUTO_INCREMENT COMMENT '用户ID',
  username      VARCHAR(50)   NOT NULL UNIQUE COMMENT '用户名',
  password      VARCHAR(100)  NOT NULL COMMENT '密码(BCrypt加密)',
  nickname      VARCHAR(50)   NOT NULL COMMENT '昵称',
  avatar        VARCHAR(200)  NULL COMMENT '头像URL',
  gender        TINYINT(1)    DEFAULT 0 COMMENT '性别: 0保密 1男 2女',
  mobile        VARCHAR(20)   NULL COMMENT '手机号',
  email         VARCHAR(100)  NULL COMMENT '邮箱',
  status        TINYINT(1)    DEFAULT 1 COMMENT '状态: 0禁用 1正常',
  role          VARCHAR(20)   DEFAULT 'USER' COMMENT '角色',
  signature     VARCHAR(200)  NULL COMMENT '个性签名',
  department    VARCHAR(100)  NULL COMMENT '部门',
  position      VARCHAR(100)  NULL COMMENT '职位',
  receipt_enabled TINYINT(1)   DEFAULT 1 COMMENT '已读回执开关: 1开启 0关闭 (C-002)',
  clear_unread_mode TINYINT(1) DEFAULT 0 COMMENT '未读清零规则: 0进入即清零 1已读回执清零 (C-003)',
  last_online_time DATETIME   NULL COMMENT '最后在线时间',
  birthday      DATE          NULL COMMENT '生日',
  create_time   DATETIME      DEFAULT CURRENT_TIMESTAMP,
  update_time   DATETIME      DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);
```

**字段映射**：
| 数据库字段 | 实体字段 | 类型 | 说明 |
|-----------|---------|------|------|
| id | userId / id | Long | 主键 |
| username | username | String | 登录用户名 |
| nickname | nickname | String | 显示昵称 |
| avatar | avatar | String | 头像URL |
| department | departmentName | String | 部门名称 |
| position | position | String | 职位 |
| mobile | mobile | String | 手机号 |

### 2.2 会话表 (im_conversation)

```sql
CREATE TABLE im_conversation (
  id                BIGINT(20)    PRIMARY KEY AUTO_INCREMENT COMMENT '会话ID',
  type              VARCHAR(20)   NOT NULL COMMENT '类型: PRIVATE私聊 GROUP群聊',
  target_id         BIGINT(20)    NULL COMMENT '目标ID(私聊为对方用户ID,群聊为群组ID)',
  name              VARCHAR(100)  NULL COMMENT '会话名称',
  avatar            VARCHAR(200)  NULL COMMENT '会话头像',
  last_message_id   BIGINT(20)    NULL COMMENT '最后一条消息ID',
  last_message_time DATETIME      NULL COMMENT '最后消息时间',
  is_deleted        TINYINT(1)    DEFAULT 0 COMMENT '是否删除',
  is_sensitive      TINYINT(4)    DEFAULT 0 COMMENT '是否敏感会话',
  no_forward        TINYINT(4)    DEFAULT 0 COMMENT '禁止转发',
  no_copy           TINYINT(4)    DEFAULT 0 COMMENT '禁止复制',
  create_time       DATETIME      DEFAULT CURRENT_TIMESTAMP,
  update_time       DATETIME      DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);
```

**字段映射**：
| 数据库字段 | 实体字段 | 类型 | 说明 |
|-----------|---------|------|------|
| id | id / conversationId | Long | 会话ID |
| type | type | String | PRIVATE/GROUP |
| target_id | targetId | Long | 目标ID |
| name | name | String | 会话名称 |

### 2.4 消息表 (im_message)

```sql
CREATE TABLE im_message (
  id                      BIGINT(20)    PRIMARY KEY AUTO_INCREMENT COMMENT '消息ID',
  client_msg_id           VARCHAR(64)   NULL COMMENT '客户端消息ID(去重)',
  send_status             VARCHAR(20)   DEFAULT 'PENDING' COMMENT '发送状态: PENDING/SENDING/SENT/DELIVERED/FAILED/REVOKED',
  send_retry_count        INT(11)       DEFAULT 0 COMMENT '发送重试次数',
  send_error_code         VARCHAR(50)   NULL COMMENT '发送错误码',
  send_error_msg          VARCHAR(500)  NULL COMMENT '发送错误信息',
  delivered_time          DATETIME      NULL COMMENT '送达时间',
  conversation_id         BIGINT(20)    NOT NULL COMMENT '会话ID',
  sender_id               BIGINT(20)    NOT NULL COMMENT '发送者ID',
  message_type            VARCHAR(20)   NOT NULL COMMENT '消息类型: TEXT/IMAGE/VIDEO/AUDIO/FILE/LINK/VOICE',
  content                 TEXT          NULL COMMENT '消息内容',
  voice_text              TEXT          NULL COMMENT '语音转文字结果 (C-004)',
  reply_to_message_id     BIGINT(20)    NULL COMMENT '回复的消息ID',
  reply_snapshot          VARCHAR(500)  NULL COMMENT '被回复消息的内容快照',
  at_user_ids             VARCHAR(500)  NULL COMMENT '@提醒用户ID列表(JSON数组)',
  forward_from_message_id BIGINT(20)    NULL COMMENT '转发来源消息ID',
  file_url                VARCHAR(500)  NULL COMMENT '文件URL',
  file_name               VARCHAR(200)  NULL COMMENT '文件名',
  file_size               BIGINT(20)    NULL COMMENT '文件大小(字节)',
  sensitive_level         VARCHAR(20)   DEFAULT 'NORMAL' COMMENT '敏感级别',
  is_revoked              TINYINT(1)    DEFAULT 0 COMMENT '是否撤回',
  is_edited               TINYINT(1)    DEFAULT 0 COMMENT '是否已编辑',
  edited_content          TEXT          NULL COMMENT '编辑后的内容',
  edit_count              INT(11)       DEFAULT 0 COMMENT '编辑次数',
  edit_time               DATETIME      NULL COMMENT '最后编辑时间',
  is_deleted              TINYINT(1)    DEFAULT 0 COMMENT '是否删除',
  revoked_time            DATETIME      NULL COMMENT '撤回时间',
  revoker_id              BIGINT(20)    NULL COMMENT '撤回者ID',
  is_encrypted            TINYINT(4)    DEFAULT 0 COMMENT 'E2E加密标志',
  version                 INT(11)       DEFAULT 0 COMMENT '版本号(乐观锁)',
  create_time             DATETIME      DEFAULT CURRENT_TIMESTAMP,
  update_time             DATETIME      NULL,
  INDEX idx_conversation_time (conversation_id, create_time),
  INDEX idx_sender_id (sender_id),
  FULLTEXT INDEX ft_content_cn (content) WITH PARSER ngram
);
```

**消息类型枚举**：

| TEXT | 文本消息 |
| IMAGE | 图片消息 |
| FILE | 文件消息 |
| VIDEO | 视频消息 |
| AUDIO | 音频消息 |
| VOICE | 语音消息 |
| LINK | 链接卡片消息 |
| SYSTEM | 系统消息 |
| RECALLED | 撤回消息 |

**发送状态枚举**：

| SENDING | 发送中 |

## 十、性能与存储专项

### 8.1 全文检索规范 (MySQL 5.7)
- **要求**：`im_message` 表的 `content` 字段必须建立 `FULLTEXT` 索引。
- **配置**：必须指定 `ngram` 分词器，以支持中文字符检索。
- **AI 指令**：在生成建表 SQL 时，必须包含 `FULLTEXT INDEX ft_content (content) WITH PARSER ngram`。

### 8.2 内网文件存储路径映射
- **物理根目录**：`/home/im/uploads/` (Linux) 或 `D:/im/uploads/` (Windows)。
- **访问 URL**：统一映射为 `/api/files/display/{fileName}`。
- **AI 指令**：严禁在数据库存入绝对物理路径，只能存入相对路径（如 `avatar/2026/03/xx.png`），由后端 `ResourceHandler` 进行映射。
