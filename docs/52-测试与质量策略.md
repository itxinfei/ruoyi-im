# 52 测试与质量策略

> **用途**：为 IM 与平台模块制定完整的测试层级、质量门控以及 CI/CD 集成规范，确保交付质量符合企业级要求。

---

## 1. 验证层级与目标

| 层级 | 目标 | 关键检查点 |
|------|------|------------|
| **冒烟** | 基础功能可用，页面不崩溃 | 页面加载、关键交互、无 JS 错误 |
| **功能回归** | 所有核心业务路径稳定 | 参考 `54-验收用例清单.md`，覆盖发送/接收、群聊、文件、语音、通话 |
| **接口契约** | 前后端 API 一致性 | HTTP 状态码、返回字段、错误码匹配 |
| **性能/压力** | 响应时间 & 并发能力 | 滚动渲染 ≥55fps，消息发送 ≤200ms，WS 并发 ≥5000 连接 |
| **安全审计** | 数据与权限安全 | XSS/CSRF 防护、权限校验、敏感信息脱敏 |
| **可用性** | UI/UX 稳定性 | 交互流畅、无卡顿、无视觉错误 |

---

## 2. 自动化测试策略

1. **单元测试**（Vitest）
   - 业务函数、Vue 组合式 API、工具类
   - 覆盖率目标 **≥ 80%**（行/分支）
2. **组件/集成测试**（Vue Test Utils + Vitest）
   - 关键 UI 交互：消息发送、抽屉弹出、文件预览、通话面板
   - 使用 `@testing-library/vue` 进行可访问性检查
3. **端到端测试**（Playwright）
   - 场景：登录 → 发送文本 → 发送文件 → 语音转文字 → 发起通话 → 退出
   - 运行在 CI 中的 **Chrome** 与 **Headless** 模式
4. **性能基准**（k6）
   - 模拟并发用户发送消息、文件上传、WS 心跳
   - 报告响应时间、错误率、资源使用
5. **安全扫描**（OWASP ZAP）
   - 自动化扫描常见漏洞，生成报告并在 PR 中附加

## 3. CI/CD 集成

```yaml
name: CI
on: [push, pull_request]
jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - name: Setup Node
        uses: actions/setup-node@v3
        with:
          node-version: '16'
      - name: Install deps
        run: npm ci
      - name: Unit & Component Tests
        run: npm run test:ci   # vitest --coverage
      - name: E2E Tests
        run: npm run test:e2e   # playwright
      - name: Performance Test
        run: npm run test:perf   # k6 script
      - name: Security Scan
        run: npm run security:scan   # zap-cli
      - name: Upload Coverage
        uses: actions/upload-artifact@v3
        with:
          name: coverage-report
          path: coverage/
```

---

## 4. 测试环境与数据

- **独立测试库**：使用 Docker Compose 启动 MySQL、Redis、Mongo（日志）等，避免污染生产数据。
- **测试账号**：统一在 `test-data/users.json` 中维护，包含普通用户、管理员、群主等角色。
- **Mock 数据**：对外部服务（文件存储、语音转写）使用本地 Mock Server，确保 CI 可离线运行。

---

## 5. 回归与风险管理

- 每次功能交付后必须在 `03-实现状态矩阵.md` 记录 `✅ 冒烟 + ✅ 功能回归`。
- 高风险功能（通话、语音、文件）需额外编写 **失败场景** 用例（网络中断、权限错误、文件大小超限）。
- 若出现 **测试阻塞**（依赖服务不可用），需在 `04-待确认问题.md` 标记并提供临时方案。

---

> **业务背景**：本策略确保在企业内部部署的 IM 系统具备高可靠性、可维护性和安全合规性，满足业务连续性与合规审计需求。
