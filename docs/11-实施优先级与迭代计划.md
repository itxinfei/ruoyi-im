# 11 实施优先级与迭代计划

> 用途：把目标需求转成可持续推进的阶段计划。  
> 说明：本文件描述的是目标实施顺序，不等于当前已完成状态。当前实现状态以 [03-实现状态矩阵.md](03-实现状态矩阵.md) 为准。

## 1. 规划原则

1. 产品目标是“私有化 Web 版钉钉”，默认体验参考钉钉 Windows 客户端。
2. IM 是核心中的核心，任何迭代都不得削弱 IM 主链路。
3. 前端按 Vue3 项目处理，后端按单体 Java 项目处理。
4. 并发不是主要约束，优先保证功能闭环、界面清晰、维护简单。
5. 左侧导航展示出来的一级模块，最终都必须有真实页面和可用主流程。

## 2. 总体阶段

### 2.1 Phase 1：IM 主链路可用

目标：

1. 让用户可以稳定完成“找会话 -> 看消息 -> 发消息 -> 搜索 -> 群协作”的主路径。
2. 让消息页面的布局、信息密度、交互节奏明显靠近钉钉 Windows 客户端。

范围：

1. 会话列表
2. 聊天区
3. 输入区
4. 搜索
5. 群详情
6. 群文件
7. 通话入口

完成标志：

1. 新用户进入消息页面后，不需要学习成本即可完成基本沟通。
2. 页面不再存在“开发中”死路入口。
3. 视觉和交互不再明显偏移动端或临时后台页面风格。

### 2.2 Phase 2：IM 协作增强

目标：

1. 补齐消息收藏、链接卡片、语音消息、通话闭环等高频增强能力。
2. 让 IM 不只是基础聊天，而是可承载企业内高频协作。

范围：

1. 消息收藏
2. 链接卡片
3. 语音消息
4. 语音通话
5. 视频通话

完成标志：

1. 高频消息能力形成闭环。
2. 核心能力有稳定入口、异常反馈和回退策略。

### 2.3 Phase 3：平台核心协同模块

目标：

1. 补齐左侧导航 P0/P1 模块，让产品从“聊天系统”升级为“协同平台”。

范围：

1. 通讯录
2. 工作台
3. 云盘/文档
4. 日历
5. 待办
6. 审批

完成标志：

1. 左侧导航核心模块均有真实落地页。
2. 每个模块至少有一个可验证主流程。

### 2.4 Phase 4：平台配套完善

目标：

1. 补齐平台配套模块，提升整体完整度。

范围：

1. 邮箱
2. AI 助手
3. 设置
4. 个人资料
5. 管理入口

完成标志：

1. 主要一级模块均具备可用入口。
2. 平台在企业内网场景下可作为统一办公入口使用。

## 3. IM 详细建设顺序

### 3.1 第一批

1. 会话列表对齐钉钉
2. 聊天头部与消息列表对齐钉钉
3. 输入区与发送交互闭环
4. 搜索闭环

原因：

1. 这是消息模块最核心的高频路径。
2. 用户几乎每天都会反复使用。

### 3.2 第二批

1. 群详情重构
2. 群文件闭环
3. 收藏闭环
4. 链接卡片闭环

原因：

1. 这些能力直接影响企业协作效率。

### 3.3 第三批

1. 语音消息
2. 语音通话
3. 视频通话

原因：

1. 当前代码链路部分存在，但联调与完整闭环仍未完全确认。
2. 这类功能要在主链路稳定后推进，避免分散资源。

## 5. 每阶段交付要求

1. 每次只交付单个主目标，不同时推进多个大闭环。
2. 每个交付必须有真实入口、主流程、异常流程和验收标准。
3. UI 对齐优先解决布局、信息层级、交互节奏，不追求品牌像素级复刻。
4. 不允许为了低并发项目引入高复杂度分布式设计。
5. 每轮交付后必须回写 [03-实现状态矩阵.md](03-实现状态矩阵.md) 和 [04-待确认问题.md](04-待确认问题.md)。
