# 24 页面缺陷与功能缺口清单

> 用途：统一记录"页面不完善、功能有问题"的具体缺陷，避免口头反馈丢失。
> 说明：每个问题都应绑定 `task_id`；未确认事实写 `unknown`。

## 1. 记录规范

1. 每条问题使用固定格式：`模块`、`页面/组件`、`问题描述`、`影响级别`、`建议任务`、`状态`。
2. 影响级别分为 `P0`（阻断主流程）、`P1`（影响高频体验）、`P2`（可延期优化）。
3. 问题修复后，状态从 `open` 改为 `done`，并在 `notes` 写明对应提交或变更说明。

## 3. 更新流程

1. 用户或测试反馈新问题后，先在本表新增行，再补充到 `llm-tasks.yaml`。
2. 若问题涉及 UI 复刻，必须同时引用 `16-设计复刻差异.md`。
3. 若问题涉及功能闭环，必须同时引用 `17-接口与入口清单.md` 与 `52-测试与质量策略.md`。
4. 每周至少更新一次本表状态，避免“任务已改、文档未更新”。
