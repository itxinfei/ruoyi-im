# 15 产品形态与导航清单

> 用途：明确本项目的目标产品形态、左侧导航能力范围和模块优先级。
> 说明：本文件描述的是目标需求，不等于当前已实现状态。当前实现状态以 [03-实现状态矩阵.md](03-实现状态矩阵.md) 为准。

## 1. 目标产品定义

本项目的目标不是“做一个有 IM 功能的后台系统”，而是：

1. 做一个可内网部署、私有化交付的 Web 版钉钉
2. 参考钉钉 Windows 客户端的功能结构、导航布局、页面层级、交互方式和视觉风格
3. 在企业内网场景下提供一套可日常使用的协同平台

核心判断标准：

1. 用户是否可以把它当成“企业内部的钉钉 Web 端”使用
2. IM 是否构成平台核心，而不是附属模块
3. 左侧导航是否具备完整协同入口，而不是只保留几个孤立页面

## 2. 对标原则

### 2.1 必须对标的内容

1. 左侧导航的信息架构
2. 消息主链路的页面布局和操作逻辑
3. 搜索、会话、聊天、群详情、通话、文件等高频协作路径
4. 桌面办公场景下的模块入口密度和交互层级

### 2.2 不要求逐像素复刻的内容

1. 钉钉品牌视觉素材
2. 非当前业务需要的全部外围生态能力
3. 与内网私有化场景不匹配的开放平台能力

### 2.3 目标口径

“钉钉有的功能都要有”在本项目中的解释是：

1. 左侧导航上的核心协同模块应完整具备
2. 每个模块至少要有真实入口和可用主流程
3. 当前版本允许分阶段完成，但最终目标不应只停留在 IM 单模块

## 3. 左侧导航目标清单

以下清单用于定义“目标形态下左侧导航应具备哪些一级能力”。  
具体实现优先级仍由 [10-产品需求总纲.md](10-产品需求总纲.md) 和 [13-需求明细.md](13-需求明细.md) 控制。

### 3.1 核心模块

1. 消息
2. 搜索
3. 通讯录
4. 工作台

### 3.2 高频协同模块

1. 云盘/文档
2. 日历
3. 待办
4. 审批
5. 邮箱

### 3.3 辅助模块

1. AI 助手或智能助手
2. 设置
3. 个人资料

### 3.4 管理与运维模块

1. 后台管理
2. 组织和权限管理入口

## 4. 模块优先级

### 4.1 P0 平台核心

1. 消息
2. 搜索
3. 通讯录

说明：

1. 这三者共同构成“找人 -> 找会话 -> 沟通”的闭环
2. 其中消息是核心中的核心

### 4.2 P1 高价值协同

1. 工作台
2. 云盘/文档
3. 日历
4. 待办
5. 审批

说明：

1. 这些模块让产品从“聊天工具”升级为“协同平台”

### 4.3 P2 增强与配套

1. 邮箱
2. AI 助手
3. 后台管理
4. 设置和个人资料的精细化能力

## 5. 左侧导航验收要求

### 5.1 结构要求

1. 一级导航必须稳定存在
2. 一级导航项顺序应服务办公主路径
3. 模块之间不应出现明显割裂风格

### 5.2 行为要求

1. 每个左侧导航项都必须有真实落地页
2. 不允许点击后进入空白页
3. 不允许保留“开发中”死路入口作为已交付模块

### 5.3 视觉要求

1. 整体风格应统一参考钉钉 Windows 端
2. 导航图标、密度、状态反馈保持统一
3. 不出现偏移动端化的菜单视觉

## 6. IM 与其他模块的关系

1. IM 是平台主引擎
2. 搜索、文件、群协作、通话都属于 IM 的扩展主链路
3. 工作台、审批、待办、日历等模块是平台协同层
4. 任何设计和开发决策都不能削弱 IM 的主地位

## 7. 待确认项

1. 需要后续明确“钉钉 Windows 客户端”具体参考版本，避免后续因版本差异造成范围漂移
2. 左侧导航最终是否要求覆盖钉钉所有一级模块，还是覆盖企业内网常用模块的完整子集
