## 一、需求总览

### 1.1 项目定位

RuoYi-IM 是面向企业内网场景的即时通讯与协同平台，产品形态对标钉钉 Windows 客户端。

### 1.2 需求分层

| 层级 | 说明 | 优先级 |
|-----|------|-------|
| P0 | IM 核心能力，必须优先完成 | 最高 |
| P1 | 高频协同能力，支撑日常办公 | 高 |
| P2 | 增强能力，提升产品完整度 | 中 |
| P3 | 管理后台，面向管理员 | 中 |

## 六、技术需求

### 6.1 性能与安全需求 (C-019/C-020)

- 消息延迟 ≤300ms，WS 长连接维持心跳。
- 支持单频道 ≥500 人并发。
- 数据传输加密，JWT 安全周期（2h/7d）。
- **红线**：严禁硬编码 API 路径，严禁使用 TypeScript。

## 七、核心业务规则汇总 (C-Series)

| 编号 | 业务规则项 | 确认结论 |
|-----|-----------|---------|
| C-001 | 消息编辑时限 | 发送后 5 分钟 内允许编辑 |
| C-002 | 已读回执开关 | 单聊可关闭；群聊强制开启 |
| C-003 | 未读清零规则 | 用户可选（进入即清零/已读回执清零） |
| C-004 | 语音消息规格 | 最大时长 60秒；支持语音转文字 |
| C-005 | 群详情入口 | 统一使用右侧抽屉（GroupDetailDrawer） |
| C-006 | 群管理员权限 | 修改群名、编辑公告、禁言、移除成员 |
| C-007 | 消息撤回时限 | 发送后 2 分钟 内允许撤回 |
| C-008 | 文件上传限制 | 单文件最大 100MB；支持拖拽与粘贴 |
| C-018 | UI交互基准 | 严格遵循钉钉 Windows 8.2.0 |
