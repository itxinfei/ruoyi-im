# 59 群详情与群管理UI设计规范

> **用途**：定义群组详细资料、群成员管理及群设置的交互细节与 UI 标准。  
> **设计基准**：钉钉 Windows 8.2.0 桌面端，遵循 `docs/27-PC版钉钉UI设计蓝图.md`。

---

## 1. 页面定位与入口

*   **模块 ID**：`group_detail`
*   **页面模板**：详情抽屉 (`Right Drawer`)
*   **主要任务**：
    1.  展示群基本信息（群名、头像、群公告）。
    2.  群成员的增删（邀请/移除）与搜索。
    3.  群主/管理员的功能配置（禁言、群二维码、转让）。

---

## 2. 空间布局 (Layout)

### 2.1 右侧详情抽屉 (Group Drawer)
*   **尺寸**：固定宽度 `320px`，动画时长 `0.3s` 缓动滑出。
*   **层级结构**：
    *   **头部**：群头像（居中，48x48）、群名、成员计数。
    *   **成员网格**：前 9-12 位成员头像预览，带 `(+)` 邀请按钮。
    *   **设置列表**：按功能分组的菜单项列表。

### 2.2 菜单项规范 (Setting Item)
*   **高度**：`44px`。
*   **结构**：左侧图标/标题，右侧状态（开关/箭头/摘要文字）。
*   **样式**：Hover 变色，且带有极细底部分割线。

---

## 3. 组件级设计规范

### 3.1 群公告看板 (Group Announcement)
*   **交互**：点击群公告项 -> 弹出全屏蒙层弹窗 -> 支持富文本查看与编辑。
*   **提醒**：新公告发布后，聊天窗口顶部显示“橙色跑马灯”提醒。

### 3.2 成员管理列表 (Member Manager)
*   **入口**：点击“查看更多成员”。
*   **结构**：弹出全屏/大弹窗，带搜索框。
*   **权限标记**：群主显示“皇冠”图标，管理员显示“蓝盾”图标。

### 3.3 群二维码 (Group QR Code)
*   **样式**：卡片式，顶部带群头像和名称，中间二维码，底部“扫码加入”。
*   **交互**：支持“保存到本地”或“一键转发”。

---

## 4. 状态清单 (States)

| 状态 | 表现 |
|-----|------|
| **Muted** | 群列表显示“小喇叭斜线”图标，输入框显示“全员禁言中”。 |
| **Transferring** | 转让群主确认弹窗，带警告色强调风险。 |
| **Invite Success** | 弹出绿色 Toast“邀请已发送”。 |
| **Empty Members** | 搜索无结果时提示“未找到匹配成员”。 |

---

## 5. 交互路径 (Interactions)

1.  **修改群名**：点击群名 -> 直接进入内联编辑模式（Inline Edit） -> 失去焦点或按 Enter 自动保存。
2.  **移除成员**：点击头像右键 -> 弹出“移除出群组”确认。
3.  **消息清空**：点击“清空本地聊天记录” -> 弹出危险操作二次确认。

---

## 6. 验收点 (AC)

- [ ] AC1: 群详情抽屉支持滚动加载成员列表，不影响聊天主区流畅度。
- [ ] AC2: 管理员可见操作项与普通成员可见项严格隔离。
- [ ] AC3: 群头像支持点击查看高清大图。
- [ ] AC4: 所有 UI 颜色和圆角 100% 映射至 `design-tokens.scss`。
