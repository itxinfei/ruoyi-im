# 41 设置与个人资料模块UI设计规范

> **用途**：定义个人资料修改、系统设置（通知/隐私/通用）的页面布局与交互规范。  
> **设计基准**：钉钉 Windows 8.2.0 桌面端，遵循 `docs/27-PC版钉钉UI设计蓝图.md`。

---

## 1. 页面定位与模板

*   **模块 ID**：`profile`, `settings`
*   **页面模板**：表单/流程页 (`L1 + Main View`)
*   **主要任务**：
    1.  展示与编辑个人基本信息（头像、昵称、职位等）。
    2.  配置系统行为（消息提醒、主题切换、快捷键）。
    3.  管理账号安全与黑名单。

---

## 2. 空间布局 (Layout)

### 2.1 整体骨架
*   **侧边 Tab 导航**：宽度 `200px`，固定在左侧（或在 Main View 内作为二级 Tab）。
*   **内容区**：居中对齐，最大宽度 `800px`。
*   **操作区**：底部固定操作栏（保存/取消），带 `1px` 上边框。

### 2.2 模块划分
1.  **个人资料**：头像、姓名、工号、部门、手机、邮箱。
2.  **通知设置**：桌面通知开关、声音开关、消息预览开关。
3.  **通用设置**：语言切换、自动启动开关、暗色模式切换。
4.  **快捷键**：截图、全局搜索、弹出主窗口等热键自定义。

---

## 3. 组件级设计规范

### 3.1 资料项布局 (Info Item)
*   **头像项**：`64px * 64px`，圆角 `8px`。右侧带“修改头像”文字链接。
*   **标准项**：
    *   Label: `13px` Secondary，宽度 `100px`。
    *   Value: `14px` Primary。
    *   可编辑态：使用 Element Plus `el-input`，高度 `32px`。

### 3.2 开关项 (Switch Item)
*   **样式**：`flex-between` 布局。
*   **文字**：标题（14px Primary） + 描述（12px Tertiary）。
*   **组件**：使用 `el-switch`，开启色为 `--dt-brand-color`。

### 3.3 快捷键录入 (Hotkey Input)
*   **交互**：点击输入框进入“录入模式” -> 捕获键盘组合键 -> 显示为 `Ctrl + Alt + A` 样式标记。
*   **校验**：重复或冲突时，输入框背景变红并提示“快捷键已被占用”。

---

## 4. 状态清单 (States)

| 状态 | 表现 |
|-----|------|
| **Edited** | 页面检测到内容变更，底部操作栏由透明变为背景白色，显示“保存”按钮。 |
| **Saving** | 保存按钮进入 Loading 态，禁用所有表单项。 |
| **Success** | 保存后弹出顶部居中提示“设置已保存”。 |
| **Safe Verify** | 修改敏感信息（如密码/手机号）前，弹出二次验证弹窗。 |

---

## 5. 交互路径 (Interactions)

1.  **修改头像**：点击头像 -> 弹出本地文件选择器 -> 选图 -> 弹出简易裁剪器 -> 确认上传。
2.  **主题切换**：点击“暗色模式”开关 -> 全局切换 `html.dark` 类名 -> 状态即时持久化到 `localStorage`。
3.  **导航跳转**：点击左侧“通知” -> 页面平滑滚动至对应锚点，或切换二级路由。

---

## 6. 验收点 (AC)

- [ ] AC1: 所有设置项在页面刷新后能正确回显当前状态。
- [ ] AC2: 快捷键设置支持“一键重置为默认值”。
- [ ] AC3: 个人资料页支持“只读模式”与“编辑模式”的平滑切换。
- [ ] AC4: UI 严格遵循 `design-tokens.scss`，不出现硬编码颜色。
