# Java 后端子任务拆解（IM-API-001 ~ IM-API-004）

> 适用：`ruoyi-im-api`  
> 约束：JDK8、禁止魔法值、禁止复杂表达式链、优先可维护性

## IM-API-001 通话信令后端接口与状态存储

1. `API-001-A` 定义 DTO/VO 和错误码常量（禁止硬编码字符串）
2. `API-001-B` 新增 Controller 接口：
   - 发起通话
   - 接听
   - 拒绝
   - 挂断
3. `API-001-C` Service 实现状态机（邀请中/已接听/已拒绝/已结束）
4. `API-001-D` Mapper + XML 落库与查询（按会话ID、通话ID）
5. `API-001-E` 异常路径：
   - 重复接听
   - 非参与人操作
   - 会话不存在
6. `API-001-F` 联调验证：
   - 与 `CallDialog.vue` 状态一致
   - 编译命令通过

## IM-API-002 会话状态统一写入 im_user_session

1. `API-002-A` 审计现有 `ImConversationServiceImpl/ImMessageServiceImpl` 状态写路径
2. `API-002-B` 新增统一状态更新方法（置顶/免打扰/未读/已读游标）
3. `API-002-C` 迁移期双写（新表主写 + 旧表兼容回写）
4. `API-002-D` 统一查询优先级（优先 `im_user_session`，回退旧表）
5. `API-002-E` 增加空值与边界保护（未读不能负数）
6. `API-002-F` 联调验证：
   - 会话列表、会话详情、未读总数一致

## IM-API-003 消息收藏后端闭环

1. `API-003-A` 定义收藏请求/响应模型与分页 VO
2. `API-003-B` Controller：
   - 收藏
   - 取消收藏
   - 分页查询收藏列表
3. `API-003-C` Service 幂等逻辑（重复收藏不报错）
4. `API-003-D` Mapper/XML：
   - 收藏唯一约束查询
   - 分页排序（按收藏时间倒序）
5. `API-003-E` 错误处理：
   - 消息不存在
   - 权限不足
6. `API-003-F` 联调验证：
   - 与 `im-message.js` 收藏状态一致

## IM-API-004 文件上传与消息落库一致性

1. `API-004-A` 定义上传状态常量与失败码
2. `API-004-B` Service 事务策略：
   - 上传成功才写“发送成功”
   - 上传失败写“失败可重试”
3. `API-004-C` Mapper/XML 补齐发送状态字段更新
4. `API-004-D` 重试接口（限制最大重试次数，常量化）
5. `API-004-E` 异常恢复：
   - 文件存在但消息失败
   - 消息存在但文件失败
6. `API-004-F` 联调验证：
   - 前端上传进度/失败重试行为与后端状态一致

## 通用验收（每个子任务都要满足）

1. `mvn -DskipTests compile` 通过
2. 无魔法值（状态码/类型/阈值全部常量化）
3. 无多层三元、无复杂表达式链
4. 关键分支有日志上下文，可定位问题
5. 对外错误响应可读（错误码 + 信息）
