# 23 UI 复刻任务计划

> 用途：把"完美复刻钉钉 Windows 桌面体验"的 UI 工作拆成可落地的设计与开发任务。
> 说明：针对差距矩阵中的每个项建立一组任务，结合 `llm-tasks.yaml`，确保每条任务可跟踪、可验收、可回退。

## 1. 任务原则

1. 每个 UI 复刻任务必须引用 `21-IM桌面端页面规范.md` 与 `16-设计复刻差异.md`。
2. 任务 scope 仅覆盖一个页面/组件层级，优先对齐差距最高的 P0 项（会话列表、消息气泡、输入区、聊天头部）。
3. 任务必须同时填入 `manual_test_steps`、`acceptance`，并与 `54-验收用例清单.md` 保持一致。
4. 任务完成后，需在 `52-测试与质量策略.md` 记录验收级别、测试账号与环境。

## 2. 高优先任务

- **会话列表精简与状态对齐**：任务目标为取消临时标签、统一未读反馈、保证 240px 宽度/头像+气泡结构，并在 `16-设计复刻差异.md` 中打勾 P0 差距。
- **消息气泡与时间线复刻**：修正气泡对齐、最大宽度、间距、时间分割线样式，确保“气泡在前、头像在后”。
- **输入区与工具栏清理**：只保留真实入口、统一 Enter/Ctrl+Enter 行为，粘贴上传有页面反馈。
- **聊天头部精简+通话状态**：保持 60px 高度，按钮密度与钉钉一致，去除无效按钮/悬浮菜单。

## 3. 复刻任务流程

1. 选择对应 `task_id`（当前优先使用 `IM-UI-001` ~ `IM-UI-004`，功能断点优先使用 `IM-FIX-001` ~ `IM-FIX-003`）。
2. 在任务 `notes` 中引用 `16-设计复刻差异.md#2`，指出所对齐的差距项。
3. 设计稿+截图+色值按钉钉复刻完后，在任务 `acceptance` 中列明“视觉、布局、交互完全对齐钉钉 Windows”。
4. 完成后在 `54-验收用例清单.md` 标记对应条目，并更新 `52-测试与质量策略.md` 的验证记录。

## 4. 支持与审查

1. 设计/前端在 PR 中附上“设计差异复盘图”（当前 vs 钉钉，如对比图、表格）。  
2. 重复出现的差距（如气泡宽度、输入栏工具）在 `16-设计复刻差异.md` 中打上 `[已提交 task_id]` 标签。  
3. QA 要求重新运行 `verify_cmd` 包括 `npm run build` + 手动验收 `54-验收用例清单.md`。
