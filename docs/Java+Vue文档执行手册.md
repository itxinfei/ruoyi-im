# Java + Vue 文档执行手册（仅文档模式）

> 适用场景：你只希望大模型输出文档（需求、方案、任务拆解、验收），不改代码。  
> 固定基线：JDK8、Vue3、MySQL5.7、开发Win11、生产CentOS7。

## 1. 使用目标

1. 先把需求、范围、验收写清楚
2. 再让模型按文档产出任务拆解
3. 最后由你决定是否进入开发

## 2. 文档输出顺序（强制）

1. 需求文档更新（目标、范围、边界）
2. 技术方案文档（前后端架构、接口草案、数据表影响）
3. 任务清单文档（P0/P1/P2 + 负责人 + 验收）
4. 风险与回滚文档
5. 测试与验收文档

## 3. 给大模型的固定前置提示词

```text
你现在处于“仅文档模式”，禁止修改代码与SQL文件。
项目固定参数：JDK1.8、Vue3、MySQL5.7、开发Win11、生产CentOS7。
必须遵守：禁止魔法值、禁止复杂表达式写法、强调可维护性/可靠性/可用性。
输出内容必须是可执行文档，包含目标、范围、AC验收、风险、回滚。
```

## 4. 文档验收清单

1. 是否明确一级/二级/三级信息架构
2. 是否同时覆盖 Java 侧与 Vue 侧
3. 是否有 AC（至少3条）且可验证
4. 是否有失败路径和回滚策略
5. 是否标注依赖文档和任务ID

## 5. 推荐文档模板

1. 需求冻结：`docs/需求文档.md`
2. UI规范：`docs/UI对标设计规范.md`
3. LLM规范：`docs/大模型研发规范.md`
4. 双端规范：`docs/Java+Vue双端开发规范.md`
5. API拆解：`docs/Java后端子任务拆解.md`
6. 任务池：`docs/llm-tasks.yaml`

## 6. 常用文档任务指令（示例）

1. “只更新需求文档，不动代码，补全二级菜单和三级弹窗验收标准”
2. “只输出 Java 后端接口方案文档，包含 DTO/VO/错误码，不写代码”
3. “只输出 Vue 页面交互方案文档，包含状态流转和失败反馈”
4. “把 P0 任务改写成每日计划，按前后端配对”
