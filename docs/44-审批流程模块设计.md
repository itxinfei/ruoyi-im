## 二、数据库设计

### 2.1 审批模板表 (im_approval_template)

```sql
CREATE TABLE im_approval_template (
  id            BIGINT(20)    PRIMARY KEY AUTO_INCREMENT COMMENT '模板ID',

  icon          VARCHAR(200)  NULL COMMENT '图标',
  is_system     TINYINT(1)    DEFAULT 0 COMMENT '是否系统模板',
  status        VARCHAR(10)   DEFAULT 'ACTIVE' COMMENT '状态: ACTIVE/DISABLED',
  sort_order    INT(11)       DEFAULT 0 COMMENT '排序',
  create_by     VARCHAR(64)   NULL COMMENT '创建者',
  create_time   DATETIME      DEFAULT CURRENT_TIMESTAMP,
  update_by     VARCHAR(64)   NULL COMMENT '更新者',
  update_time   DATETIME      DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  UNIQUE INDEX uk_code (code),
  INDEX idx_category (category),
  INDEX idx_status (status)
```

## 三、后端实现 (JDK 1.8 规范)

### 3.1 审批流转核心逻辑

```java
/**
 * 审批工作流后端核心服务
 * 负责状态机流转与动态 JSON 表单路由跳转机制
 */
@Service
@RequiredArgsConstructor
public class ImApprovalServiceImpl implements ImApprovalService {

    private final ImApprovalMapper approvalMapper;

    /**
     * 将审批流转至下一个流向节点
     * 遵循纯正 JDK 1.8 规范，严防非法语法穿透
     *
     * @param approvalId 审批工单的唯一 ID (主键)
     */
    @Transactional(rollbackFor = Exception.class)
    public void flowToNextNode(Long approvalId) {
        ImApproval approval = approvalMapper.selectById(approvalId);
        if (Objects.isNull(approval)) {
            throw new BusinessException(500, "审批单不存在");
        }
        
        // 核心逻辑实现...
        approval.setStatus("APPROVED");
        approval.setUpdateTime(new Date());
        approvalMapper.updateById(approval);
    }
}
```

## 四、前端实现 (纯 JS 规范)

### 4.1 审批列表逻辑

```javascript
/**
 * 审批列表逻辑封装 (纯 JS)
 */
import { ref, onMounted } from 'vue';
import { approvalApi } from '@/api/approval';

export function useApprovalList() {
  const activeTab = ref('pending');
  const approvals = ref([]);

  const loadApprovals = async () => {
    // 接口返回 AjaxResult 结构
    const result = await approvalApi.getMyApprovals(activeTab.value);
    approvals.value = result.list || [];
  };

  const switchTab = (key) => {
    activeTab.value = key;
    loadApprovals();
  };

  return {
    activeTab,
    approvals,
    switchTab,
    loadApprovals
  };
}
```

## 五、UI 表现与对齐规范 (DingTalk Style)

### 5.1 审批列表栏 (Approval List)
- **顶栏 Tabs**：包含“待处理”、“已发起”、“已处理”。激活态文字及底边粗线使用 `#0089FF`。
- **卡片项 (Card)**：高度自动，内边距 `16px`，背景 `#FFFFFF`，边框 `1px solid rgba(23, 26, 29, 0.08)`。Hover 产生轻微投影。
- **状态标签 (Status Tags)**：
  - **待审批 (Pending)**：背景浅橙色，文字 `#FF9000`。
  - **已同意 (Approved)**：背景浅绿色，文字 `#52C41A`。
  - **已拒绝 (Rejected)**：背景浅红色，文字 `#FF4D4F`。

### 5.2 审批详情抽屉 (Detail Drawer)
- **触发与展现**：点击审批卡片，从右侧滑出宽 `400px` 的 `el-drawer`。
- **流转时间线 (Timeline)**：
  - 左侧为主轴，右侧为节点信息。
  - 当前进行中节点为蓝色发光圆点，已完成为蓝色实心钩，未开始为灰色空心圆。
