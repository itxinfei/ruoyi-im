# 14 页面路由与菜单映射

> 用途：把左侧导航、路由、组件、入口约定写清，避免模块入口混乱。  
> 说明：本文件只记录当前已知的路径与组件映射，未知项写 `unknown`，后续更新时补充。

## 1. 使用规则

1. 新增导航项/页面时，先检查本表是否需要扩充。
2. 任何入口调整，都要先在本节里记录修改前后对比。
3. 任务执行前，先从本表确认路由、页面、入口、组件的绑定关系。

## 3. 路由要点

1. `/workspace/messages` 为缺省路由，页面加载后应该默认进入第一个未读会话。
2. 每个 `/workspace/*` 路由必须在 `<router-view>` 中加载对应主页面组件，避免出现空白。
3. 若一个能力在多个入口处出现（比如群详情），需在本表中注明所有入口并标记主入口。

## 4. 组件归属

1. `SearchPanel.vue` 与 `GlobalSearch.vue` 均可唤起搜索，但 `SearchPanel` 是完整页，`GlobalSearch` 为聊天头部弹层。
2. 群详情目前存在 `GroupProfileDialog.vue` 和 `GroupDetailDrawer/index.vue`，下一步任务应统一主入口并在本表标出。
3. 通话能力由 `ChatHeader.vue` + `CallDialog.vue` + `useWebRTC.js` 共同承担，入口映射在 `14` 表唯一记录。

## 5. 更新流程

1. 每次添加新导航项或页面，立即更新本文件。
2. 若某个组件重命名或路径改变，请同步修改 “主要组件” 一栏。
3. 若新增路由参数或动态子路由，在 `备注` 中补充说明。
