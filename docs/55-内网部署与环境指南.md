# 55 内网部署与环境指南

> **定位**：为企业内部部署的 RuoYi‑IM 系统提供完整的 Nginx 代理、HTTPS/WSS 配置、反向代理、健康检查以及日志管理指南，确保系统在内网气密环境下安全、可靠、可运维。

---

## 1. 环境前置要求

- **操作系统**：Linux (CentOS 7/8、Ubuntu 20.04 推荐)。
- **JDK**：1.8（已在后端 `pom.xml` 中声明）。
- **Node.js**：16+（用于前端构建）。
- **Docker & Docker‑Compose**（可选，用于快速搭建完整环境）。
- **防火墙**：确保 80、443、8080、8443 端口对内部机器开放，外部仅允许 443（HTTPS）访问。

---

## 2. Nginx 核心配置

### 2.1 静态资源托管（前端）
```nginx
server {
    listen 80;
    server_name im.example.com;

    # 重定向到 HTTPS
    return 301 https://$host$request_uri;
}

server {
    listen 443 ssl http2;
    server_name im.example.com;

    ssl_certificate     /etc/nginx/ssl/im.crt;
    ssl_certificate_key /etc/nginx/ssl/im.key;
    ssl_protocols       TLSv1.2 TLSv1.3;
    ssl_ciphers         HIGH:!aNULL:!MD5;

    # 前端构建产物所在目录（Docker 镜像或手动拷贝）
    root /opt/im-frontend/dist;
    index index.html index.htm;

    # 支持 Vue Router History 模式
    location / {
        try_files $uri $uri/ /index.html;
    }

    # 代理 API 请求到后端服务
    location /api/ {
        proxy_pass http://im-backend:8080/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_http_version 1.1;
        proxy_set_header Connection "";
        proxy_buffering off;
    }

    # WebSocket 代理（必须开启协议升级）
    location /ws/ {
        proxy_pass http://im-backend:8080/;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "Upgrade";
        proxy_read_timeout 3600s;   # 防止长连接被防火墙踢掉
    }

    # 健康检查端点（后端提供 /actuator/health）
    location /health {
        proxy_pass http://im-backend:8080/actuator/health;
        proxy_set_header Host $host;
    }

    # 日志配置（统一 JSON 格式，便于 ELK）
    access_log  /var/log/nginx/im-access.log  json;
    error_log   /var/log/nginx/im-error.log  warn;
}
```

### 2.2 HTTPS 与 WSS 配置要点
- **证书**：使用企业内部 CA 签发的证书或 Let’s Encrypt（仅内部可访问时使用自签）。
- **强制 HSTS**：`add_header Strict-Transport-Security "max-age=31536000; includeSubDomains" always;`
- **WSS**：只要 `ws://` 代理改为 `wss://`（即使用 `https` 端口），浏览器会自动升级。

## 3. 反向代理与服务发现（Docker‑Compose 示例）
```yaml
version: "3.8"
services:
  nginx:
    image: nginx:1.21-alpine
    ports:
      - "80:80"
      - "443:443"
    volumes:
      - ./nginx.conf:/etc/nginx/nginx.conf:ro
      - ./ssl:/etc/nginx/ssl:ro
      - ./frontend/dist:/opt/im-frontend/dist:ro
    depends_on:
      - backend

  backend:
    image: im-backend:latest
    environment:
      - SPRING_PROFILES_ACTIVE=prod
    ports:
      - "8080:8080"
    volumes:
      - ./logs:/app/logs
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:8080/actuator/health"]
      interval: 30s
      timeout: 10s
      retries: 3
```

---

## 4. 健康检查与自动重启
- **Nginx**：使用 `ngx_http_upstream_module` 的 `max_fails` 与 `fail_timeout` 参数，自动剔除不可用后端。
- **后端**：Spring Boot 提供 `/actuator/health`，配合 Docker‑Compose `healthcheck`，K8s 可使用 `livenessProbe` 与 `readinessProbe`。

---

## 5. 日志与监控
- **后端日志**：统一使用 Logback 输出 JSON，路径 `logs/`，通过 Filebeat 收集。
- **Nginx 日志**：已在 `access_log` 中使用 `json` 格式。
- **监控**：Prometheus 抓取 `/actuator/prometheus`，Grafana 可视化。

---

## 6. 常见故障排查
| 场景 | 检查点 | 解决方案 |
|------|--------|----------|
| HTTPS 证书错误 | `openssl s_client -connect im.example.com:443` | 确认证书链完整、CN 与域名匹配。
| WebSocket 断连 | Nginx `proxy_read_timeout`、后端 `server.keepAliveTimeout` | 增大 `proxy_read_timeout`，确保后端保持心跳。
| 静态资源 404 | `root` 路径是否指向 `dist` 目录 | 检查 Docker 挂载或本地路径。
| 健康检查失败 | Docker `healthcheck` 输出 | 查看后端日志，确认 `actuator/health` 可访问。

---

## 7. 部署步骤概览
1. **准备证书**：放置于 `./ssl/`，确保文件权限 600。
2. **构建前端**：`npm run build`，产物拷贝到 `frontend/dist/`。
3. **构建后端 Docker 镜像**：`docker build -t im-backend:latest .`
4. **启动 Compose**：`docker-compose up -d`
5. **验证**：访问 `https://im.example.com`，检查页面、API、WebSocket 是否正常。
6. **监控接入**：配置 Prometheus 抓取 `http://im-backend:8080/actuator/prometheus`，在 Grafana 中添加仪表盘。
