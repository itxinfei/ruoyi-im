# 38 审批模块UI设计规范

> **用途**：定义审批模块（流程中心）的页面布局、动态表单及流程节点设计规范。  
> **设计基准**：钉钉 Windows 8.2.0 桌面端，遵循 `docs/27-PC版钉钉UI设计蓝图.md`。

---

## 1. 页面定位与模板

*   **模块 ID**：`approval`
*   **页面模板**：列表-详情协同页 (`L1 + L2 + L3`) + 表单页
*   **主要任务**：
    1.  处理“待我审批”的任务。
    2.  发起各类办公审批（请假、报销、用车等）。
    3.  查看历史审批进度和抄送信息。

---

## 2. 空间布局 (Layout)

### 2.1 列表页骨架
*   **L2 (分类侧栏)**：宽度 `240px`，含“待我审批”、“我发起的”、“抄送我的”及“已处理”。
*   **L3 (列表主区)**：承载审批单据列表，含搜索、类型筛选、状态筛选。
*   **Drawer (详情)**：点击列表项，从右侧滑出审批详情抽屉。

### 2.2 审批详情抽屉规范
*   **头部**：单据标题、当前状态标签（PENDING/APPROVED/REJECTED）。
*   **内容区**：
    *   `Form Data`：单据详细内容。
    *   `Process Timeline`：审批链路图（垂直时间轴）。
*   **底部固定操作栏**：含“同意”、“拒绝”、“转交”、“加签”按钮。

---

## 3. 组件级设计规范

### 3.1 审批状态标签 (Status Tags)
*   **待审批**：`var(--dt-warning-color)` 背景淡化，橙色文字。
*   **已通过**：`var(--dt-success-color)` 背景淡化，绿色文字。
*   **已拒绝**：`var(--dt-error-color)` 背景淡化，红色文字。
*   **已撤销**：`var(--dt-text-tertiary)` 背景淡化，灰色文字。

### 3.2 流程时间轴 (Process Timeline)
*   **节点图标**：
    *   已完成：实心绿圆 + 白色对号图标。
    *   进行中：空心蓝圆 + 旋转 Loading。
    *   未开始：空心灰圆。
*   **交互**：点击节点可查看审批人意见和处理时间。

### 3.3 动态表单项 (Form Items)
*   **行高**：`40px`。
*   **对齐**：Label 固定宽度 `80px`，居右对齐；Value 居左。
*   **附件展示**：多附件使用缩略图列表展示，支持点击预览。

---

## 4. 状态清单 (States)

| 状态 | 表现 |
|-----|------|
| **Pending Action** | 底部操作栏高亮显示，且带呼吸灯动效提示用户处理。 |
| **Success Feedback** | 审批通过后弹出绿色 Toast，详情状态立即更新为“已通过”。 |
| **Form Validation** | 发起申请时，必填项缺失显示红色边框，并自动定位到第一个错误项。 |
| **Empty** | 列表为空时显示“咖啡杯”插画，提示“暂无待处理审批”。 |

---

## 5. 交互路径 (Interactions)

1.  **发起申请**：工作台点击“请假” -> 弹出全屏表单页 -> 填写 -> 点击“提交” -> 成功后跳转至“我发起的”列表。
2.  **快捷审批**：列表行 Hover 态直接显示“同意/拒绝”小按钮，无需打开详情即可处理（针对批量场景）。
3.  **多级联动**：选择“审批类型”后，下方表单字段动态刷新。
