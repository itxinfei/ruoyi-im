# 17 接口与入口清单

> 用途：为大模型开发提供"从哪里进入、改哪里、调什么接口"的快速映射。
> 说明：本文件优先记录当前已确认映射；未确认项明确写 `unknown`，不补事实。

## 1. 使用规则

1. 任务开始前，先从本文件确认入口、页面、接口来源。
2. 如果本文件中某项为 `unknown`，模型必须保守处理，不得自行发明接口。
3. 本文件只记录高价值映射，不替代详细接口文档。
4. 代码变更后，如果入口或接口关系发生变化，应同步回写本文件。

## 7. 入口统一性风险

以下问题在后续开发前应优先识别：

1. 是否存在同一能力多个入口，但行为不一致。
2. 是否存在页面已展示，但没有稳定后端来源。
3. 是否存在前端 API 存在，但当前产品主路径没有入口。

当前已识别风险：

1. 群详情存在两套入口承载。
2. 收藏面板组件存在，但当前消息主页面是否标准接入仍需确认。
3. URL 元数据前端调用存在，后端实现位置仍需明确。
4. 日历、AI 助手等模块的后端来源尚未在本轮文档中确认。

## 8. 后续维护建议

1. 每确认一个模块的 controller/service 来源，就把 `部分确认` 或 `未确认` 更新掉。
2. 若某一级模块长期不准备做，应回写到需求文档，避免“钉钉有的功能都要有”与实际规划冲突。
3. 若某能力入口发生归并，例如统一群详情入口，应同步回写本文件。
