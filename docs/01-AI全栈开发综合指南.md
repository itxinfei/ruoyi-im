## 一、项目定位

**RuoYi-IM**：企业内网私有化部署的 Web 版钉钉式协同平台

- IM 即时通讯是核心中的核心
- 其他协同能力围绕 IM 展开
- 产品体验参考钉钉 Windows 客户端

**目标用户**：10-500 人规模企业员工

## 六、开发约束

### 禁止行为 (Zero Hallucination Rules)
1. 禁止把未实现功能伪装成已完成。
2. 禁止只改文案不打通事件逻辑的表层敷衍。
3. 禁止随意脱离 Scope 的无关大面积重构。
4. **禁止用猜测补全未知事实**（如遇未知结构、接口名缺失、依赖缺失，强制触发 **HALT 停机指令** 并请求人工确认，对幻觉零容忍）。
5. 禁止前端使用外部 CDN，全量私有化。

### 质量护城河与 UI 像素级操纵准则 (Quality & UI Defenses)
为彻底杜绝大模型常见的“屎山代码”与“设计灾难”，在生成代码时强制挂载以下滤镜：
1. **彻底剥夺自由调色权**：在编写 Vue 组件时，严禁自行发挥使用 `#FF0000` 或 `red` 等硬编码颜色。必须强制调用现有的全局 CSS Tokens 变量（如 `var(--el-color-primary)`、`var(--text-color-regular)`）。
2. **强制内存释放契约**：IM 属于长线保活状态，凡是涉及 WebSocket (`ws.on`)、全局事件总线 (`mitt.on`) 或自动轮询定时器，**必须强制**在 `onUnmounted` 中执行拆卸摧毁，严防内存泄漏。
3. **高频事件防抖**：对 WebSocket 广播或大量 DOM 刷新，必须套嵌 `debounce` 或 `throttle` 防抖层，杜绝 Vue VDOM 重绘雪崩。
4. **架构死守红线**：前端无论多复杂的实体定义均**严禁使用 TypeScript** (`interface`, `type`)；后端即便写最短的工具类也**坚决禁用 Java 9+ 语法** (`var`, `record`)，保证两端底座绝对纯净。

### 验证要求
- 每次开发必须运行 `npm run build` 验证
- 必须确保主流程可闭环
- 未知项登记到 `04-待确认问题.md`

### 完成定义
- 用户能从真实入口触达
- 主流程可闭环，不依赖占位提示
- 异常场景有明确反馈
- 前后端状态一致
- 构建验证通过
