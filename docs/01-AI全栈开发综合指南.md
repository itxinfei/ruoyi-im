## 一、项目定位

**RuoYi-IM**：企业内网私有化部署的 Web 版钉钉式协同平台

- IM 即时通讯是核心中的核心
- 其他协同能力围绕 IM 展开
- 产品体验参考钉钉 Windows 客户端

**目标用户**：10-500 人规模企业员工

---

## 二、技术栈

| 层级 | 技术 | 版本 |
|-----|------|------|
| 前端 | Vue 3 + Vite + Element Plus + Pinia (JavaScript) | 3.3+ |
| 后端 | Spring Boot + MyBatis Plus + Spring Security + JWT | 2.7.18 |
| 数据库 | MySQL + Redis | 5.7.44 / 6.x |
| 实时通讯 | WebSocket | - |

## 三、开发命令

```bash
# 前端
cd ruoyi-im-web && npm run dev      # 启动开发服务器
cd ruoyi-im-web && npm run build    # 构建验证（必须通过）

# 后端
cd ruoyi-im-api && mvn spring-boot:run
```

## 六、开发约束

### 禁止行为 (Zero Hallucination Rules)
1. 禁止把未实现功能伪装成已完成。
2. 禁止只改文案不打通事件逻辑的表层敷衍。
3. 禁止随意脱离 Scope 的无关大面积重构。
4. **禁止用猜测补全未知事实**（如遇未知结构、接口名缺失、依赖缺失，强制触发 **HALT 停机指令** 并请求人工确认，对幻觉零容忍）。
5. 禁止前端使用外部 CDN，全量私有化。

### 验证要求
- 每次开发必须运行 `npm run build` 验证
- 必须确保主流程可闭环
- 未知项登记到 `04-待确认问题.md`

### 完成定义
- 用户能从真实入口触达
- 主流程可闭环，不依赖占位提示
- 异常场景有明确反馈
- 前后端状态一致
- 构建验证通过
