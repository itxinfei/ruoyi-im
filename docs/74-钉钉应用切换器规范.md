# 74 钉钉应用切换器规范 (App Switcher Spec)

> **用途**：定义左下角"应用切换器"（红色 + 按钮）的视觉与交互规范。这是钉钉桌面端独有的快捷入口设计。
> **权威依据**：docs/00-钉钉8.2像素级权威基线.md / 视觉资产 #29-app-switcher.png
> **优先级**：⭐⭐ 中（产品形态层面，不影响日常聊天）

---

## 1. 设计目标

钉钉 PC 端的左下角"+"按钮是企业用户最高频的快捷入口：
- 快速发起新会话（聊天/会议/电话）
- 添加新好友/群组
- 创建文档/审批/待办
- 切换到其他应用模块

它不是普通菜单，而是**主操作中心**，是钉钉与微信/Slack 形态最大的差异之一。

## 6. 交互规则

| 触发 | 行为 |
|-----|-----|
| 点击红色 + 按钮 | 打开面板（200ms 淡入 + 上移 8px） |
| 点击操作项 | 关闭面板 + 触发对应 action |
| 点击面板外 | 关闭面板（200ms 淡出） |
| Esc 键 | 关闭面板 |
| 面板打开时再次点击按钮 | 关闭面板 |
| Tab 键 | 在操作项之间循环聚焦 |

### 6.1 出现/消失动效

```scss
.switcher-panel-enter-active,
.switcher-panel-leave-active {
  transition: opacity 200ms ease-out, transform 200ms ease-out;
}

.switcher-panel-enter-from,
.switcher-panel-leave-to {
  opacity: 0;
  transform: translateY(8px);
}
```

## 7. 验收清单 (AC)

### 7.1 视觉
- [ ] 红色 + 按钮位于 L1 底部，紧邻头像上方
- [ ] 按钮 40×40 圆形，背景 #F53F3F (var(--dt-error-color))
- [ ] 面板宽 280px，弹出位置紧靠 L1 右侧
- [ ] 操作项使用 2 列网格，每项含 icon 容器 + 标签
- [ ] icon 容器配色按功能语义（蓝/绿/橙/紫/青）

### 7.2 交互
- [ ] 点击 + 按钮立刻弹出面板（< 100ms）
- [ ] 弹出动效 200ms 淡入 + 上移
- [ ] 点击操作项关闭面板 + 触发对应路由/弹窗
- [ ] 点击面板外或 Esc 关闭
- [ ] 支持键盘 Tab 循环聚焦

### 7.3 内容
- [ ] 至少包含 3 个分组：快速发起、添加联系人、办公协同
- [ ] 每个分组至少 4 个操作项
- [ ] 所有操作项必须有真实 action（不允许占位）

## 8. 反例（禁止）

| 反例 | 错误 | 正确 |
|-----|------|------|
| 按钮用蓝色 | background: var(--dt-brand-color) | background: var(--dt-error-color) |
| 按钮方形 | border-radius: 8px | border-radius: 50% |
| 面板从顶部下拉 | top: 0 | bottom: 64px (从底部上弹) |
| 单列布局 | grid-template-columns: 1fr | 1fr 1fr (双列) |
| 所有图标同色 | 全部品牌蓝 | 按功能分色 |
