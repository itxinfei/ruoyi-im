## 一、设计概述

### 1.1 职责边界
- **认证 (Authentication)**：基于 JWT 的无状态认证逻辑，支持 Token 刷新与多端互踢。
- **授权 (Authorization)**：基于 RBAC 模型的权限校验，包含角色管理、菜单权限、数据权限三层。
- **内容安全 (Content Security)**：敏感词过滤（AC 自动机）与消息掩码，防止 XSS/SQL 注入。
- **审计日志 (Audit Log)**：关键操作日志记录，支持按模块/用户/时间检索。

### 1.2 核心原则
1. 最小权限原则：用户仅拥有完成其工作所需的最小权限集。
2. 纵深防御：认证 → 权限 → 内容安全 → 审计，四层联动。
3. 敏感信息零泄露：日志中禁止打印密码、Token、手机号等明文。

```java
/**
 * 权限校验注解 - 支持多权限逻辑或
 * 用法：@RequirePermission({"im:message:send", "im:message:edit"})
 */
@Target({ElementType.TYPE, ElementType.METHOD})
@Retention(RetentionPolicy.RUNTIME)
public @interface RequirePermission {
    String[] value();
    /**
     * 逻辑类型：AND = 必须全部满足，OR = 满足任一即可
     */
    Logical logical() default Logical.OR;
}
```

### 2.3 JWT 认证服务

```java
@Service
@RequiredArgsConstructor
public class JwtTokenService {

    @Value("${jwt.secret}")
    private String secret;

    @Value("${jwt.expire-seconds}")
    private Long expireSeconds;

    @Value("${jwt.refresh-expire-seconds}")
    private Long refreshExpireSeconds;

    /**
     * 生成访问令牌
     * 遵循 JDK 1.8，严禁 var
     */
    public String generateAccessToken(Long userId, String username, List<String> roles, List<String> permissions) {
        Map<String, Object> claims = new HashMap<>(8);
        claims.put("userId", userId);
        claims.put("username", username);
        claims.put("roles", roles);
        claims.put("permissions", permissions);
        claims.put("type", "access");
        claims.put("iat", System.currentTimeMillis() / 1000);
        return Jwts.builder()
                .setClaims(claims)
                .setExpiration(new Date(System.currentTimeMillis() + expireSeconds * 1000))
                .signWith(SignatureAlgorithm.HS256, secret)
                .compact();
    }

    /**
     * 生成刷新令牌
     */
    public String generateRefreshToken(Long userId) {
        Map<String, Object> claims = new HashMap<>(4);
        claims.put("userId", userId);
        claims.put("type", "refresh");
        return Jwts.builder()
                .setClaims(claims)
                .setExpiration(new Date(System.currentTimeMillis() + refreshExpireSeconds * 1000))
                .signWith(SignatureAlgorithm.HS256, secret)
                .compact();
    }

    /**
     * 获取用户权限列表
     * 遵循 JDK 1.8，严禁 var
     */
    @SuppressWarnings("unchecked")
    public List<String> getPermissions(String token) {
        Claims claims = parseClaims(token);
        Object permissionsObj = claims.get("permissions");
        if (Objects.nonNull(permissionsObj)) {
            return (List<String>) permissionsObj;
        }
        return new ArrayList<>();
    }

    /**
     * 获取用户角色列表
     */
    @SuppressWarnings("unchecked")
    public List<String> getRoles(String token) {
        Claims claims = parseClaims(token);
        Object rolesObj = claims.get("roles");
        if (Objects.nonNull(rolesObj)) {
            return (List<String>) rolesObj;
        }
        return new ArrayList<>();
    }

    /**
     * 解析 JWT 令牌
     * 异常时返回 null，由调用方处理
     */
    private Claims parseClaims(String token) {
        try {
            return Jwts.parser()
                    .setSigningKey(secret)
                    .parseClaimsJws(token)
                    .getBody();
        } catch (ExpiredJwtException e) {
            log.warn("JWT令牌已过期: {}", e.getMessage());
            return null;
        } catch (JwtException e) {
            log.warn("JWT令牌解析失败: {}", e.getMessage());
            return null;
        }
    }
}
```

### 2.4 权限拦截器

```java
@Component
@RequiredArgsConstructor
public class PermissionInterceptor implements HandlerInterceptor {

    private final JwtTokenService jwtTokenService;

    @Override
    public boolean preHandle(HttpServletRequest request, HttpServletResponse response, Object handler) {
        if (!(handler instanceof HandlerMethod)) {
            return true;
        }

        HandlerMethod handlerMethod = (HandlerMethod) handler;
        
        // 检查 @RequireRole 注解
        RequireRole requireRole = handlerMethod.getMethodAnnotation(RequireRole.class);
        if (requireRole != null) {
            String token = extractToken(request);
            List<String> userRoles = jwtTokenService.getRoles(token);
            boolean hasRole = Arrays.stream(requireRole.value())
                    .anyMatch(role -> userRoles.contains(role));
            if (!hasRole) {
                response.setStatus(HttpStatus.FORBIDDEN.value());
                return false;
            }
        }

        // 检查 @RequirePermission 注解
        RequirePermission requirePermission = handlerMethod.getMethodAnnotation(RequirePermission.class);
        if (requirePermission != null) {
            String token = extractToken(request);
            List<String> userPermissions = jwtTokenService.getPermissions(token);
            boolean hasPermission;
            if (requirePermission.logical() == Logical.AND) {
                hasPermission = Arrays.stream(requirePermission.value())
                        .allMatch(userPermissions::contains);
            } else {
                hasPermission = Arrays.stream(requirePermission.value())
                        .anyMatch(userPermissions::contains);
            }
            if (!hasPermission) {
                response.setStatus(HttpStatus.FORBIDDEN.value());
                return false;
            }
        }

        return true;
    }

    private String extractToken(HttpServletRequest request) {
        String bearerToken = request.getHeader("Authorization");
        if (StringUtils.isNotBlank(bearerToken) && bearerToken.startsWith("Bearer ")) {
            return bearerToken.substring(7);
        }
        return null;
    }
}
```

### 2.6 敏感词过滤服务

```java
@Service
@RequiredArgsConstructor
public class ContentSecurityServiceImpl implements ContentSecurityService {

    private final ImSensitiveWordMapper sensitiveWordMapper;
    private AhoCorasickMatcher matcher;

    /**
     * 初始化 AC 自动机（启动时加载）
     */
    @PostConstruct
    public void init() {
        List<String> words = sensitiveWordMapper.selectAllWords();
        this.matcher = new AhoCorasickMatcher(words);
        log.info("敏感词库加载完成，共 {} 条", words.size());
    }

    /**
     * 检查文本是否包含敏感词
     * @return true = 安全，false = 包含敏感词
     */
    public boolean isSafe(String text) {
        if (StringUtils.isBlank(text)) {
            return true;
        }
        return !matcher.containsMatch(text);
    }

    /**
     * 过滤敏感词（替换为 ***）
     */
    public String filter(String text) {
        if (StringUtils.isBlank(text)) {
            return text;
        }
        return matcher.replace(text, "***");
    }
}
```

```java
@Service
@RequiredArgsConstructor
public class AuditLogServiceImpl implements AuditLogService {

    private final ImAuditLogMapper auditLogMapper;

    /**
     * 记录审计日志（异步）

        log.setModule(module);
        log.setAction(action);
        log.setTargetId(targetId);

    }
```

## 三、前端实现 (纯 JS 规范)

### 3.1 权限校验逻辑

```javascript
/**
 * 权限校验逻辑 (纯 JS)
 * 强制要求：严禁使用 TypeScript
 */
import { useUserStore } from '@/stores/user';

/**
 * 检查是否拥有指定权限
 * @param {string|Array<string>} permission - 权限标识或权限数组
 * @returns {boolean}
 */
export function hasPermission(permission) {
  const userStore = useUserStore();
  const userPermissions = userStore.permissions || [];
  if (!permission) return true;

  if (Array.isArray(permission)) {
    return permission.some(function(p) { return userPermissions.includes(p); });
  }
  return userPermissions.includes(permission);
}

/**
 * 检查是否拥有指定角色
 * @param {string|Array<string>} role - 角色标识或角色数组
 * @returns {boolean}
 */
export function hasRole(role) {
  const userStore = useUserStore();
  const userRoles = userStore.roles || [];
  if (!role) return true;

  if (Array.isArray(role)) {
    return role.some(function(r) { return userRoles.includes(r); });
  }
  return userRoles.includes(role);
}
```

### 3.2 权限指令封装

```javascript
/**
 * 权限指令 v-permission
 * 用法：v-permission="'im:message:send'"
 * 用法：v-permission="['im:message:send', 'im:message:edit']"
 */
export const permissionDirective = {
  mounted(el, binding) {
    var permissions = binding.value;
    if (!hasPermission(permissions)) {
      el.parentNode && el.parentNode.removeChild(el);
    }
  }
};

/**
 * 角色指令 v-role
 * 用法：v-role="'admin'"
 */
export const roleDirective = {
  mounted(el, binding) {
    var roles = binding.value;
    if (!hasRole(roles)) {
      el.parentNode && el.parentNode.removeChild(el);
    }
  }
};
```

### 3.3 Token 管理策略

```javascript
/**
 * Token 管理工具 (纯 JS)
 * 安全要求：Token 必须设置过期时间，禁止永久存储
 */
var TOKEN_KEY = 'im_access_token';
var REFRESH_TOKEN_KEY = 'im_refresh_token';
var TOKEN_EXPIRE_KEY = 'im_token_expire';

export var tokenManager = {
  /**
   * 保存 Token（默认 2 小时过期）
   */
  save: function(accessToken, refreshToken, expireSeconds) {
    var expireSeconds = expireSeconds || 7200;
    var expireTime = Date.now() + expireSeconds * 1000;
    localStorage.setItem(TOKEN_KEY, accessToken);
    localStorage.setItem(REFRESH_TOKEN_KEY, refreshToken);
    localStorage.setItem(TOKEN_EXPIRE_KEY, String(expireTime));
  },

  /**
   * 获取 Token（自动检查过期）
   * @returns {string|null}
   */
  getAccessToken: function() {
    var expireTime = localStorage.getItem(TOKEN_EXPIRE_KEY);
    if (!expireTime || Date.now() > Number(expireTime)) {
      this.clear();
      return null;
    }
    return localStorage.getItem(TOKEN_KEY);
  },

  /**
   * 获取刷新 Token
   */
  getRefreshToken: function() {
    return localStorage.getItem(REFRESH_TOKEN_KEY);
  },

  /**
   * 清除 Token
   */
  clear: function() {
    localStorage.removeItem(TOKEN_KEY);
    localStorage.removeItem(REFRESH_TOKEN_KEY);
    localStorage.removeItem(TOKEN_EXPIRE_KEY);
  }
};
```

## 五、安全红线 (Security Checklist)

1. **[S1]** 所有 API 入口必须经过 JWT 认证（白名单除外）。
2. **[S2]** 密码存储必须使用 BCrypt 加密，严禁明文存储。
3. **[S3]** 日志中禁止打印 Token、密码、手机号等敏感信息。
4. **[S4]** 文件上传必须校验类型白名单和大小限制。
5. **[S5]** SQL 必须使用参数化查询（MyBatis `#{}`），禁止拼接。
6. **[S6]** 前端禁止使用 `v-html` 未清洗内容，防止 XSS。
7. **[S7]** 文件路径禁止用户可控拼接，防止目录遍历攻击。
8. **[S8]** 接口请求频率限制：登录接口 5次/分钟，普通接口 60次/分钟。

## 六、约束与限制

### 6.1 密码策略
- 最小长度：8 位
- 复杂度：必须包含字母和数字
- 过期策略：90 天强制修改（可配置）
- 历史密码：禁止使用最近 5 次密码

### 6.2 登录策略
- 最大尝试次数：5 次后锁定 30 分钟
- 多端互踢：同一账号最多 3 个设备同时在线
- Session 超时：2 小时无操作自动登出

### 6.3 Token 策略
- Access Token 有效期：2 小时
- Refresh Token 有效期：7 天
- Token 黑名单：退出时将 Token 加入 Redis 黑名单（TTL = 剩余有效期）
