# 钉钉对齐 V1 范围锁定表

> 目标：冻结 RuoYi-IM Web 版第一阶段对齐范围，避免需求漂移。  
> 生效日期：2026-03-03

## 1. 技术栈与环境（硬约束）

1. JDK：`1.8`
2. 前端：`Vue 3`
3. 数据库：`MySQL 5.7`
4. 开发环境：`Windows 11`
5. 生产环境：`CentOS 7`

## 2. V1 功能范围（只做这些）

1. 消息域：聊天主链路（会话、消息、搜索、上传、语音/视频入口、消息操作）
2. 通讯录域：组织架构、好友、群组、新联系人处理
3. 工作台域：审批、考勤、待办、日程、工作报告（高频路径）

## 3. 一级/二级菜单冻结

1. 消息：最近会话 / 未读 / 置顶 / 免打扰 / 群聊 / 文件与链接
2. 通讯录：组织架构 / 我的好友 / 群组 / 新的联系人 / 常用联系人
3. 工作台：常用应用 / 审批 / 考勤 / 待办 / 日程 / 工作报告

## 4. 三级弹窗冻结（必须实现）

1. 新建会话弹窗（单聊/群聊）
2. 会话详情弹窗（置顶、免打扰、清空未读、删除）
3. 文件预览弹窗（预览、下载、转发）
4. 发起通话弹窗（语音/视频邀请与状态）
5. 成员详情弹窗（发消息、发DING）
6. 好友申请处理弹窗（同意、拒绝、备注）
7. 群设置弹窗（公告、成员管理、禁言）
8. 发起审批弹窗
9. 补卡申请弹窗
10. 新建待办弹窗
11. 新建日程弹窗
12. 新建报告弹窗

## 5. 验收口径（统一）

1. 主链路可达：从一级导航到二级菜单再到三级弹窗全可达。
2. 弹窗可用：可关闭、可提交、可校验、可反馈。
3. 不得出现“开发中”死路入口。
4. 前端 `npm run build` 必须通过。
5. 后端 `mvn -DskipTests compile` 必须通过。
