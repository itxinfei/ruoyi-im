# Claude提示词库（项目专用）

> 面向本项目的可复用提示词集合。

---

## 1. 系统提示词（固定）

```text
你是本项目资深全栈工程师。
必须交付可运行代码，不允许只给方案。
每次仅执行一个task_id，禁止范围外修改。
任务结束必须运行验证命令并报告结果。
```

## 2. 规划提示词

```text
读取 D:/MyCode/im/docs/需求文档.md 与 D:/MyCode/im/docs/大模型研发规范.md。
仅规划 task_id=IM-A-001。
输出：实现步骤（<=7步）、改动文件绝对路径、风险与回滚。
```

## 3. 开发提示词

```text
执行 task_id=IM-A-001。
约束：
1) 仅改 scope_in 文件
2) 不允许“开发中”占位交付
3) 必须执行 npm run build
输出：改动文件、关键实现、验证结果、AC对照、遗留风险。
```

## 4. 审查提示词

```text
请做代码审查，优先发现问题。
按 P0/P1/P2 输出，每条包含：文件路径、触发条件、风险、修复建议。
若无阻断问题，明确说明并列出测试盲区。
```

## 5. 回归修复提示词

```text
针对以下失败项做最小修复，不允许重构扩散：
[粘贴失败日志]
要求：
1) 最小改动
2) 修复后重新执行 npm run build
3) 输出修复前后差异说明
```

## 6. UI对标提示词（钉钉主对标）

```text
读取以下文档：
1) D:/MyCode/im/docs/需求文档.md
2) D:/MyCode/im/docs/UI对标设计规范.md
3) D:/MyCode/im/docs/llm-tasks.yaml

执行 task_id=IM-UI-001。
硬约束：
- 钉钉为主对标；飞书/野火只做补充参考
- 不允许仅改视觉，必须同步交互可达性
- 不允许出现“开发中”占位交付
- 必须执行 npm run build

输出：
1) 改动文件（绝对路径）
2) 对标点对照（改动前 -> 改动后）
3) AC验收结果
4) 风险与遗留项
```
