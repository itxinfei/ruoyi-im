# Claude 提示词库

## 1. 规划提示词

```text
基于当前 docs 体系规划一个单任务。

先按《05-AI执行文档使用协议.md》判断任务类型与必读文档，再开始规划。

必读基准：
1. docs/05-AI执行文档使用协议.md
2. docs/01-AI全栈开发综合指南.md
3. docs/02-大模型研发规范.md
4. docs/30-技术架构与约束.md

业务真相源：
5. docs/10-产品需求总纲.md
6. docs/12-需求点清单.md
7. docs/13-需求明细.md 或 docs/18-平台模块与管理后台需求.md
8. docs/19-需求追溯矩阵.md

页面与设计（按任务类型选读）：
9. docs/14-页面路由与菜单映射.md
10. docs/15-产品形态与导航清单.md
11. docs/16-设计复刻差异.md
12. docs/20-UI交互规范总纲.md
13. docs/21-IM桌面端页面规范.md
14. docs/25-钉钉Windows复刻基线.md
15. docs/26-IM交互深度行为规范.md
16. docs/27-PC版钉钉UI设计蓝图.md
17. docs/35-前端全局样式与主题配置.md
18. docs/49-通话界面UI设计规范.md

技术契约（按需）：
19. docs/17-接口与入口清单.md
20. docs/31-实体与状态模型.md
21. docs/32-数据库设计.md
22. docs/33-API接口设计.md
23. docs/34-WebSocket协议.md

执行与验收：
24. docs/03-实现状态矩阵.md
25. docs/23-UI复刻任务计划.md
26. docs/24-页面缺陷与功能缺口清单.md
27. docs/52-测试与质量策略.md
28. docs/54-验收用例清单.md
29. 指定任务卡或 docs/llm-tasks.yaml 中的单个 task_id

要求：
1. 只规划一个 task_id
2. 先明确 task_type，以及本任务必读 docs
3. 输出实现步骤，最多 7 步
4. 输出改动文件列表
5. 输出需要同步回写的文档列表
6. 输出验收条目对照
7. 输出风险与回滚方案
8. 如果信息缺失，必须立即触发 HALT 停机阻断，严禁依靠常识猜测或捏造 Mock 数据框架
9. 默认产品目标是“私有化部署的 Web 版钉钉”，默认体验参考钉钉 Windows 客户端
10. 如果任务与 IM 主链路冲突，IM 优先
11. 在设计任何新技术方案前，强制先执行 Glob/Grep 搜索工作区，绝不允许凭空新建已存在的工具类或同名 Bean
12. 所有前端样式严格限制在 CSS Tokens 库内取值，严禁 AI 自由编造十六进制颜色
13. 禁止引用不存在的文档路径；若历史模板中存在失效路径，必须替换为当前 docs 中真实文档
```

## 2. 开发提示词

```text
执行一个单任务开发。

输入（必读基准）：
1. docs/05-AI执行文档使用协议.md
2. docs/02-大模型研发规范.md
3. docs/30-技术架构与约束.md
4. docs/13-需求明细.md 或 docs/18-平台模块与管理后台需求.md
5. docs/14-页面路由与菜单映射.md
6. docs/17-接口与入口清单.md
7. docs/31-实体与状态模型.md
8. docs/32-数据库设计.md
9. docs/33-API接口设计.md
10. docs/34-WebSocket协议.md（如任务涉及 WS）
11. docs/27-PC版钉钉UI设计蓝图.md（如任务涉及 UI）
12. docs/20-UI交互规范总纲.md（如任务涉及 UI）
13. docs/25-钉钉Windows复刻基线.md（如任务涉及 UI）
14. docs/21-IM桌面端页面规范.md / docs/26-IM交互深度行为规范.md / docs/35-前端全局样式与主题配置.md（按需）
15. 具体任务卡

硬约束重点执行：
1. 只改 scope_in
2. 绝对不可使用 TypeScript，严格依据《docs/02-大模型研发规范.md》写纯 JS `<script setup lang="js">`
3. 极速阻断原则：凡是未在上下文中找到的 API 方法、字段、变量定义或文档依据，立刻停机并向用户抛出疑问，严禁凭名称相似度自行捏造 `handleXxx` 等占位符
4. 后端必须完全遵照 JDK 1.8，不用 `var`、不吞异常，返回接口严格遵循 `DTO`/`VO` 分层结构
5. 必须运行 `npm run build` 并报告结果
6. 目标形态是“私有化 Web 版钉钉”，UI、布局、交互优先对齐钉钉 Windows 客户端，禁止自行造多余特效或动画
7. 建立组件或写配置前，必须使用搜索工具全局核实是否重复，执行防重合并原则
8. UI 开发必须完全依靠 CSS Tokens 与当前设计文档，禁止硬编码非标准颜色、圆角和间距
9. 强制内存防泄漏规约：确保前端组件在 `onUnmounted` 中有完整的定时器与全局监听 (`mitt.off`, `ws.off`) 摧毁闭环
10. 完成后必须根据《docs/05-AI执行文档使用协议.md》同步回写相关 docs

输出：
1. 改动文件
2. 关键实现
3. 验收对照
4. 验证结果
5. 文档回写结果
6. 风险与遗留项
```

## 3. 审查提示词

```text
请审查本次任务改动，重点找问题而不是总结。

输入：
1. 任务卡
2. 改动文件
3. docs/05-AI执行文档使用协议.md
4. docs/02-大模型研发规范.md（审核核心：是否出现 TypeScript、垃圾命名、Java9+结构、裸异常、打印明文）
5. docs/13-需求明细.md 或 docs/18-平台模块与管理后台需求.md
6. docs/30-技术架构与约束.md
7. docs/33-API接口设计.md / docs/34-WebSocket协议.md（按需）
8. docs/20-UI交互规范总纲.md / docs/25-钉钉Windows复刻基线.md / docs/27-PC版钉钉UI设计蓝图.md（如涉及 UI）

要求：
1. 按 P0/P1/P2 输出问题
2. 每个问题必须包含文件路径、触发条件、风险后果、修复建议
3. 若无阻断问题，明确写“未发现阻断问题”
4. 最后列出测试盲区
5. 额外检查是否偏离钉钉 Windows 客户端的布局、层级、交互节奏
6. 额外检查是否遗漏了应当同步更新的 docs
```

## 4. 文档回写提示词

```text
请根据本次任务改动，回写对应的 docs。

输入：
1. docs/05-AI执行文档使用协议.md
2. 当前任务卡
3. 已改动的代码文件
4. 已改动的文档文件

要求：
1. 先判断本次改动属于哪类变化：路由、UI、接口、状态机、测试、索引、提示词
2. 按《docs/05-AI执行文档使用协议.md》的回写矩阵列出必须同步更新的 docs
3. 明确哪些文档需要新增内容，哪些只需更新索引或变更记录
4. 如果存在“代码已变但文档未同步”的断层，直接指出并补齐
5. 禁止引用不存在的文档路径
```
