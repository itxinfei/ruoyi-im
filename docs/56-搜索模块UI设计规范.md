# 56 搜索模块UI设计规范

> **用途**：定义全局搜索与会话内搜索的 UI 布局、结果呈现及交互路径。  
> **设计基准**：钉钉 Windows 8.2.0 桌面端，遵循 `docs/27-PC版钉钉UI设计蓝图.md`。

---

## 1. 页面定位与入口

*   **模块 ID**：`search`
*   **页面模板**：三栏主链路页 / 全局悬浮弹窗
*   **主要任务**：
    1.  全局搜索联系人、群组、聊天记录、文件。
    2.  搜索结果的分类展示与快速预览。
    3.  点击搜索结果自动跳转并定位到原始上下文。

---

## 2. 空间布局 (Layout)

### 2.1 全局搜索弹窗 (Global Search Dialog)
*   **触发**：点击左侧导航搜索图标，或使用快捷键 `Ctrl + F`。
*   **尺寸**：宽度 `720px`，高度自适应（最高 560px）。
*   **头部**：全宽输入框（无边框风格），高度 `56px`，带 `Close` 按钮。
*   **内容区**：
    *   **左侧分类导航**：宽度 `160px`，含：全部、联系人、群聊、聊天记录、文件。
    *   **右侧结果列表**：`flex: 1`，按分类显示前 N 条结果。

### 2.2 搜索结果项规范
*   **联系人/群聊**：同会话列表样式，高度 `56px`。
*   **聊天记录**：显示发送者、日期，以及带有**关键词高亮**的消息片段（两行预览）。
*   **文件**：显示文件图标、文件名（高亮）、大小、来源会话。

---

## 3. 组件级设计规范

### 3.1 搜索输入框 (Search Input)
*   **样式**：`var(--dt-bg-body)` 背景，圆角 `var(--dt-radius-md)`。
*   **状态**：
    *   空状态：显示“搜索联系人、群聊、文件等”。
    *   激活态：边框高亮为品牌蓝，显示清除按钮 `(X)`。

### 3.2 关键词高亮 (Highlighting)
*   **Token**：使用自定义变量 `--dt-search-highlight: #FF7D00` (橙色) 或 `--dt-brand-color` 加背景淡化。
*   **CSS 类**：`.search-highlight { font-weight: 600; color: var(--dt-brand-color); background: rgba(39, 126, 251, 0.1); }`。

---

## 4. 状态清单 (States)

| 状态 | 表现 |
|-----|------|
| **Searching** | 输入框下方显示线性加载条（Linear Progress），结果区显示“正在搜索...”。 |
| **No Result** | 显示中心化空状态插画，提示“未找到相关结果”，并建议尝试其他关键词。 |
| **Hover** | 搜索结果行变色 `var(--dt-bg-hover)`，右侧显示“跳转到聊天”文字。 |

---

## 5. 交互路径 (Interactions)

1.  **跳转定位**：点击消息搜索结果 -> 自动切换到该会话 -> 滚动条自动定位到该消息位置 -> **该消息气泡背景闪烁 3 次**（黄色至透明）以引导视觉。
2.  **分类过滤**：点击左侧“文件” -> 右侧列表立即切换为纯文件列表，并在顶部增加“文件类型（PDF/PPT等）”和“时间范围”的二级筛选。
3.  **搜索历史**：输入框获取焦点且无内容时，展示最近 10 条搜索记录。

---

## 6. 验收点 (AC)

- [ ] AC1: 搜索结果支持键盘 `↑` `↓` 键选择，`Enter` 键确认。
- [ ] AC2: 会话内搜索结果支持“按日期”和“按成员”进行二次过滤。
- [ ] AC3: 消息定位逻辑准确，即便在万级聊天记录中也能实现瞬间跳转并高亮。
- [ ] AC4: UI 严格对齐钉钉 8.2.0 的紧凑布局标准。
