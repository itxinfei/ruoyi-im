# 22 UI 视觉与交互红线 (UI Quality Checklist)

> **定位**：定义 IM 核心页面的"对齐钉钉"验收硬指标。任何代码生成的 UI 必须通过此清单检查，否则视为不合格。

---

## 1. 核心骨架 (Skeleton) 红线

- **[S1] 三栏布局固定化**：左侧 L1 (`64px`)、会话 L2 (`250px`) 必须固定宽度。严禁使用百分比布局导致宽度在宽屏下无限拉伸。
- **[S2] 根滚动条禁令**：全局 `body` 和 `#app` 必须禁止滚动 (`overflow: hidden`)。仅允许在 L2 列表、L3 聊天区域进行局部滚动。
- **[S3] 分割线规范**：三栏之间的分割线颜色必须是 `rgba(23, 26, 29, 0.08)`，宽度 `1px`。

---

## 2. 消息气泡 (Message Bubble) 精准红线

- **[M1] 气泡圆角逻辑**：
  - **接收方 (Left)**: `4px 8px 8px 8px` (左上角尖)。
  - **发送方 (Right)**: `8px 4px 8px 8px` (右上角尖)。
- **[M2] 头像与基线**：头像顶部必须与气泡容器顶部 **绝对对齐**（严禁垂直居中对齐）。
- **[M3] 连续消息收纳**：同一分钟内、同一发送者的消息，第二条开始 **必须隐藏头像和昵称**，且消息间距压缩至 `4px`。
- **[M4] 交互浮层**：消息操作栏 (复制/转发等) 必须在 `hover` 消息时浮现，严禁常驻。

---

## 3. 颜色与字体 (Colors & Fonts) 红线

- **[C1] 品牌蓝锁定**：主色调、激活态、发送气泡背景色必须统一使用 `#277EFB`（钉钉官方蓝）。
- **[C2] 文本灰度分层**：
  - 核心正文：`#171A1D` (90% 不透明度)。
  - 辅助/摘要：`rgba(23, 26, 29, 0.4)` (40% 不透明度)。
- **[C3] 头像底色**：无头像时，默认底色需从钉钉蓝、橙、绿三色中随机选择（或按 ID 散列），背景色为浅色，文字为深色。

---

## 4. 输入区 (Input Area) 红线

- **[I1] 工具栏简洁度**：图标尺寸统一 `20x20`，图标间距固定 `16px`。禁止出现 3D 或 拟物化图标。
- **[I2] 焦点感官**：点击输入区时，输入框本身不应出现蓝色边框，而是保持背景无缝衔接。
- **[I3] 引用回复区**：引用消息展示在输入框上方，背景色 `#F2F4F5`，高度自适应但最大不超过 `60px`。

---

## 5. 通用组件 (General Components) 红线

- **[G1] 头像形状**：桌面端头像严禁圆形，必须是 **圆角矩形 (Radius: 4px)**。
- **[G2] Switch 开关**：统一使用 Element Plus 的 `small` 开关，高度 `20px`。
- **[G3] 未读 Badge**：背景色为钉钉红 `#FF4D4F`，文字字号 `10px`，超过 99 显示 `99+`。

---

## 6. 禁止行为 (Hard Bans)

1. **禁止**：在桌面端使用 `el-row/el-col` 进行核心栅格分配。
2. **禁止**：消息主页面出现任何超过 `12px` 的超大圆角。
3. **禁止**：背景使用纯黑 (`#000000`) 或 纯白 (`#FFFFFF`) 以外的强渐变。
4. **禁止**：在聊天主区域展示与业务无关的横向滚动条。
