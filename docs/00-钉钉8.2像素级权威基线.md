## 1. 全局骨架（Three-Pane Layout）

### 1.1 三栏宽度（**禁止改动**）

| 区域 | 宽度 | CSS Token | 说明 |
|------|------|-----------|------|
| L1 一级导航 | **64px** | `--dt-nav-sidebar-width` | 固定不可调，`flex-shrink: 0` |
| L2 二级列表 (消息模块) | **250px** | `--dt-session-panel-width` | 会话列表专用宽度 |
| L2 二级列表 (其他模块) | **240px** | `--dt-contact-panel-width` | 通讯录/待办/云盘/工作台等模块统一 240px |

| L4 详情抽屉 | **320px** | `--dt-drawer-width` | 从右侧滑出 |

**为什么消息模块用 250px，其他用 240px？**
钉钉 8.2 客户端实测：消息会话列表确实比通讯录目录宽 10px，是为了容纳更长的消息预览。其他模块用 240px 即可。

**不要再争议侧边栏宽度问题**：
- ❌ 235px / 245px / 280px / 300px 都是错的
- ✅ 消息：250px；其他：240px

## 3. 间距系统（8px 栅格）

### 3.1 允许值（**严格禁止其他**）

```
4px  / 8px  / 12px / 16px / 24px / 32px / 40px / 48px
```

| Token | 值 | 典型用途 |
|------|----|---------|
| `--dt-spacing-xs` | 4px | 图标内距/微调 |
| `--dt-spacing-sm` | 8px | 同侧元素间距 |
| `--dt-spacing-md` | 12px | 紧凑组件内边距 |
| `--dt-spacing-lg` | 16px | 标准组件内边距/列表项 padding |
| `--dt-spacing-xl` | 24px | 模块间距/页面边距 |
| `--dt-spacing-2xl` | 32px | 大间距/分组间距 |
| `--dt-spacing-3xl` | 48px | 板块间距 |

### 3.2 禁止值

❌ 7px / 10px / 13px / 14px / 15px / 17px / 18px / 20px / 22px / 25px / 30px

例外：消息气泡内边距上下 `8px`、左右 `12px` 是钉钉特殊值，**不可调整**。

## 4. 圆角系统

| Token | 值 | 用途 | **死规定** |
|-------|----|------|-----------|
| `--dt-radius-xs` | 3px | 滚动条滑块 | 仅滚动条 |
| `--dt-radius-sm` | 4px | 按钮/输入框/头像/Tag | **桌面端死规定** |
| `--dt-radius-md` | 6px | 二级按钮/小标签 | 谨慎使用 |
| `--dt-radius-lg` | 8px | 卡片/弹窗/气泡/工具栏图标容器 | 标准卡片圆角 |
| `--dt-radius-xl` | 12px | 大型卡片（仅工作台应用图标） | 仅工作台 |
| `--dt-radius-2xl` | 16px | **❌ 禁止使用** | — |
| `--dt-radius-full` | 9999px | 未读 Badge / 状态点 | 仅圆形元素 |
| `--dt-bubble-radius` | 8px | 气泡圆角基础值 | 与 `--dt-radius-lg` 同 |
| `--dt-bubble-radius-sent` | `8px 4px 8px 8px` | 发送方气泡（右上尖角） | 不可改 |
| `--dt-bubble-radius-received` | `4px 8px 8px 8px` | 接收方气泡（左上尖角） | 不可改 |

**钉钉桌面端的"圆角基因"**：
- ❌ 头像不可以是圆形（`border-radius: 50%`）
- ❌ 卡片不可以超过 12px 圆角
- ❌ 按钮不可以是胶囊形（`border-radius: 9999px`）— 仅 Badge 例外
- ✅ 头像统一 `4px`（圆角矩形）
- ✅ 按钮/输入框统一 `4px`
- ✅ 卡片/弹窗/气泡统一 `8px`

## 5. 头像与图标

### 5.1 头像尺寸

| 场景 | 尺寸 | CSS Token | 圆角 |
|------|------|-----------|------|
| 小头像（详情列表） | 24px | `--dt-avatar-size-sm` | 4px |
| 中头像（消息气泡） | **36px** | `--dt-avatar-size-md` | 4px |
| 会话列表头像 | **40px** | `--dt-avatar-session` | 4px |
| 大头像（个人详情） | 64px | `--dt-avatar-size-lg` | 4px |
| 巨大头像（个人主页） | 80px | — | 4px |

**为什么消息头像 36px 而会话列表 40px？**
钉钉 8.2 实测：消息列表里头像稍小（36px），让出空间给气泡；会话列表头像稍大（40px），让信息密度更紧凑。**这是钉钉刻意设计的细节**，不要"为了统一"全部改成同一尺寸。

### 5.2 图标尺寸

| 场景 | 尺寸 | CSS Token |
|------|------|-----------|
| Tab/小图标 | 12px | `--dt-icon-size-xs` |
| 列表小图标 | 14px | `--dt-icon-size-sm` |
| 通用图标 | 16px | `--dt-icon-size-md` |
| 工具栏图标 | **20px** | `--dt-icon-size-lg` |
| 空状态图标 | 64px | `--dt-icon-size-xl` |
| 大型 illustrations | 80px | `--dt-icon-size-2xl` |

## 8. 过渡动画时长

| 用途 | 时长 | CSS Token |
|------|------|-----------|
| 极快反馈（按钮/开关） | 100ms | — |
| 快速过渡（hover/淡入） | **150ms** | `--dt-transition-fast` |
| 标准过渡（颜色/状态） | 250ms | `--dt-transition-base` |
| 较慢动画（抽屉/弹窗） | 400ms | `--dt-transition-slow` |

**缓动函数**：统一使用 `cubic-bezier(0.4, 0, 0.2, 1)`（material-design 标准）。

## 9. 禁止行为汇总（一票否决）

| 禁止项 | 错误示例 | 正确做法 |
|--------|----------|----------|
| 圆形头像 | `border-radius: 50%` | `border-radius: 4px` |
| 大圆角 | `border-radius: 16px` | `≤ 12px` |
| 气泡渐变背景 | `linear-gradient(...)` | 纯色 `#277EFB` 或 `#FFF` |
| 复杂阴影 | `0 4px 12px rgba(0,0,0,0.5)` | 极轻阴影或无阴影 |
| 全局滚动条 | `body { overflow: auto }` | `overflow: hidden` + 局部滚动 |
| 复杂 CSS 动画 | `@keyframes float`, `transform: translateY(-2px) scale(1.05)` | 仅颜色/背景过渡 |
| el-row/el-col 主布局 | `<el-row>` 做三栏 | `display: flex` |
| 移动端响应式折叠 | `@media (max-width: 750)` 折叠 L2 | 出底部横向滚动条 |
| 旧主色 | `#0089FF` / `#1677FF` | `#277EFB` |
| 浮动位移 hover | `:hover { transform: translateY(-2px) }` | 仅颜色变化 |
| filter 模糊 | `backdrop-filter: blur(20px)` | 实色背景 |
