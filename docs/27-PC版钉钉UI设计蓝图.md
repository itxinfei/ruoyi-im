# 27 PC版钉钉UI设计蓝图

> **用途**：将散落在 `20/21/22/25/26/35/49` 的设计规范收口成一份面向产品、设计、前端与 AI 的 PC 版钉钉 UI 总蓝图。  
> **说明**：本文件只定义桌面端页面设计策略、页面模板、模块映射、交付物与验收路径；像素级尺寸、颜色和 CSS 实现仍以 `20-UI交互规范总纲.md`、`21-IM桌面端页面规范.md`、`26-IM交互深度行为规范.md`、`35-前端全局样式与主题配置.md`、`49-通话界面UI设计规范.md` 及 `.claude/rules/dingtalk-ui-prompt.md` 为准。

---

## 1. 目标与适用范围

### 1.1 目标口径

1. 本项目要做的是“企业内网可部署的 Web 版钉钉桌面端”，不是移动端拉伸版，也不是后台管理系统套壳。
2. UI 设计必须优先还原钉钉 Windows 客户端的桌面办公体验：稳定框架、高信息密度、低装饰、强状态反馈、主流程闭环。
3. 任何新增页面、重构页面、组件复刻、AI 生成代码任务，都必须先回到本文件确认页面类型与设计交付要求。

### 1.2 适用模块

1. 消息、搜索、通讯录、群详情/个人详情、通话
2. 工作台、云盘、审批、日历、待办
3. 设置、个人资料、系统配置类页面

### 1.3 不在本文件直接展开的内容

1. 移动端/响应式折叠方案
2. 品牌素材、运营插画、开放平台生态能力
3. 具体接口、数据库、状态机实现细节

## 3. PC版钉钉UI的五条设计原则

### 3.1 Desktop First，不做移动端化让步

1. 页面根布局必须服务桌面办公场景，优先保证横向多栏稳定，不允许把 L2 折叠成移动端抽屉。
2. 宽度不足时允许出现底部横向滚动，不允许破坏导航、列表、内容区的固定层级。

### 3.2 稳定骨架优先于局部炫技

1. 先定义 L1/L2/L3 主骨架，再设计列表、卡片、表单和局部操作。
2. 次级信息默认通过抽屉、弹窗、右键菜单、Popover 暴露，不新增常驻第四列。

### 3.3 IM 是平台中心，其他模块共享同一桌面语言

1. 消息、搜索、通话、文件、群详情必须优先贴近钉钉桌面端。
2. 工作台、审批、日历、待办、云盘等模块可以有业务差异，但不能偏离统一的空间逻辑、密度和状态反馈。

### 3.4 低噪音、高密度、强反馈

1. 页面应以留白、边框、浅底色和轻量 hover 区分层级，避免大阴影、大圆角、渐变、悬浮位移。
2. 列表项、工具栏、输入区必须体现“桌面效率优先”，而不是移动端卡片堆叠。
3. 所有关键操作都必须有 hover、active、disabled、loading、empty、error 六类状态定义。

### 3.5 设计必须能直接落到代码和验收

1. 文档中的每条 UI 规则都要能映射到具体 Token、组件、CSS 或验收用例。
2. 不允许只写“参考钉钉风格”这类主观描述，必须写清结构、状态、入口、尺寸或引用文档。

## 4. 全局页面骨架

### 4.1 标准桌面骨架

| 层级 | 作用 | 设计要求 | 参考文档 |
|-----|------|----------|---------|
| `L1` | 一级导航 | 固定 `64px`，深色侧边栏，承载全局主模块切换 | `20` / `21` / `25` |
| `L2` | 二级导航/列表区 | 固定 `250px`，承载会话、筛选、目录、二级导航 | `20` / `21` / `25` |
| `L3` | 主内容区 | `flex: 1`，最小宽度受桌面办公场景约束，承载主任务流 | `20` / `21` / `25` |
| `Overlay` | 抽屉/弹窗/菜单 | 仅承载详情、补充操作、临时确认，不打断主骨架 | `20` / `26` / `49` |

### 4.2 常见页面层级

1. **一级导航层**：消息、搜索、通讯录、工作台、云盘、日历、待办、审批、设置
2. **二级结构层**：筛选、标签、分组、树、列表、工具栏
3. **主任务层**：聊天、审批详情、日历主视图、文件列表、待办详情
4. **补充信息层**：详情抽屉、操作弹窗、右键菜单、轻提示

### 4.3 统一的次级面板策略

1. 详情信息优先使用右侧抽屉，而不是在主区永久占位。
2. 高风险操作使用确认弹窗，轻量操作使用下拉菜单或气泡菜单。
3. 页面内临时创建动作优先使用模态弹窗；跨页面长流程则进入独立主页面。

## 5. PC页面模板库

| 页面模板 | 适用模块 | 标准结构 | 必须强调的体验 | 严禁出现的问题 |
|---------|---------|----------|----------------|----------------|
| **三栏主链路页** | 消息、搜索主页面 | `L1 + L2 + L3` | 导航稳定、信息密度高、详情触发式展开 | 响应式折叠、卡片化聊天区、右侧常驻详情列 |
| **列表-详情协同页** | 通讯录、云盘、待办、审批、日历 | `L1 + 左侧筛选/目录 + 主列表/主视图 + 抽屉详情` | 快速筛选、批量操作、详情不打断主任务 | 每条记录做成大卡片、操作入口隐藏过深 |
| **工作台聚合页** | 工作台首页、门户页 | `L1 + 主内容分组区` | 入口清晰、优先级明确、模块分组稳定 | 大量装饰插画、无效快捷入口、视觉噪音过高 |
| **表单/流程页** | 审批发起、设置、资料编辑 | `L1 + 表单主区 + 固定操作区` | 输入效率、校验反馈、保存/提交路径明确 | 提交按钮位置漂移、错误信息无锚点 |
| **全屏协作页** | 音视频通话、分享演示 | 全屏主画布 + 轻量控制层 + 悬浮补充信息 | 聚焦主任务、控制区低干扰、异常状态清晰 | 多余侧栏、复杂阴影、抢视觉的背景层 |

### 5.1 模板选择规则

1. 页面以持续沟通、持续筛选、持续查看为主时，优先使用“三栏主链路页”或“列表-详情协同页”。
2. 页面以任务汇总和入口分发为主时，使用“工作台聚合页”。
3. 页面以结构化录入和审批提交为主时，使用“表单/流程页”。
4. 页面以实时音视频为主时，使用“全屏协作页”。

## 7. 跨模块统一组件契约

### 7.1 必须统一的基础组件

1. **一级导航项**：图标尺寸、选中态、hover 态、底部头像区风格一致。
2. **列表项**：高度、左右 padding、标题/摘要层级、未读/状态标签风格一致。
3. **工具栏**：按钮尺寸、排列密度、图标反馈一致，禁止出现一页一套按钮风格。
4. **抽屉/弹窗**：标题、操作区、关闭逻辑、遮罩和层级统一。
5. **空状态/错误状态**：说明文案简洁，带主行动按钮，不使用花哨插画抢占空间。

### 7.2 必须统一的状态集合

1. 默认态
2. Hover 态
3. Active/Selected 态
4. Disabled 态
5. Loading 态
6. Empty 态
7. Error/Offline 态

### 7.3 必须统一的 Token 入口

1. 颜色、字体、边框、阴影、圆角、间距一律优先引用 `35-前端全局样式与主题配置.md` 中的 CSS 变量。
2. AI 生成代码时，禁止硬编码非标准蓝、非标准圆角和非栅格间距。
3. 若新增组件需要新 Token，必须先补 `35`，再补具体组件文档。

## 8. 设计交付与文档回写要求

### 8.1 每个 PC UI 设计任务至少要产出

1. **页面定位**：属于哪个模块、哪个页面模板、主要用户任务是什么。
2. **结构图**：L1/L2/L3 或主视图/抽屉/弹窗的区域划分。
3. **状态清单**：默认、hover、selected、loading、empty、error、disabled。
4. **关键尺寸**：宽度、高度、间距、圆角、图标尺寸、按钮规格。
5. **交互路径**：入口、主流程、异常流程、返回路径。
6. **验收点**：必须写成可检查条目，而不是笼统描述。

### 8.2 发生下列变化时，必须同步更新的文档

| 变化类型 | 必须同步更新的文档 |
|---------|------------------|
| 一级导航或路由变化 | `14-页面路由与菜单映射.md`、`15-产品形态与导航清单.md` |
| 与钉钉对齐策略变化 | `16-设计复刻差异.md`、`23-UI复刻任务计划.md` |
| IM 页面结构/细节变化 | `21-IM桌面端页面规范.md`、`26-IM交互深度行为规范.md` |
| 页面级验收口径变化 | `25-钉钉Windows复刻基线.md`、`54-验收用例清单.md` |

### 8.3 AI 任务卡强制引用

1. 若任务目标是“PC 版钉钉 UI 设计/复刻”，任务卡必须同时引用本文件、`20`、`25`，以及对应模块规范。

## 9. 设计完成定义（Definition of Done）

1. 页面骨架符合 PC 桌面布局，不出现移动端化折叠。
2. 视觉风格与钉钉 Windows 客户端一致：低噪音、轻阴影、8px 栅格、圆角矩形头像。
3. 关键路径具备真实入口和真实反馈，不存在死按钮、空白页、假菜单。
4. 默认/hover/active/loading/empty/error 等状态齐全。
5. 已补充对应模块文档、验收口径和任务引用，不留“代码改了但文档没跟上”的断层。
