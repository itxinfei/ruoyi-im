# 73 钉钉撤回与重新编辑规范 (Recall & Re-edit Spec)

> **用途**：定义消息撤回提示和"重新编辑"功能的视觉与交互。这是钉钉桌面端的标志性微交互。
> **权威依据**：docs/00-钉钉8.2像素级权威基线.md / 视觉资产 #12-bubble-recall.png
> **优先级**：⭐⭐ 高（用户高频使用）

---

## 1. 撤回流程总览

### 1.1 业务规则

| 规则 | 说明 |
|------|------|
| 撤回时限 | 发送后 2 分钟内可撤回 |
| 谁能撤回 | 仅发送者本人 |
| 撤回提示 | 群内所有人可见 |
| 重新编辑 | 仅发送者自己可看到"重新编辑"链接，且仅限文本/链接消息 |
| 撤回媒体 | 图片/文件等可撤回，但不显示"重新编辑" |
| 已读消息 | 仍可撤回（即使对方已读） |

### 1.2 视觉特点

撤回后的消息**不显示原始气泡**，而是替换为**居中的灰色提示文字**：

```
普通消息：
  ┌──────────────────┐
  │  原始消息内容     │
  └──────────────────┘

撤回后：
        "你撤回了一条消息  重新编辑"
        (居中、灰色、单行)
```

## 2. 撤回提示的视觉规范

### 2.1 三种文案变体

| 场景 | 显示文案 | 备注 |
|-----|---------|------|
| 自己撤回（文本类） | "你撤回了一条消息  重新编辑" | 含蓝色"重新编辑"链接 |
| 自己撤回（媒体类） | "你撤回了一条消息" | 无重新编辑链接 |
| 他人撤回 | "张三 撤回了一条消息" | 显示对方昵称 |
| 群主撤回他人消息 | "群主 撤回了 张三 的一条消息" | 行政场景 |

| 属性 | 值 | CSS Token |
|-----|----|-----|
| 容器 | 单行居中 | text-align: center |
| 上下间距 | 8px | var(--dt-spacing-sm) |
| 文字色 | #8A9099 灰 | var(--dt-recall-text-color) |
| 文字大小 | 12px | var(--dt-font-size-sm) |
| 字重 | 400 | var(--dt-font-weight-normal) |

### 2.4 模板结构

```vue
<template>
  <div class="message-recall">
    <span class="recall-text">{{ recallText }}</span>
    <span
      v-if="canReedit"
      class="recall-edit-link"
      @click="$emit('reedit', message)"
    >
      重新编辑
    </span>
  </div>
</template>

<script setup>
import { computed } from 'vue'

const props = defineProps({
  message: Object,
  isOwn: Boolean,
  currentUserId: [Number, String]
})

const recallText = computed(() => {
  if (props.isOwn) {
    return '你撤回了一条消息'
  }
  // 群主撤回他人消息
  if (props.message.recallByAdmin) {
    return `群主撤回了 ${props.message.senderName} 的一条消息`
  }
  // 他人撤回
  return `${props.message.senderName} 撤回了一条消息`
})

const canReedit = computed(() => {
  if (!props.isOwn) return false
  // 仅文本和链接类消息允许重新编辑
  const editableTypes = ['TEXT', 'LINK']
  if (!editableTypes.includes(props.message.originalMessageType)) return false
  // 撤回时间内
  const recallTime = new Date(props.message.recallTime).getTime()
  const now = Date.now()
  return now - recallTime < 5 * 60 * 1000  // 5 分钟内可重新编辑
})
</script>
```

## 3. 重新编辑流程

### 3.1 触发与行为

```
用户点击"重新编辑"
   ↓
读取原始消息内容
   ↓
填充到输入框（替换当前草稿，给出提示 toast "已恢复撤回内容"）
   ↓
焦点跳转到输入框末尾
   ↓
用户继续编辑/直接重新发送
```

### 3.2 实现要点

```javascript
function handleReedit(recalledMessage) {
  // 1. 恢复内容到输入框
  const editor = chatInputAreaRef.value
  editor.setContent(recalledMessage.originalContent)

  // 2. 恢复 @ 提及上下文（如果原消息有）
  if (recalledMessage.mentions?.length) {
    editor.setMentions(recalledMessage.mentions)
  }

  // 3. 提示用户
  ElMessage.success({
    message: '已恢复撤回内容，可继续编辑',
    duration: 1500
  })

  // 4. 焦点 + 移到末尾
  editor.focus()
  editor.moveCursorToEnd()
}
```

### 3.3 重新编辑窗口期

- **可编辑时长**：撤回后 5 分钟内
- **超时处理**：5 分钟后"重新编辑"链接灰显，hover 时 tooltip "撤回超过 5 分钟，无法恢复"
- **新消息覆盖**：用户已经发了新消息后，"重新编辑"仍可用，恢复内容会追加到当前草稿

## 4. 撤回交互入口

### 4.1 触发途径

| 入口 | 操作 |
|-----|-----|
| 消息 hover 操作浮条 | 点击 ⋯ 更多 -> 撤回 |
| 消息右键菜单 | "撤回"项 |
| 多选模式 | 选中后底部"撤回"按钮 |

### 4.2 确认对话框

撤回操作**无需二次确认**（与微信不同），直接撤回。

但需要服务端校验：
- 时限校验（2 分钟内）
- 权限校验（仅本人或群主）
- 失败时 toast 错误："消息发送超过 2 分钟，无法撤回"

---

## 5. 撤回后会话列表的处理

### 5.1 最后一条消息的更新

如果被撤回的是会话的"最后一条消息"，会话列表预览区需要：

| 场景 | 处理 |
|-----|-----|
| 被撤回的是最新消息 | 替换为"你撤回了一条消息"（无前缀） |
| 撤回后还有新消息 | 显示新消息内容 |
| 群聊场景 | 显示"张三撤回了一条消息" |

### 5.2 未读数处理

- 如果消息被撤回时**对方还未读**：未读数 -1
- 如果消息被撤回时**对方已读**：未读数不变

## 7. 反例（禁止）
