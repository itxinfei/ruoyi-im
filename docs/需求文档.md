## 一、 信息架构 (IA) 与菜单层级协议 (强制执行)

### 1.1 一级导航 (Main Nav - 64px)
固定侧边模块：消息、通讯录、工作台、我的。

### 1.2 二级菜单 (Sub Panel - 280px)
每个模块点入后，中栏必须展现以下固定的分类或树结构：

#### 1.2.1 消息 (Chat) 模块二级布局
- **顶部**：全局搜索框 + 发起群聊 (+)
- **分类 Tab (Pills)**：全部、未读、@我、置顶。
- **列表区**：会话单元（头像 + 名称 + 摘要 + 时间 + 未读红点 + 置顶标志）。

#### 1.2.2 通讯录 (Contacts) 模块二级布局
- **顶部**：搜索联系人/部门
- **固定项**：新的联系人、我的群组、我的好友。
- **动态树**：组织架构树（支持异步展开、显示部门人数）。

#### 1.2.3 工作台 (Workbench) 模块二级布局
- **列表式导航**：全部应用、我发起的、我审批的、待办任务。

### 1.3 三级菜单 (Context Menus & Toolbars)
统一全站微交互入口：

#### 1.3.1 会话/联系人右键菜单 (Context Menu)
- **标准项**：置顶/取消置顶、标记已读/未读、免打扰开关、删除。
- **视觉**：180px 宽度，10px 圆角，Icon 居左（字重 300）。

#### 1.3.2 消息气泡操作栏 (Hover Toolbar)
- **触发**：消息气泡悬停时出现在右侧/上方。
- **标准动作**：回复、表情回应、转发、更多（编辑、撤回、收藏、多选、删除）。

### 1.4 四级详情 (Pop-up Dialogs)
废弃侧边栏，所有深层详情必须以弹窗呈现：
- **UserProfileDialog**：名片式个人资料。
- **GroupProfileDialog**：群设置、成员管理、公告、文件库入口。
- **ForwardDialog**：多选会话转发。

## 二、 UI 布局与审美红线 (v8.0 强制标准)

### 2.1 黄金尺寸锁定
1. **中栏侧边栏**：**`280px`**。必须配置 `flex-shrink: 0`。
2. **气泡圆角**：**`10px`** 对称圆角。
3. **栅格系统**：严格执行 `8pt` (4/8/12/16/24/32)。
