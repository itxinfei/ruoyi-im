## 一、环境准备

### 1.1 JDK 安装

**Windows**：

1. 下载 JDK 1.8：https://www.oracle.com/java/technologies/javase/javase8-archive-downloads.html
2. 安装到 `C:\Program Files\Java\jdk1.8.0_xxx`
3. 配置环境变量：
   - `JAVA_HOME` = `C:\Program Files\Java\jdk1.8.0_xxx`
   - `Path` 添加 `%JAVA_HOME%\bin`

**验证**：

```bash
java -version
# 输出：java version "1.8.0_xxx"
```

### 1.2 Node.js 安装

**Windows**：

1. 下载 Node.js 16+：https://nodejs.org/
2. 安装 LTS 版本
3. 验证安装：

```bash
node -v
# 输出：v16.x.x 或更高

npm -v
# 输出：8.x.x 或更高
```

**配置 npm 镜像（可选）**：

```bash
npm config set registry https://registry.npmmirror.com
```

### 1.3 MySQL 安装

**Windows**：

1. 下载 MySQL 5.7：https://dev.mysql.com/downloads/mysql/5.7.html
2. 安装并设置 root 密码
3. 配置字符集为 utf8mb4

**my.ini 配置**：

```ini
[mysqld]
character-set-server=utf8mb4
collation-server=utf8mb4_general_ci
max_connections=500

[client]
default-character-set=utf8mb4
```

**验证**：

```bash
mysql -u root -p
# 输入密码后进入 MySQL 命令行
```

### 1.4 Redis 安装

**Windows**：

1. 下载 Redis for Windows：https://github.com/microsoftarchive/redis/releases
2. 解压到 `C:\Redis`
3. 启动 Redis：

```bash
redis-server.exe redis.windows.conf
```

**验证**：

```bash
redis-cli ping
# 输出：PONG
```

### 1.5 IDE 配置

**后端 IDE（IntelliJ IDEA）**：

1. 下载 IntelliJ IDEA：https://www.jetbrains.com/idea/
2. 安装插件：
   - Lombok
   - MyBatisX
   - Spring Boot Assistant

**前端 IDE（VS Code）**：

1. 下载 VS Code：https://code.visualstudio.com/
2. 安装插件：
   - Vue - Official
   - ESLint
   - Prettier

## 二、项目配置

### 2.1 克隆项目

```bash
# 克隆项目
git clone https://your-repo/im.git
cd im
```

### 2.2 后端配置

**1. 导入数据库**：

```bash
# 登录 MySQL
mysql -u root -p

# 创建数据库
CREATE DATABASE im DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_general_ci;

# 导入数据
USE im;
SOURCE sql/im.sql;
```

**2. 修改配置文件**：

编辑 `ruoyi-im-api/src/main/resources/application-dev.yml`：

```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/im?useUnicode=true&characterEncoding=utf8mb4&useSSL=false&serverTimezone=Asia/Shanghai
    username: root
    password: your_password
  
  redis:
    host: localhost
    port: 6379
    password: 
    database: 0

# 文件上传路径
file:
  upload:
    path: D:/im-files
```

**3. IDE 导入项目**：

- IntelliJ IDEA：File → Open → 选择 `ruoyi-im-api` 目录
- 等待 Maven 依赖下载完成

### 2.3 前端配置

**1. 安装依赖**：

```bash
cd ruoyi-im-web
npm install
```

**2. 配置环境变量**：

创建 `.env.development`：

```env
VITE_API_BASE_URL=http://localhost:8080/api
VITE_WS_URL=ws://localhost:8080/ws
VITE_APP_TITLE=RuoYi-IM (Dev)
```

## 五、开发规范

### 5.1 后端开发规范

**代码规范**：

- 使用 Lombok 注解简化代码
- Service 层统一抛出 BusinessException
- Controller 层统一返回 AjaxResult

**命名规范**：

- Controller：`XxxController`
- Service：`XxxService` / `XxxServiceImpl`
- Mapper：`XxxMapper`
- Entity：`ImXxx`

**示例**：

```java
@RestController
@RequestMapping("/api/message")
@RequiredArgsConstructor
public class MessageController {

    private final MessageService messageService;

    @PostMapping("/send")
    public AjaxResult send(@RequestBody MessageDTO dto) {
        return AjaxResult.success(messageService.send(dto));
    }
}
```

> **依赖注入规范**：必须使用构造器注入（`@RequiredArgsConstructor` + `final` 字段），禁止使用 `@Autowired` 字段注入。

### 5.2 前端开发规范

**代码规范**：

- 使用 Composition API + `<script setup>`
- 组件命名：PascalCase

**目录结构**：

```
src/
├── api/           # API 接口
├── assets/        # 静态资源
├── components/    # 公共组件
├── composables/   # 组合式函数
├── router/        # 路由配置
├── stores/        # Pinia 状态
├── types/         # 类型定义
└── views/         # 页面组件
```

**示例**：

```vue
<template>
  <div class="message-list">
    <MessageItem 
      v-for="msg in messages" 
      :key="msg.id" 
      :message="msg"
    />
  </div>
</template>

<script setup lang="js">
import { ref, onMounted } from 'vue';
import { messageApi } from '@/api/message';
import MessageItem from './MessageItem.vue';

const props = defineProps({
  conversationId: Number
});

const messages = ref([]);

const loadMessages = async () => {
  messages.value = await messageApi.getList(props.conversationId);
};

onMounted(() => {
  loadMessages();
});
</script>
```

## 六、常见问题

### 6.1 后端问题

| 问题 | 原因 | 解决方案 |
|-----|------|---------|
| 端口 8080 被占用 | 其他程序占用 | 修改端口或关闭占用程序 |
| 数据库连接失败 | MySQL 未启动或配置错误 | 检查 MySQL 状态和配置 |
| Redis 连接失败 | Redis 未启动 | 启动 Redis 服务 |
| Maven 依赖下载慢 | 网络问题 | 配置 Maven 镜像 |

**Maven 镜像配置**（settings.xml）：

```xml
<mirror>
  <id>aliyun</id>
  <mirrorOf>central</mirrorOf>
  <name>Aliyun Maven</name>
  <url>https://maven.aliyun.com/repository/public</url>
</mirror>
```

### 6.2 前端问题

| 问题 | 原因 | 解决方案 |
|-----|------|---------|
| npm install 失败 | 网络问题 | 配置 npm 镜像 |
| 页面白屏 | 路由或组件错误 | 检查控制台错误 |
| API 请求跨域 | 后端未配置 CORS | 配置后端 CORS |
| WebSocket 连接失败 | 后端未启动 | 启动后端服务 |

**Vite 代理配置**（vite.config.ts）：

```javascript
export default defineConfig({
  server: {
    proxy: {
      '/api': {
        target: 'http://localhost:8080',
        changeOrigin: true
      },
      '/ws': {
        target: 'ws://localhost:8080',
        ws: true
      }
    }
  }
});
```

### 6.3 数据库问题

| 问题 | 原因 | 解决方案 |
|-----|------|---------|
| 字符集错误 | 未配置 utf8mb4 | 修改数据库和表字符集 |
| 连接数不足 | 连接池配置不当 | 调整 max_connections |
| 查询慢 | 缺少索引 | 添加索引 |
