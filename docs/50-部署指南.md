## 二、后端部署

### 2.1 打包

```bash
# 进入后端项目目录
cd ruoyi-im-api

# Maven 打包（跳过测试）
mvn clean package -DskipTests

# 打包产物
# target/ruoyi-im-api.jar
```

### 2.2 配置文件

**application-prod.yml**：

```yaml
server:
  port: 8080

spring:
  datasource:
    url: jdbc:mysql://localhost:3306/im?useUnicode=true&characterEncoding=utf8mb4&useSSL=false
    username: im_user
    password: ${DB_PASSWORD}
    driver-class-name: com.mysql.cj.jdbc.Driver
  
  redis:
    host: localhost
    port: 6379
    password: ${REDIS_PASSWORD}
    database: 0

# 文件上传配置
file:
  upload:
    path: /data/im/files
    max-size: 104857600

# JWT 配置
security:
  jwt:
    secret: ${JWT_SECRET}
    access-token-expiration: 7200000
    refresh-token-expiration: 604800000

# 日志配置
logging:
  level:
    root: INFO
    com.ruoyi.im: DEBUG
  file:
    name: /data/im/logs/app.log
```

### 2.3 启动脚本

**start.sh**：

```bash
#!/bin/bash

APP_NAME="ruoyi-im-api.jar"
APP_HOME="/opt/im"
LOG_FILE="/data/im/logs/startup.log"

# JVM 参数
JAVA_OPTS="-Xms512m -Xmx1024m -XX:+UseG1GC"

# 启动服务
nohup java $JAVA_OPTS -jar $APP_HOME/$APP_NAME \
  --spring.profiles.active=prod \
  > $LOG_FILE 2>&1 &

echo "Service started. PID: $!"
```

**stop.sh**：

```bash
#!/bin/bash

APP_NAME="ruoyi-im-api.jar"

PID=$(ps -ef | grep $APP_NAME | grep -v grep | awk '{print $2}')

if [ -n "$PID" ]; then
  kill -15 $PID
  echo "Service stopped. PID: $PID"
else
  echo "Service not running"
fi
```

### 2.4 Systemd 服务

**/etc/systemd/system/im-api.service**：

```ini
[Unit]
Description=RuoYi-IM API Service
After=network.target mysql.service redis.service

[Service]
Type=simple
User=im
WorkingDirectory=/opt/im
ExecStart=/usr/bin/java -Xms512m -Xmx1024m -jar /opt/im/ruoyi-im-api.jar --spring.profiles.active=prod
ExecStop=/bin/kill -15 $MAINPID
Restart=on-failure
RestartSec=10

[Install]
WantedBy=multi-user.target
```

```bash
# 启用服务
systemctl enable im-api

# 启动服务
systemctl start im-api

# 查看状态
systemctl status im-api
```

## 三、前端部署

### 3.1 构建

```bash
# 进入前端项目目录
cd ruoyi-im-web

# 安装依赖
npm install

# 构建生产版本
npm run build

# 构建产物
# dist/
```

### 3.2 环境配置

**.env.production**：

```env
VITE_API_BASE_URL=https://im.yourcompany.com/api
VITE_WS_URL=wss://im.yourcompany.com/ws
VITE_APP_TITLE=RuoYi-IM
```

### 3.3 Nginx 配置

**/etc/nginx/conf.d/im.conf**：

```nginx
# HTTP 重定向到 HTTPS
server {
    listen 80;
    server_name im.yourcompany.com;
    return 301 https://$server_name$request_uri;
}

# HTTPS 配置
server {
    listen 443 ssl http2;
    server_name im.yourcompany.com;

    # SSL 证书
    ssl_certificate /etc/nginx/ssl/im.yourcompany.com.crt;
    ssl_certificate_key /etc/nginx/ssl/im.yourcompany.com.key;
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers HIGH:!aNULL:!MD5;

    # 前端静态资源
    root /opt/im/web/dist;
    index index.html;

    # 前端路由（Vue Router History 模式）
    location / {
        try_files $uri $uri/ /index.html;
    }

    # 后端 API 代理
    location /api/ {
        proxy_pass http://127.0.0.1:8080/api/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    # WebSocket 代理
    location /ws {
        proxy_pass http://127.0.0.1:8080/ws;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_read_timeout 86400;
    }

    # 文件上传大小限制
    client_max_body_size 100M;

    # 静态资源缓存
    location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff|woff2|ttf|eot)$ {
        expires 30d;
        add_header Cache-Control "public, immutable";
    }

    # Gzip 压缩
    gzip on;
    gzip_types text/plain text/css application/json application/javascript text/xml application/xml;
    gzip_min_length 1024;
}
```

## 五、Redis 配置

### 5.1 redis.conf

```conf
# 绑定地址
bind 127.0.0.1

# 端口
port 6379

# 密码
requirepass your_redis_password

# 数据目录
dir /data/redis

# 最大内存
maxmemory 1gb
maxmemory-policy allkeys-lru

# 持久化
appendonly yes
appendfsync everysec
```

### 5.2 启动 Redis

```bash
redis-server /etc/redis/redis.conf
```

## 七、常见问题

### 7.1 后端启动失败

| 问题 | 原因 | 解决方案 |
|-----|------|---------|
| 数据库连接失败 | MySQL 未启动或配置错误 | 检查 MySQL 状态和配置 |
| Redis 连接失败 | Redis 未启动或密码错误 | 检查 Redis 状态和密码 |
| 端口被占用 | 8080 端口被其他程序占用 | 修改端口或停止占用程序 |

### 7.2 前端访问异常

| 问题 | 原因 | 解决方案 |
|-----|------|---------|
| 白屏 | 构建失败或路由配置错误 | 检查构建日志和 Nginx 配置 |
| API 404 | 代理配置错误 | 检查 Nginx 代理配置 |
| WebSocket 连接失败 | WSS 配置错误 | 检查 SSL 证书和代理配置 |

### 7.3 性能问题

| 问题 | 原因 | 解决方案 |
|-----|------|---------|
| 响应慢 | 数据库查询慢 | 添加索引，优化查询 |
| 内存不足 | JVM 内存配置过小 | 调整 -Xmx 参数 |
| 连接超时 | 连接池配置不当 | 调整连接池参数 |

## 八、备份策略

### 8.1 数据库备份

```bash
# 每日备份脚本
#!/bin/bash
BACKUP_DIR="/data/backup/mysql"
DATE=$(date +%Y%m%d_%H%M%S)
mysqldump -u im_user -p'your_password' im > $BACKUP_DIR/im_$DATE.sql

# 保留最近 7 天备份
find $BACKUP_DIR -name "im_*.sql" -mtime +7 -delete
```

### 8.2 文件备份

```bash
# 每日备份上传文件
rsync -avz /data/im/files/ /data/backup/files/
```
