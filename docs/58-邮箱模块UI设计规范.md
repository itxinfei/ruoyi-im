# 58 邮箱模块UI设计规范

> **用途**：定义内置企业邮箱的页面布局、邮件列表呈现及写信交互规范。  
> **设计基准**：钉钉 Windows 8.2.0 桌面端，遵循 `docs/27-PC版钉钉UI设计蓝图.md`。

---

## 1. 页面定位与入口

*   **模块 ID**：`mail`
*   **页面模板**：三栏主链路页 (`L1 + L2 + L3`)
*   **主要任务**：
    1.  管理企业内外部邮件（收件箱、已发送、草稿箱、回收站）。
    2.  撰写并发送带附件的正式邮件。
    3.  邮件的分类、标记（已读/未读、红旗）与快速处理。

---

## 2. 空间布局 (Layout)

### 2.1 整体骨架
*   **L2 (邮件夹/分类)**：宽度 `200px`，含：收件箱、星标、草稿、已发送、垃圾邮件。
*   **L3 (邮件列表)**：宽度 `300px` 或 `flex: 0.4`，显示邮件摘要。
*   **L4 (读信区)**：`flex: 1`，全宽展示当前选中的邮件详情。

### 2.2 邮件摘要项 (Mail Item)
*   **高度**：`80px`（两行高度）。
*   **第一行**：发件人、日期、红旗图标。
*   **第二行**：邮件标题（Bold）、附件图标。
*   **第三行**：正文片段（Secondary Color, 12px）。

---

## 3. 组件级设计规范

### 3.1 读信界面 (Mail Viewer)
*   **头部**：标题（20px Bold）、收/发件人详情、时间。
*   **正文**：支持 HTML 渲染，背景色 `var(--dt-bg-card)`。
*   **附件展示区**：底部固定显示，卡片式布局，带“预览/下载”按钮。

### 3.2 写信编辑器 (Mail Composer)
*   **顶部栏**：含“发送”、“存草稿”、“添加附件”按钮。
*   **地址栏**：收件人、抄送、密送，支持从通讯录选取。
*   **编辑器**：使用标准富文本编辑器（如 wangeditor 或 native textarea），支持图片粘贴与拖拽。

---

## 4. 状态清单 (States)

| 状态 | 表现 |
|-----|------|
| **Unread** | 邮件摘要项标题加粗，左侧带 `4px` 蓝色圆点指示器。 |
| **Flagged** | 邮件右侧显示红色星标，背景色微调为浅橙色。 |
| **Sending** | 写信页显示遮罩进度条，提示“正在发送...”。 |
| **Empty** | “收件箱为空”状态提示。 |

---

## 5. 交互路径 (Interactions)

1.  **快速处理**：邮件列表行 Hover 时，出现“删除”、“标为已读”、“红旗”的悬浮图标。
2.  **回复/转发**：读信区顶部显示“回复”、“全部回复”、“转发”按钮，点击后原地切换为编辑器模式。
3.  **联系人联动**：点击发件人头像 -> 弹出通讯录详情抽屉 -> 可一键发起聊天或加为好友。

---

## 6. 验收点 (AC)

- [ ] AC1: 列表-详情布局稳定，且支持中间列表宽度的自由拉伸。
- [ ] AC2: 邮件搜索支持“按发件人”和“按关键词”进行多维度过滤。
- [ ] AC3: 写信页支持“附件自动扫描”和“超大附件”链接转换。
- [ ] AC4: UI 严格遵循 `design-tokens.scss` 变量。
