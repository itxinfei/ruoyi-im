# 25 钉钉 Windows 复刻基线

> 用途：按页面定义"必须复刻到什么程度"与"怎么验收"，避免复刻任务只停留在主观判断。
> 说明：本文件不描述代码实现细节，只定义页面级目标与验收口径。

## 1. 适用范围

1. 消息主页面（会话栏 + 聊天区 + 详情区）
2. 搜索页面
3. 群详情/个人详情页面
4. 左侧导航模块页面（通讯录、工作台、云盘、待办等）

## 2. 复刻等级定义

1. `L1`：结构一致。布局和模块层级与钉钉 Windows 一致。
2. `L2`：视觉一致。间距、尺寸、状态色、组件密度一致。
3. `L3`：交互一致。点击、悬停、右键、空状态、失败反馈一致。
4. `L4`：功能闭环。入口真实可达，主流程和异常流程可完成。

交付要求：

1. IM 主链路页面最低达到 `L4`。
2. 非 IM 页面最低达到 `L2`，且关键入口要达到 `L3`。

## 3. 页面基线表

| 页面 | 复刻目标 | 必须达到 | 对应任务 |
|------|----------|----------|----------|
| 消息主页面 | 三栏布局与信息密度对齐钉钉 Windows | L1 + L2 + L3 + L4 | IM-UI-001 |
| 消息气泡与时间线 | 发送方气泡在前头像在后，间距/宽度对齐 | L2 + L3 + L4 | IM-UI-002 |
| 聊天头部 + 输入区 | 按钮密度、工具入口、快捷键、上传反馈一致 | L2 + L3 + L4 | IM-UI-003 |
| 群详情/个人详情 | 入口统一、层级清晰、功能可用 | L1 + L2 + L3 + L4 | IM-UI-004 |
| 空入口与不可用按钮 | 不出现死路入口或无响应按钮 | L3 + L4 | IM-FIX-001 |
| 搜索页面 | 多入口一致、结果可跳转可定位 | L2 + L3 + L4 | IM-FIX-002 |
| 群文件页面 | 上传/下载/筛选闭环，失败反馈明确 | L3 + L4 | IM-FIX-003 |

## 4. 验收口径

1. 每个页面至少提供一组“当前 vs 钉钉”对比截图。
2. 验收必须同时覆盖视觉、交互和功能闭环。
3. 如果某项无法达到对应等级，必须在 `04-待确认问题.md` 写明原因。
4. 验收结果回写到 `03-实现状态矩阵.md` 与任务卡 `acceptance`。

## 5. 使用流程

1. 先从 `24-页面缺陷与功能缺口清单.md` 选问题。
2. 在 `llm-tasks.yaml` 选择或新增对应任务。
3. 执行前对照本基线定义本次目标等级。
4. 执行后按 `54-验收用例清单.md` + 本文档完成验收。
