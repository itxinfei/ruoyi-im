# 76 前端训练语料目录与摘要（LLM路由版）

> **用途**：把 `docs/` 中与前端页面、样式、布局、交互、响应式、性能和验收相关的文档整理成可直接供大模型读取的训练语料目录。  
> **适用范围**：前端页面设计、Vue 组件开发、UI 复刻、样式治理、性能优化、交互修复、视觉验收。  
> **使用原则**：训练或检索时，优先使用**完整文件名**而不是仅使用编号；遇到冲突时按本文件定义的权威顺序处理。

---

## 1. 语料定位

本文件不替代原始规范，而是为大模型提供三类能力：

1. 明确哪些文档属于前端训练核心语料。
2. 明确这些文档之间的权威层级、依赖关系与读取顺序。
3. 明确哪些地方存在重复、冲突或编号歧义，避免模型学错。

## 4. 文档间关系与分工

### 4.1 `00` 与 `25`

- `00-钉钉8.2像素级权威基线.md` 负责“数值、颜色、间距、圆角、截图证据”。
- `25-钉钉Windows复刻基线.md` 负责“页面级完成等级和验收口径”。
- 结论：**涉及像素真值时永远先信 `00`，涉及页面完成定义时再看 `25`。**

### 4.2 `20`、`22`、`21`、`26`

- `20` 负责跨页面统一原则。
- `22` 负责负面红线与验收清单。
- `21` 负责 IM 页面骨架。
- `26` 负责 IM 微交互、状态机和细节行为。
- 结论：**`20/22` 是总规则，`21/26` 是 IM 实现层。**

### 4.3 `35` 与 `75`

- `35` 负责“当前项目样式体系应该怎么写”。
- `75` 负责“当前项目样式体系应该怎么优化”。
- 结论：**写代码先看 `35`，做优化和修复策略再看 `75`。**

### 4.4 `16`、`23`、`24`

- `16` 负责“和钉钉相比还差什么”。
- `23` 负责“这些差距要怎么拆任务推进”。
- `24` 负责“当前页面具体坏在哪里”。
- 结论：**`16` 偏设计差距，`23` 偏计划，`24` 偏现状缺陷。**

## 5. 已识别冲突与歧义

### 5.1 已解决编号冲突：原 `41`

当前已将原先与安全文档冲突的“设置/个人资料 UI 规范”独立调整为：

- `39-设置与个人资料模块UI设计规范.md`

保留的 `41-安全与权限模块设计.md` 只作为后端/安全设计文档存在。  
这意味着前端训练和检索阶段不再需要用“前端 41 / 后端 41”的临时口径区分。

### 5.2 历史权威顺序漂移：`00` 与旧路由

部分入口文档曾优先引导到 `25`、`20`、`27`，但未把 `00-钉钉8.2像素级权威基线.md` 放在前面。  
当前整理后的统一口径是：

- **数值权威**：先看 `00`
- **页面模板**：再看 `27`
- **全局规则**：再看 `20/22`
- **页面验收**：再看 `25`
- **实现与优化**：最后看 `21/26/35/49/70-76`

### 5.3 样式实现与优化方法的角色混淆

若只看 `35`，模型容易知道“现在有哪些 token”，但不知道“如何系统优化布局与性能”。  
若只看 `75`，模型容易知道“应该怎么优化”，但不知道“项目实际样式入口和 token 真相源在哪里”。

**统一口径：**

- 样式实现以 `35` 为准。
- 样式优化与性能治理以 `75` 为准。

## 6. 训练切片建议

为了让大模型更稳定地使用这些文档，建议按以下元数据切片：

| 字段 | 含义 |
|------|------|
| `doc_id` | 完整文件名，不只保留编号 |
| `layer` | 路由层 / 权威层 / IM层 / 模块层 / 验收层 |
| `authority` | `highest` / `high` / `normal` |
| `task_type` | `layout` / `style` / `interaction` / `performance` / `qa` |
| `scope` | global / IM / module / specialized |
| `upstream_docs` | 该文档依赖哪些上游文档 |
| `downstream_docs` | 该文档影响哪些下游文档 |
| `summary` | 1-2 句摘要 |

**切片粒度建议：**

1. 目录类文档按 `##` 级标题切片。
2. 规范类文档按“页面/组件/状态/验收”切片。
3. 红线类文档按“红线类别”切片。
4. 表格类文档保留表头与表体，不要只截局部单行。
