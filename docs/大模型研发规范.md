# RuoYi-IM 大模型研发规范（LLM Engineering Spec）

> 文档目的：约束 Claude/Codex 等模型在本项目中的规划、开发、审查行为，确保输出稳定、可执行、可验收。
> 适用范围：`D:/MyCode/im/ruoyi-im-web`、`D:/MyCode/im/ruoyi-im-api`。

---

## 1. 基本原则

1. **桌面生产力第一**：本项目仅服务于 Windows 环境下的 Google Chrome 浏览器。禁止任何移动端兼容逻辑。
2. **先可用，再完美**：优先打通桌面端主流程闭环（鼠标、键盘、大屏交互）。
3. **单任务单目标**：一次只完成一个 `task_id`。
4. **证据驱动**：任何"已完成"必须有代码和验证结果。
5. **技术栈强约束**：不允许擅自切换 JDK/Vue 主版本。

---

## 2. 输入规范（模型开工前）

每次请求必须包含：

1. `task_id`
2. `goal`
3. `scope_in`（允许改动文件）
4. `scope_out`（禁止改动文件）
5. `acceptance`（AC1/AC2/AC3）
6. `verify_cmd`（至少 `npm run build`）

---

## 3. 前端设计强制红线（禁止飘逸）

1. **桌面生产力优先**：禁止编写任何移动端兼容代码。所有交互必须优先考虑“鼠标滚轮、右键菜单、键盘快捷键”。
2. **Chrome 性能锁死**：利用 Google Chrome 内核特性（如 `contain` 属性、`IntersectionObserver`、`Canvas` 渲染优化）确保长列表 60fps。
3. **变量化原则**：严禁在 `.vue` 或 `.scss` 文件中硬编码 `px` (布局除外) 和 `hex色值`。必须统一使用 `design-tokens.scss` 中的变量。
4. **布局守卫**：新生成的布局必须遵守 `flex-shrink: 0` 与 `min-width: 0` 配对原则，确保在 100vw/100vh 下不出现非预期的双重滚动条。
5. **图标规范**：仅允许使用 `Material Icons Outlined` 或 `Element Plus Icons`。
6. **状态闭环**：AI 必须为每个交互生成 `loading` 处理和 `error-catch` 后的提示逻辑，禁止“断头”逻辑。

## 5. 质量闸门（DoD）

所有任务完成必须满足：

1. 编译通过（前端 `npm run build`）
2. 主路径无"开发中"死路入口
3. 键盘/鼠标桌面端交互闭环
4. 新增功能有异常提示与降级策略
5. 文档状态同步

## 6. 布局与审美强制红线 (AI 协作协议 v1.4)

> 本章规则旨在锁定全站 UI 的“中轴线”与“细节美学”，AI 编码时必须严格执行。

1. **240px 黄金宽度**：所有具备二级导航的面板，其左侧边栏宽度必须固定为 `240px`。
2. **极简通知系统**：主导航禁止使用带数字的 `menu-badge`。统一使用 **直径 8px、带 1.5px 描边的呼吸感红点**。
3. **气泡美学锁死**：
   - **对方气泡**：左上角直角 (`4px 12px 12px 12px`)，背景纯白，边框 `rgba(31,35,41,0.08)`。
   - **自己气泡**：右上角直角 (`12px 4px 12px 12px`)，背景淡蓝 (`#E8F4FF`)，边框 `rgba(22,119,255,0.15)`。
4. **像素级对齐**：消息流中，头像顶部必须与气泡内第一行文本基线水平对齐（通常需 `margin-top: 2px`）。
5. **图标标准**：`Material Icons` 字重统一锁定为 `300`。严禁使用默认加粗状态。
