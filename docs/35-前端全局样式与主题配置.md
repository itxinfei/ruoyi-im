# 35 前端全局样式与主题配置 (Theme & Styles)

> **用途**：统一内网环境下 UI 的视觉基因，确保所有样式基于项目实际的 CSS 变量和设计令牌（design‑tokens），防止硬编码和错误的 Tailwind 引用。

---

## 1. 样式体系概述

项目采用 **CSS 变量 + SCSS 设计令牌** 的方式管理颜色、间距、圆角等全局视觉属性。所有组件样式应通过 `var(--xxx)` 或 SCSS 变量引用，避免直接写死颜色或尺寸。

- **设计令牌文件**：`src/styles/design-tokens.scss` 定义了颜色、字体、阴影等变量，并在 `:root` 中导出为 CSS 变量。
- **全局样式入口**：`src/styles/global.scss` 包含基础重置、Element Plus 覆盖以及自定义实用类（如 `.flex-center`）。
- **Tailwind**：项目中仅保留 Tailwind 的 **工具类指令**（如 `@apply`），不作为直接的 UI 类库使用，文档中不应出现 `bg-primary-sidebar` 等 Tailwind 类的硬编码示例。

---

## 2. 钉钉级视觉契约 (Visual Contract)

在 `src/assets/styles/variables.css` 中强制声明以下变量，所有组件开发必须以此为准：

```css
:root {

  --dt-brand-hover: #33A1FF;
  --dt-brand-active: #0077E6;
  --dt-brand-bg: #EBF5FF;              /* 浅蓝色背景 */

  /* 中性色 - 信息分层 (Official Neutrals) */
  --dt-text-main: #171A1D;             /* 核心正文 (90% opacity) */
  --dt-text-desc: rgba(23, 26, 29, 0.6); /* 辅助/摘要 (60% opacity) */
  --dt-text-hint: rgba(23, 26, 29, 0.4); /* 提示信息 (40% opacity) */
  --dt-text-white: #FFFFFF;
```

## 3. 组件级样式指南 (Implementation Logic)

为了让 UI 不产生“廉价感”，AI 在编写 Vue 组件时必须遵循以下逻辑：

### 3.1 会话列表项 (Session Item)
- **高度**：固定 `64px`（极致紧凑）。
- **背景**：默认透明，Hover 使用 `var(--dt-bg-hover)`，选中时使用 `var(--dt-bg-selected)` 且左侧带一条 `3px` 蓝边。
- **摘要文字**：必须限制为 `ellipsis` (单行溢出省略)，使用 `var(--dt-text-desc)`。

### 3.2 消息气泡 (Message Bubble)
- **间距**：同侧气泡间距 `4px`，异侧 `16px`。
- **头像**：圆角固定 `4px`（不要用圆形，圆形是移动端基因），尺寸 `36x36`。
- **文字**：行高 `1.5`，字体大小 `14px`。

### 3.3 全局边框
- 所有卡片和面板的边框必须是 `var(--dt-line-width) solid var(--dt-border-color)`。
- 禁止使用 `box-shadow` 做大面积投影，仅允许在悬浮菜单使用极轻微阴影。

## 3. 本地图标索引 (Local Icons)

项目统一使用本地 SVG 图标或 Element Plus 内置图标，禁止外部 CDN。通过 `<svg-icon name="..." />` 组件引用。

| 图标 ID | 对应功能 | 来源 |
| :--- | :--- | :--- |
| `icon-msg` | 消息模块 | Element Plus `ChatDotRound` |
| `icon-contact` | 通讯录 | Element Plus `User` |
| `icon-workbench` | 工作台 | Element Plus `Menu` |
| `icon-search` | 全局搜索 | Element Plus `Search` |
| `icon-file-pdf` | PDF 文件 | 本地 `src/assets/icons/file-pdf.svg` |
| `icon-file-word` | Word 文件 | 本地 `src/assets/icons/file-word.svg` |

### 调用方式
```vue
<svg-icon name="icon-msg" class="text-primary" />
```

## 4. 全局实用类 (Utility Classes)

为配合 CSS 变量，项目提供了一套基于 **Tailwind @apply** 的实用类，仅用于快速布局，**不**用于颜色或尺寸硬编码。

```scss
/* src/styles/utilities.scss */
@layer utilities {
  .flex-center { @apply flex items-center justify-center; }
  .text-primary { color: var(--im-text-primary); }
  .bg-primary { background-color: var(--im-primary-color); }
  .rounded-bubble { border-radius: var(--im-radius-bubble); }
}
```

> **注意**：在组件样式中应优先使用上述实用类或直接引用 CSS 变量，避免写死 `#1677FF`、`8px` 等值。

---

## 5. AI 编码指令 (Styling Rules)

1. **严禁硬编码颜色或尺寸**：所有样式必须通过 CSS 变量或实用类实现。
2. **统一字体**：使用系统字体栈 `PingFang SC, Microsoft YaHei, system-ui`，不引入外部 Google Fonts。
3. **气泡对齐**：聊天气泡布局使用 `flex items-start`，确保头像与气泡顶部基线对齐。
4. **暗色模式适配**：通过 `data-theme="dark"` 自动切换变量，组件无需额外处理。
5. **全局滚动条**：在 `global.scss` 中统一样式，确保在暗色模式下颜色可见。

---

## 6. 维护与扩展指引

- **新增颜色**：在 `design-tokens.scss` 中添加 SCSS 变量并同步到 `variables.css`。
- **新增实用类**：在 `utilities.scss` 中使用 `@apply` 组合已有原子类，保持一致性。
- **图标扩展**：将 SVG 放入 `src/assets/icons/`，并在 `icon-index.md` 中登记。
