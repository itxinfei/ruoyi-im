# 57 AI助手模块UI设计规范

> **用途**：定义 AI 助手模块的对话交互、气泡样式、Prompt 模板及快捷工具 UI。  
> **设计基准**：钉钉 Windows 8.2.0 桌面端，遵循 `docs/27-PC版钉钉UI设计蓝图.md`。

---

## 1. 页面定位与入口

*   **模块 ID**：`assistant`
*   **页面模板**：三栏主链路页 / 独立抽屉
*   **主要任务**：
    1.  与企业级 AI 大模型进行文本对话。
    2.  利用 AI 对长聊天记录、长文档进行自动总结。
    3.  调用预设的办公 Prompt（如周报润色、翻译、起草通知）。

---

## 2. 空间布局 (Layout)

### 2.1 对话主界面 (Chat View)
*   **左侧列表**：历史对话记录，按时间排序。
*   **中间对话区**：`flex: 1`，采用流式输出布局。
*   **底部输入区**：
    *   **快捷指令栏**：高度 `36px`，横向滚动，显示常用 Prompt。
    *   **文本输入框**：支持换行，最大高度 `200px`，带“发送”和“停止生成”按钮。

### 2.2 总结侧边栏 (AI Summary Sidepanel)
*   **触发**：在聊天窗口、云盘文档页点击“AI 总结”按钮。
*   **结构**：固定宽度 `320px`，顶部显示“AI 正在阅读...”，下方显示结构化要点。

---

## 3. 组件级设计规范

### 3.1 AI 专属气泡 (AI Bubble)
*   **背景色**：
    *   用户端：同普通消息发送方（品牌蓝渐变）。
    *   AI 端：`#F2F4F7`（浅灰）或 `var(--dt-brand-bg)`（浅蓝），带极细紫色描边区分。
*   **字体**：`var(--dt-font-family)`，行高 `1.6` 以适应长文本阅读。
*   **流式输出 (Streaming)**：文字逐个弹出时，末尾带一个闪烁的 `|` 光标。

### 3.2 结构化响应卡片 (Card Response)
*   当 AI 返回代码、表格或多级标题时：
    *   **代码块**：使用深色背景（`#1E1E1E`），带“点击复制”图标。
    *   **Markdown 表格**：全宽展示，带斑马纹区分。

### 3.3 Prompt 快捷入口 (Command Chips)
*   **样式**：胶囊型按钮，圆角 `var(--dt-radius-full)`，背景白底蓝边。
*   **文字**：如“✍️ 润色周报”、“📊 数据分析”、“🌐 中英翻译”。

---

## 4. 状态清单 (States)

| 状态 | 表现 |
|-----|------|
| **Thinking** | AI 头像显示呼吸灯动效，输入框上方显示“AI 正在思考...”。 |
| **Streaming** | 自动滚动到底部，允许用户随时点击“停止”。 |
| **History Loading** | 对话历史触顶加载时显示转圈图标。 |
| **Failed** | AI 气泡末尾显示红色重试图标。 |

---

## 5. 交互路径 (Interactions)

1.  **发起问答**：点击“起草通知”芯片 -> 输入框自动填入预设 Prompt 模板 -> 用户修改后按 Enter 发送。
2.  **内容反馈**：每条 AI 回复下方带“点赞/点踩”和“复制”图标。
3.  **引用聊天记录**：在普通聊天页多选消息 -> 点击“AI 总结” -> AI 助手自动弹出并开始分析选中内容。

---

## 6. 验收点 (AC)

- [ ] AC1: 支持完整的 Markdown 渲染（加粗、列表、链接、公式）。
- [ ] AC2: 历史对话按天进行时间分割，并支持一键清空全部历史。
- [ ] AC3: 输入区支持 `Ctrl + Enter` 发送，`Enter` 换行（或用户可配置）。
- [ ] AC4: UI 严格遵循 `design-tokens.scss`，不出现硬编码颜色。
