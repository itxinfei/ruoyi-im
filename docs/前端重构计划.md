# 前端全面重构计划

## 重构目标
将RuoYi-IM前端从"能用"升级到"专业级"，提升代码可维护性、可读性和一致性。

## 当前问题清单

### P0 严重问题
1. **全局样式过长**：`global.scss` 1045行，维护困难
2. **样式紧凑可读性差**：多个CSS属性写在一行，难以阅读和修改
3. **Token缺失**：部分硬编码值未使用Design Token

### P1 中等问题
4. **组件过大**：`ChatMessageBubble.vue` 916行，`ChatWindow.vue` 490行
5. **SCSS/Tailwind混用**：可能导致样式冲突

### P2 轻微问题
6. **注释不足**：部分代码缺少必要注释
7. **命名不统一**：部分class命名风格不一致

### Phase 2: 组件样式可读性优化
**目标**：将紧凑的单行样式拆分为多行，提高可读性

**具体任务**：
1. `ChatMessageBubble.vue` - 拆分916行中的紧凑样式
2. `ChatWindow.vue` - 拆分490行中的紧凑样式
3. `ImSideNavNew/index.vue` - 拆分159行中的紧凑样式
4. `ContactsPanel.vue` - 拆分198行中的紧凑样式
5. `MainPage.vue` - 拆分101行中的紧凑样式

**拆分规则**：
- 每个CSS属性单独一行
- 添加适当的空行分隔逻辑块
- 添加必要的注释说明

**示例对比**：
```scss
// 之前（紧凑）
.nav-v4-item-wrap {
  position: relative; width: 100%; height: 48px; display: flex; align-items: center; justify-content: center;
  cursor: pointer; transition: background-color 0.2s;
}

// 之后（可读）
.nav-v4-item-wrap {
  position: relative;
  width: 100%;
  height: 48px;
  display: flex;
  align-items: center;
  justify-content: center;
  cursor: pointer;
  transition: background-color 0.2s;
}
```

**验证方式**：
- 运行`npm run build`确认无报错
- 目视检查样式是否一致

**预计耗时**：3-4小时

### Phase 3: Design Token完善
**目标**：补充缺失的Token变量，消除硬编码值

**具体任务**：
1. 扫描所有组件文件，找出硬编码的：
   - 颜色值（如`#277EFB`、`rgba(0,0,0,0.5)`）
   - 尺寸值（如`4px`、`16px`）
   - 圆角值（如`4px`、`8px`）
   - 间距值（如`8px`、`16px`）
2. 在`design-tokens.scss`中补充缺失的Token
3. 将硬编码值替换为Token引用

**验证方式**：
- 使用grep命令检查是否还有硬编码值
- 运行`npm run build`确认无报错

**预计耗时**：2-3小时

   - `MessageBubbleText.vue` - 文本消息
   - `MessageBubbleImage.vue` - 图片消息
   - `MessageBubbleVideo.vue` - 视频消息
   - `MessageBubbleFile.vue` - 文件消息
   - `MessageBubbleCard.vue` - 名片消息
   - `MessageBubbleLocation.vue` - 位置消息
   - `MessageBubbleVoice.vue` - 语音消息
   - `MessageActions.vue` - 悬浮操作栏

2. `ChatWindow.vue` (490行) 拆分：
   - `ChatWindowHeader.vue` - 头部（已存在）
   - `ChatMessageList.vue` - 消息列表（已存在）
   - `ChatInputArea.vue` - 输入区（已存在）
   - `ChatDetailDrawer.vue` - 详情抽屉（已存在）
   - `EmojiPicker.vue` - 表情选择器（新组件）

**验证方式**：
- 运行`npm run build`确认无报错
- 测试聊天功能是否正常

**预计耗时**：4-5小时

---

### Phase 5: 代码规范统一
**目标**：统一命名风格、添加必要注释

**具体任务**：
1. 统一CSS类命名：
   - 使用BEM命名规范（如`chat-window__header`）
   - 或保持现有风格（如`chat-window-header`），但确保一致
2. 添加必要的注释：
   - 组件用途说明
   - 复杂逻辑说明
   - 钉钉规范引用

**验证方式**：
- 代码审查确认命名一致性
- 运行`npm run build`确认无报错

**预计耗时**：2-3小时

---

### Phase 6: 最终验证
**目标**：确保所有重构不影响功能

**具体任务**：
1. 运行`npm run build`确认构建成功
2. 运行`npm run dev`启动开发服务器
3. 手动测试核心功能：
   - 登录/登出
   - 聊天消息发送/接收
   - 文件上传
   - 通讯录浏览
   - 暗色模式切换
4. 运行`npm run lint`检查代码规范

**验证方式**：
- 构建成功
- 功能正常
- 代码规范通过

**预计耗时**：1-2小时
