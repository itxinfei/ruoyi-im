# 10 产品需求总纲

## 1. 项目背景与目标
- **项目名称**：RuoYi-IM
- **核心定位**：基于 RuoYi 框架的企业级内网私有化即时通讯平台。
- **对标产品**：钉钉 (DingTalk) Windows 客户端 8.2.0。
- **核心目标**：实现"找人快、发信准、协作稳"。IM 消息流是所有业务的驱动引擎，非 IM 功能（审批、日历）必须与 IM 深度联动。

## 2. 核心功能逻辑指令集 (Execution-Ready Logic)

### 2.1 消息生命周期状态机 (The Message State Machine)
AI 在实现消息功能时必须遵循以下流转逻辑：
- **发送流**：`PENDING` (UI 创建占位) -> `SENDING` (WS 发出) -> `SENT` (收到 ACK) / `FAILED` (超时)。
- **操作流**：
    - `REVOKE`：仅限 `SENT` 状态且发送时间 < 2min，触发后 UI 替换为“你撤回了一条消息”。
    - `EDIT`：仅限 `SENT` 状态且发送时间 < 5min，成功后全员推送 `MSG_EDIT_NOTIFY` 并实时替换内容。

### 2.2 实时通讯 (WebSocket) 推送协议
- **消息可靠性**：必须实现"应用层 ACK"。客户端发送消息后 5s 未收到服务端回执，必须自动转为 `FAILED` 状态并提示重试。
- **并发控制**：单会话并发上限 500 人。

### 2.3 权限与隔离指令
- **离职处理逻辑**：封禁 `sys_user` -> 断开 WS -> 遍历并移除所有 `im_group_member` -> 自动转让其名下群主。
- **数据可见性**：普通用户禁止调用 `admin` 命名空间下的任何 API；管理员操作必须记录到审计表。

### 2.4 组织架构与联系人 (Contacts) - P1
- **架构树**：支持无限级部门展示，点击部门异步加载成员。
- **用户画像**：展示本地头像、在线状态、部门职位、入职时间。

### 2.5 协同办公模块 (Collaboration) - P1
- **审批**：基于 JSON 配置的动态表单，支持流程审批与 WSS 实时通知。
- **待办**：支持“消息一键转待办”，待办过期推送。
- **云盘**：内网分片上传，支持文件夹管理与权限隔离。

### 2.6 核心用户旅程 (User Journeys)

### 旅程1：新用户首次登录

```
用户角色：新入职员工
触发条件：首次登录系统
目标：发现同事并发起第一次会话

步骤：
1. 登录系统 → 进入消息首页
2. 点击通讯录 → 查看组织架构
3. 展开部门 → 找到同事
4. 点击同事头像 → 查看详情
5. 点击"发消息" → 进入会话
6. 输入消息 → 发送

成功标准：≤5步完成首次会话发起
```

### 旅程2：日常沟通

```
用户角色：普通员工
触发条件：收到新消息
目标：查看消息并回复

步骤：
1. 收到桌面通知/未读提示
2. 点击会话 → 进入聊天
3. 查看消息内容
4. 输入回复 → 发送
5. 消息状态变为"已发送"

成功标准：消息发送成功率≥99.9%
```

### 旅程3：群组协作

```
用户角色：项目经理
触发条件：需要创建项目群
目标：创建群组并邀请成员协作

步骤：
1. 点击"创建群组"
2. 搜索并选择成员（≥2人）
3. 设置群名称
4. 创建成功 → 进入群聊
5. 发布群公告
6. 上传共享文件

成功标准：群组创建≤3步，文件上传成功率≥99%
```

### 旅程4：消息搜索

```
用户角色：普通员工
触发条件：需要查找历史消息
目标：找到特定消息并定位上下文

步骤：
1. 点击搜索框
2. 输入关键词
3. 查看搜索结果（消息/联系人/群组/文件）
4. 点击消息结果 → 跳转到原消息位置
5. 查看上下文

成功标准：搜索响应≤500ms，结果可跳转
```

### 旅程5：文件协作

```
用户角色：普通员工
触发条件：需要发送文件给同事
目标：上传并发送文件

步骤：
1. 进入目标会话
2. 点击文件按钮/拖拽文件/粘贴文件
3. 选择本地文件
4. 查看上传进度
5. 文件发送成功

成功标准：上传成功率≥99%，进度可见
```

## 3. 全栈技术栈选型 (Tech Stack)

| 维度 | 选型 | 约束条件 |
| :--- | :--- | :--- |
| **后端语言** | Java 1.8 | **绝对禁止** Java 9+ 语法 (如 var, record)。核心消息链路必须参照 `docs/backend/01-消息模块后端代码重构与规范.md` 执行 DTO/VO 隔离。 |
| **框架** | Spring Boot 2.x + RuoYi | 遵循 Controller -> Service -> Mapper 单向调用。 |
| **数据库** | MySQL 5.7 + Redis | 使用 MyBatis Plus 进行 Lambda 查询，消息 ID 等采用雪花算法避免递增依赖。 |
| **前端框架** | Vue 3 (Vite) | 必须使用 `<script setup>` 组合式 API，消息模块参照 `docs/frontend/01-消息模块Vue组件化与性能优化.md` 进行彻底组件化拆解。 |
| **UI 组件库** | Element Plus | 仅允许使用本地图标，**禁止外部 CDN**。全局体验必须对齐钉钉，参照 `docs/ui-ux/01-消息核心模块交互与视觉规范.md`。 |
| **实时通讯** | WebSocket | 必须具备心跳检测与断线重连逻辑，前端考虑弱网发送队列缓存机制，后端采用事件解耦推送体系。 |
