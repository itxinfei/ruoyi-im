# 29 工作台模块UI设计规范

> **用途**：定义工作台（应用中心）的页面布局、应用卡片规范及交互标准。
> **设计基准**：钉钉 Windows 8.2.0 桌面端，遵循 `docs/27-PC版钉钉UI设计蓝图.md`、`docs/22-UI视觉与交互红线.md`。
> **版本**：v2.0 - 反例修订版（移除磨砂玻璃 / 图标位移 / 渐变背景违规项）

---

## 1. 页面定位与模板

*   **模块 ID**：`workbench`
*   **页面模板**：工作台聚合页 (`L1 + L2 + Main View`)
*   **主要任务**：
    1.  快速访问常用业务系统（审批、待办、报销等）。
    2.  查看个人办公数据概览（今日待办、未读 DING）。
    3.  搜索及发现企业内部应用。

---

## 2. 空间布局 (Layout)

### 2.1 整体骨架
*   **L2 (二级导航)**：宽度 `240px`，承载"应用中心 / 我的应用 / 应用管理"三档导航；背景使用 `var(--dt-bg-body)`，与白色主区域形成层级对比。
*   **L3 (主体区域)**：宽度 `flex: 1`，背景 `var(--dt-bg-body)`，内容区最大宽度限制为 `1100px`（大屏下水平居中），上下 `padding: 24px`，左右 `padding: 32px`。
*   **头部 Banner**：高度 `160px`，**纯品牌蓝单色背景**（`var(--dt-brand-color)`），**严禁使用渐变或品牌插画**，仅承载问候语 + 个人概览。
*   **应用分组区**：纵向排列，每组包含标题（带 `4px` 品牌蓝竖线指示器）和应用栅格。

### 2.2 栅格规范 (App Grid)
*   **应用卡片尺寸**：宽 `minmax(240px, 1fr)`，高 `auto`，使用 `display: grid; grid-template-columns: repeat(auto-fill, minmax(240px, 1fr)); gap: 16px;`。
*   **每行卡片数量**：根据容器宽度自适应（最多 4 列），**不固定为 5 列**（避免桌面端宽屏空白过大或窄屏挤压）。
*   **最近使用横向滚动区**：固定高度 `92px`（图标 48 + 间距 12 + 文字 12 + 上下留白），横向间距 `24px`，溢出时使用主题滚动条样式。

## 3. 组件级设计规范

### 3.1 应用图标卡片 (App Card)
*   **卡片整体**：背景 `var(--dt-bg-card)`，内边距 `16px`，圆角 `var(--dt-radius-lg)` (`8px`)，边框 `1px solid transparent`。
*   **图标区**：`44px * 44px`。
    *   圆角：`var(--dt-radius-lg)` (`8px`)，**严禁超过 12px 圆角**。
    *   背景：浅色业务色，优先复用 `var(--dt-brand-bg)` 或现有语义浅色 token，避免在模块文档中继续扩散新的十六进制样本。
*   **文字区**：
    *   主标题：`14px` Primary `font-weight: 600`，单行截断。
    *   描述：`11px` `var(--dt-text-quaternary)`，单行截断。
*   **交互**：
    *   Hover 时仅边框变品牌蓝（`border-color: var(--dt-brand-color)`）+ 极轻微阴影 `var(--dt-shadow-1)`。
    *   **严禁** `transform: translateY` 上移、`transform: scale` 缩放或背景渐变。
    *   过渡时长统一为 `var(--dt-transition-fast)` (200ms)。

### 3.2 概览统计块 (Quick Stats)
*   **位置**：Banner 右侧内嵌（与 Banner 同一容器），**不使用悬浮窗或独立卡片**。
*   **内容**：
    *   "待办"：显示未完成数，点击跳转待办模块。
    *   "未读 DING"：显示未读 DING 数，点击跳转 DING 模块。
*   **样式**：在品牌蓝 Banner 上覆盖 `rgba(255, 255, 255, 0.15)` 半透明白底（**纯色叠加，禁止使用 backdrop-filter 模糊效果**），圆角 `var(--dt-radius-lg)`，边框 `1px solid rgba(255, 255, 255, 0.2)`。
*   **数字与文字**：数字 `22px` `font-weight: 700`，文字 `11px` `opacity: 0.8`，均为白色。
*   **分隔线**：两个 stat 之间使用 `1px * 30px` 的 `rgba(255,255,255,0.2)` 竖线。

### 3.3 应用分类组 (Group Header)
*   **标题**：`15px` `font-weight: 700`，左侧 `4px` 品牌蓝竖线指示器（高度等于标题行高）。
*   **操作**：右侧带"管理"或"更多"链接（`14px` 品牌蓝，hover 仅出现下划线，**不使用** `box-shadow` / `transform`）。
*   **段间距**：每个分组上下间距 `32px`，分组内卡片栅格间距 `16px`。

### 3.4 最近使用项 (Recent Item)
*   **整体**：`flex-direction: column; align-items: center; gap: 10px;`。
*   **图标圆形容器**：`48px * 48px`，圆角 `var(--dt-radius-lg)` (`8px`)，**禁止使用 50% 圆形**（与会话头像规范一致）。
*   **图标内容**：白色，`24px` 字号。
*   **文字**：`12px` `var(--dt-text-secondary)`，单行截断。
*   **交互**：Hover 时整项 `cursor: pointer`，背景透明度无变化（保持简洁）。

## 4. 状态清单 (States)

| 状态 | 表现 | 实现要点 |
|-----|------|---------|
| **Default** | 卡片白底 + 透明边框 + 文字 Primary | 默认基线 |
| **Hover** | 卡片边框变品牌蓝 `var(--dt-brand-color)` + 轻阴影 `var(--dt-shadow-1)` | 仅样式变化，无位移/缩放 |
| **Active (按下)** | 卡片背景轻微加深至 `var(--dt-bg-hover)` | 持续 100ms 内出现 |
| **Loading** | 卡片整体替换为骨架屏（`120px * 76px` 浅色占位矩形 + `40 * 40` 图标占位 + `120 * 14` 文字占位） | 骨架优先使用 `var(--dt-bg-hover)` / `var(--dt-border-lighter)` 作为占位底色，不新增专用 token |
| **Empty** | 分类下无应用时居中显示"暂无可用应用"图标 + 文字（图标 64px `var(--dt-text-quaternary)`，文字 14px Tertiary） | 与统一空状态规范一致 |
| **Disabled** | 卡片整体 `opacity: 0.5; cursor: not-allowed;` | Hover 时显示 Tooltip "暂无访问权限" |
| **Error** | 应用加载失败时卡片右上角显示红色感叹号 + 文字"加载失败" | 点击可重试 |

## 5. 交互路径 (Interactions)

### 5.1 主路径
1.  **应用启动**：点击图标 → 检查应用类型 → 内部路由应用直接跳转；外部应用通过受控的 iframe 或新标签页打开。
2.  **搜索应用**：顶部搜索框输入 → 结果实时过滤 → 优先匹配"常用应用"，次匹配应用名称，再匹配应用描述。
3.  **自定义布局**：点击"管理常用" → 进入编辑模式 → 应用卡片右上角出现 `×` 按钮 → 支持拖拽重排（Drag & Drop） → 退出编辑模式自动保存。

### 5.2 键盘快捷键
| 快捷键 | 行为 |
|--------|------|
| `/` | 聚焦搜索框 |
| `Esc` | 退出搜索 / 退出编辑模式 |
| `↑ ↓ ← →` | 在应用栅格中切换焦点 |
| `Enter` | 启动当前焦点应用 |
| `Ctrl + E` | 进入/退出编辑模式 |

### 5.3 拖拽反馈
*   拖拽过程中被拖拽卡片 `opacity: 0.5`，目标位置出现 `2px` 虚线品牌蓝指示。
*   拖拽时鼠标光标变为 `grabbing`，松开后恢复 `pointer`。
*   **禁止使用 ghost image 模糊效果**，保持简洁。

## 6. 验收点 (AC)

- [ ] AC1: 应用栅格在 1280 / 1440 / 1920 三档分辨率下自动适配（不同列数自适应）。
- [ ] AC2: 应用图标在 Retina 屏下保持清晰（使用 SVG 或 3x 图片）。
- [ ] AC3: 概览数据与后端实时联动（支持 WebSocket 推送更新），无数据时不显示空白卡。
- [ ] AC4: Banner **绝对纯色**，不出现任何 `linear-gradient` / `radial-gradient`。
- [ ] AC5: 应用卡片 hover 仅边框 + 阴影变化，**无任何 transform 位移或缩放**。
- [ ] AC6: 概览统计块**不使用** `backdrop-filter`（纯色半透明叠加替代）。
- [ ] AC7: 状态切换（Loading/Empty/Error）有完整 UI 占位，不出现"瞬间空白"或"白屏"。
