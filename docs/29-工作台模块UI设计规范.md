# 29 工作台模块UI设计规范

> **用途**：定义工作台（应用中心）的页面布局、应用卡片规范及交互标准。  
> **设计基准**：钉钉 Windows 8.2.0 桌面端，遵循 `docs/27-PC版钉钉UI设计蓝图.md`。

---

## 1. 页面定位与模板

*   **模块 ID**：`workbench`
*   **页面模板**：工作台聚合页 (`L1 + Main View`)
*   **主要任务**：
    1.  快速访问常用业务系统（审批、待办、报销等）。
    2.  查看个人办公数据概览（今日待办、待处理审批）。
    3.  搜索及发现企业内部应用。

---

## 2. 空间布局 (Layout)

### 2.1 整体骨架
*   **主体区域**：宽度 `flex: 1`，最大宽度建议限制为 `1120px`（大屏下居中），左右保持 `var(--dt-spacing-xl)` 的留白。
*   **头部 Banner**：高度 `160px`，使用轻量化渐变或极简品牌插画，承载全局搜索和个人概览。
*   **应用分组区**：纵向排列，每组包含标题和应用栅格。

### 2.2 栅格规范 (App Grid)
*   **应用卡片尺寸**：固定宽度 `96px`，高度 `112px`。
*   **排列方式**：`display: grid; grid-template-columns: repeat(auto-fill, 96px); gap: 24px;`。

---

## 3. 组件级设计规范

### 3.1 应用图标卡片 (App Card)
*   **图标区**：`48px * 48px`。
    *   圆角：`12px` (`var(--dt-radius-xl)`)。
    *   背景：纯色或轻微阴影。
*   **文字区**：
    *   主标题：`13px` Primary，单行截断。
    *   副标题（可选）：`11px` Tertiary。
*   **交互**：Hover 态卡片背景变为 `var(--dt-bg-hover)`，图标轻微向上位移 `2px`。

### 3.2 概览统计块 (Quick Stats)
*   **位置**：Banner 右侧悬浮或固定在顶部。
*   **内容**：
    *   “待办”：显示未完成数，点击跳转待办模块。
    *   “审批”：显示待处理数，点击跳转审批模块。
    *   “日志”：显示今日是否提交。
*   **样式**：磨砂玻璃效果，圆角 `var(--dt-radius-lg)`。

### 3.3 应用分类组 (Group Header)
*   **标题**：`16px` Bold，左侧带 `4px` 品牌蓝竖线指示器。
*   **操作**：右侧带“管理”或“更多”链接。

---

## 4. 状态清单 (States)

| 状态 | 表现 |
|-----|------|
| **Loading** | 应用卡片显示灰色占位块（Skeleton Screen）。 |
| **Empty** | 分类下无应用时显示“暂无可用应用”。 |
| **Hover** | 卡片浮现，显示“添加到常用”星星图标。 |
| **Disabled** | 无权限应用图标显示 50% 透明度，Hover 提示“暂无访问权限”。 |

---

## 5. 交互路径 (Interactions)

1.  **应用启动**：点击图标 -> 检查应用类型 -> 内部路由应用直接跳转；外部应用打开新标签页。
2.  **搜索应用**：顶部搜索框输入 -> 结果实时过滤 -> 优先匹配“常用应用”。
3.  **自定义布局**：点击“管理常用” -> 进入编辑模式 -> 应用卡片出现 `Delete` 按钮 -> 支持拖拽重排（Drag & Drop）。

---

## 6. 验收点 (AC)

- [ ] AC1: 应用栅格在不同宽度下自动适配（自适应列数）。
- [ ] AC2: 应用图标在 Retina 屏下保持清晰（使用 SVG 或 3x 图片）。
- [ ] AC3: 概览数据与后端实时联动（支持 WebSocket 推送更新）。
- [ ] AC4: 背景渐变和 Banner 严格遵循 `design-tokens.scss`。
