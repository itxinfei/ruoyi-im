# 16 设计复刻差异

> 用途：把当前 UI/页面与钉钉 Windows 客户端对齐所需的差异点写清，作为大模型/设计师复刻参考。  
> 说明：以 `21-IM桌面端页面规范.md` 作为基准样式说明，结合实际页面输出差异并给出落地方向。

## 1. 使用规则

1. 每次调整样式前，先对照本表确认属于哪组差异。
2. 明显偏离钉钉界面时，不直接动手补功能，而是先更新差异表及任务卡 `notes`。
3. 设计复刻任务必须附带视觉对比（稿、截图），并在 `llm-tasks.yaml` 中引用本文件。

## 2. 总体差距矩阵

| 模块/能力 | 目标钉钉表现 | 当前差距 | 优先级 | 建议方向 |
|---------|-------------|--------|------|----------|
| 会话列表 | 240px 固定宽度、信息密度适中、未读/选中态明显、头像+气泡紧凑 | 目前存在标签过多、未读样式偏移动端、卡片感略强 | P0 | 精简信息、按钉钉留白重排、统一未读红点/数字 |
| 消息列表（气泡/头像） | 发送方气泡在前、头像在右、最大宽度 70%、同侧 4px/异侧 16px、整体齐整 | 有时候气泡与头像未对齐，部分消息宽度超标，时间分割线间距不均 | P0 | 强制气泡/头像对齐、限制宽度、补齐时间线空白 |
| 聊天头部 | 60px 高、精简按钮、通话/附件/详情操作在右侧 | 当前按钮较多、有些悬浮菜单仍绑定空事件 | P0 | 按钉钉精简，保证按钮真实可触发，去除多余图标 |
| 输入区 | 工具栏只放真实入口、Enter 发送、Ctrl+Enter 换行、粘贴上传反馈 | 工具入口较多、未统一快捷键说明、有些按钮无反馈 | P0 | 剔除空入口；完善快捷键提示；统一上传反馈 | 
| 搜索/跳转 | 会话过滤 + 全局页 + 会话内定位，结果卡片一致，并可快速跳转 | 搜索页未统一样式、结果分组不清楚、跳转反馈欠缺 | P1 | 拆分结果卡片、加上下文注释、确保跳转成功提示 |
| 群详情 + 群文件 | 统一面板结构、顶部统计与入口、文件列表 + 筛选 | 存在 `GroupProfileDialog` 和 `GroupDetailDrawer` 并行，文件面板交互不稳定 | P1 | 统一主入口，优化文件筛选 + 上传反馈 |
| 通话 | 语音/视频按钮密集、状态态（calling/incoming/talking）强反馈 | 状态提示模糊，禁用/不可用入口未隐藏 | P1 | 加强状态提示、动态按钮可用性、隐藏无效入口 |


## 3. 功能缺口与设计不完善点

1. 多个页面存在空白卡或“开发中”提示，需在对应任务卡中标记为 `unknown` 并补上真实入口。
2. 组件样式与钉钉底层变量（色彩、线宽、间距）未统一，造成感觉不连贯。
3. 弹窗/详情/工具栏之间间距不一致，满足钉钉标准需要：
   - 固定高度与边距（顶部 16px、左右 20px）
   - 按键按钮在 36x36 内，不超过 300ms 圆角风格
4. 交互反馈（悬停、选中、禁止）缺少钉钉式渐变或阴影，需补 `state` 颜色规范。

## 4. 立即行动建议

1. 针对最核心的几个页面（消息首页、聊天头部、群详情）出一份“钉钉 UI vs 当前 UI”对比稿，记录颜色/字体/间距/状态的具体值。
2. 以 `llm-tasks.yaml` 中 `IM-UI-*` 任务为载体，分别在 `notes` 中引用此差异表并明确要“完美复刻钉钉 Windows”。
3. 对功能不全的页面先写 `04-待确认问题.md`，避免模型在缺失功能时贸然实现。
4. 后续每次设计调整都要在 `54-验收用例清单.md` 中确认对应流程，确保能重复验证。
