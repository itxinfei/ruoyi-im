# 26 IM 交互深度行为规范 (DingTalk Interaction & UI Spec)

> **定位**：定义 IM 核心页面的动态交互逻辑与组件的极致 UI 细节，确保"钉钉感"在操作维度的落地。
> **核心原则**：触发式展示、层级化交互、状态无损切换、8px 黄金比例。

---

## 1. 消息气泡全态 UI 与交互 (Message Bubble)

### 1.1 气泡形态与悬停 (Hover Awareness)
- **常规态**：
  - 发送方：背景 `#277EFB`，文字 `#FFFFFF`，圆角 `8px 4px 8px 8px`。
  - 接收方：背景 `#FFFFFF`，文字 `#171A1D`，圆角 `4px 8px 8px 8px`，边框 `1px solid rgba(23, 26, 29, 0.08)`。
  - **红线**：头像顶部必须与气泡第一行文字顶部绝对水平对齐。
- **悬停态 (Hover)**：
  - 触发：鼠标 hover 气泡时立即显示，0 延迟。
  - 展现：在气泡外侧（发送方左侧，接收方右侧）或上方浮现**操作工具条 (Action Bar)**。
  - **工具条定位**：距离气泡边缘 `8px`，与气泡顶部水平对齐（或气泡顶部上方 `4px`）。
  - 工具条 UI：背景 `#FFFFFF`，圆角 `4px`，阴影 `0 2px 8px rgba(0,0,0,0.08)`，包含 16x16 的浅灰色图标（悬停变 `#277EFB`）。

### 1.2 发送状态闭环 (Sending Status)

消息发送遵循完整的状态机：`PENDING` → `SENDING` → `SENT` → `DELIVERED` / `FAILED`

- **待发送 (PENDING)**：消息进入本地队列，气泡使用半透明样式（opacity: 0.6），表示等待发送。
- **发送中 (SENDING)**：WebSocket 已发出，气泡恢复完全不透明。气泡左下角显示 `14x14` 的 Loading 旋转图标，颜色 `#ADB1B8`，动画周期 1s。
- **已发送 (SENT)**：服务端已确认收到，Loading 图标消失，显示"已发送"时间戳（可选）。
- **已送达 (DELIVERED)**：消息已推送到接收方，气泡左下角显示"已送达"文字，字号 `10px`，色值 `rgba(23, 26, 29, 0.4)`。
- **发送失败 (FAILED)**：5000ms 未收到回执，气泡左下角显示红色感叹号图标 `#FF4D4F`（尺寸 `16x16`）。点击该感叹号，弹出重试菜单：[重新发送] [删除消息]。

### 1.3 已读与未读感知 (Read Receipts)
- **显示位置**：仅发送方气泡的左下侧显示。
- **UI 展现**：
  - 未读：品牌蓝色文字 `#277EFB`（如 "1人未读"）。
  - 已读：浅灰色文字 `rgba(23, 26, 29, 0.4)`（显示 "已读"）。
- **状态流转**：收到 WS 回执后，文字立刻硬切，严禁使用复杂的 CSS 过渡动画。

### 1.4 系统通知与撤回 (System Messages)
- **UI 展现**：居中，背景无，文字 `12px`，色值 `rgba(23, 26, 29, 0.4)`，上下间距 `16px`。
- **重新编辑**：撤回消息后，通知文本后方追加蓝色可点击链接 `"重新编辑"` (`#277EFB`)，点击后原文本自动填入下方输入框。

### 1.5 多媒体消息气泡规范 (Multimedia Messages)

#### 1.5.1 图片消息 (Image Message)
- **气泡尺寸**：
  - **最小宽度**：`120px`
  - **最大宽度**：`280px`（不超过屏幕宽度 40%）
  - **高度**：自适应，最大 `400px`，超过后等比缩放
- **圆角**：与文本消息一致（发送方 `8px 4px 8px 8px`，接收方 `4px 8px 8px 8px`）
- **加载态**：显示灰色占位图 `var(--dt-bg-placeholder)`（#F2F4F5），中央带 Loading 旋转图标
- **预览交互**：点击图片弹出全屏预览层，支持缩放和左右滑动查看同会话内其他图片
- **缩略图策略**：默认加载 `200px` 宽度缩略图，点击后加载高清图

#### 1.5.2 视频消息 (Video Message)
- **气泡尺寸**：
  - **最小宽度**：`200px`
  - **最大宽度**：`320px`
  - **高度**：自适应视频比例，最大 `240px`
- **封面图**：显示视频首帧或自定义封面，中央叠加播放按钮图标（`32x32`，白色半透明背景）
- **播放交互**：点击后在气泡内播放，或弹出独立播放器窗口
- **时长显示**：右下角显示视频时长标签，背景 `rgba(0,0,0,0.6)`，文字白色 `12px`

#### 1.5.3 语音消息 (Voice Message)
- **气泡尺寸**：
  - **宽度**：根据时长动态计算，最短 `80px`，最长 `200px`
  - **高度**：固定 `40px`
- **内容布局**：左侧播放图标（`20x20`）+ 波形图/进度条 + 右侧时长文字（`12px`）
- **波形图**：
  - 发送方：白色波形 `rgba(255,255,255,0.8)`
  - 接收方：品牌蓝色波形 `#277EFB`
- **播放交互**：点击播放，波形变为动态进度条，左侧图标变为暂停按钮
- **未读标识**：接收方语音消息未播放时，显示红色小圆点（直径 `6px`）在播放图标右上角

#### 1.5.4 文件消息 (File Message)
- **气泡尺寸**：
  - **宽度**：固定 `260px`
  - **高度**：`64px`
- **布局**：
  - 左侧文件图标（`40x40`，根据文件类型显示不同图标和颜色）
  - 右侧文件名（`14px`，单行截断）+ 文件大小（`12px`，灰色）
- **图标颜色**：
  - PDF：`#FF4D4F`
  - Word：`#2B7DFF`
  - Excel：`#22AB5C`
  - PPT：`#FF7A00`
  - 其他：`#ADB1B8`
- **下载交互**：点击气泡下载文件，显示下载进度条（高度 `3px`，品牌蓝色）

#### 1.5.5 链接卡片消息 (Link Card)
- **气泡尺寸**：
  - **宽度**：固定 `280px`
  - **高度**：自适应，最大 `120px`
- **布局**：
  - 标题（`14px`，粗体，单行截断）
  - 摘要（`12px`，灰色，最多两行）
  - 底部：来源图标 + 来源名称（`12px`）
  - 右侧：缩略图（`80x60`，可选）
- **交互**：点击卡片在新标签页打开链接

## 2. 输入区全态 UI 与交互 (Input Area)

### 2.1 整体感知
- **静态 UI**：顶部有一条 `1px` 宽 `rgba(23, 26, 29, 0.08)` 的横向分割线，背景色 `var(--dt-bg-chat)`（#f5f6f7）。
- **焦点态 (Focus)**：点击输入框内部时，**整个输入区的外框不变色**，保持扁平化沉浸感，严禁出现发光的蓝色边框或阴影。
- **自适应高度**：初始高度 `96px`。随输入内容换行，输入框向上平滑平展，最大高度限制为聊天总高度的 `40%`，超过后在输入框内部出现滚动条。

### 2.2 工具栏与动作 (Toolbar)
- **图标规格**：尺寸 `20x20`，色值 `#ADB1B8`。图标之间水平间距固定 `16px`。
- **Hover 反馈**：鼠标悬停工具栏图标时，图标变色为 `#171A1D`，或背景出现浅灰色 `rgba(23, 26, 29, 0.04)` 的 `4px` 圆角方块。

### 2.3 引用回复态 (Reply Context)
- **UI 展现**：在输入框的正上方，出现一个高度最大 `60px` 的长条形面板。
- **背景与边框**：背景色 `#E6F2FF`（浅蓝），左侧带 `2px` 宽的 `#277EFB` 竖线。
- **内容区**：显示原作者名（`12px`, 灰色）+ 截断为单行的原消息摘要。
- **关闭按钮**：面板右侧带一个 `14x14` 的 `×`，点击立即取消回复态。

### 2.4 发送按钮 (Send Button)
- **位置**：输入区右下角。尺寸：宽度 `64px`，高度 `32px`，圆角 `4px`。
- **空状态**：背景 `#E5E7EB`，文字 `#FFFFFF`（不可点击）。
- **有内容态**：背景 `#277EFB`，文字 `#FFFFFF`。Hover 时背景变浅为 `#1169E0`。
- **过渡动画**：背景色变化使用 `200ms ease-out` 过渡。

## 3. 会话列表 UI 与交互 (Session List)

### 3.1 列表项全态 (Session Item)
- **尺寸**：固定高度 `64px`。内边距：左 `12px`，右 `12px`。
- **默认态**：背景透明，文字颜色 `#171A1D`。
- **悬停态 (Hover)**：背景变为 `rgba(23, 26, 29, 0.04)`，右侧显现静音、置顶等快捷操作的小图标（如果有）。
- **选中态 (Active)**：
  - 背景色锁定为 `#E6F2FF`（品牌浅蓝）。
  - 左侧边缘贴边增加 `3px` 宽、`100%` 高度的 `#277EFB` 竖条。

### 3.2 未读角标 (Badge)
- **红线**：禁止使用正圆包裹 3 位数字！
- **数字 1-9**：使用 `16x16` 的正圆形，背景 `#FF4D4F`，文字 `10px` 居中。
- **数字 >9**：使用胶囊形（高度 `16px`，宽度自适应加上 `padding: 0 4px`），圆角 `8px`。

## 4. 详情抽屉展示规范 (The Drawer Principle)

### 4.1 严禁"常驻右侧"与折叠
- **错误行为**：聊天区右侧始终显示一个占位符，或者将 L2 会话列表折叠成汉堡菜单。
- **正确行为**：详情栏必须是 **`el-drawer` (侧边抽屉)**，宽度固定 `320px`。
- **弹出遮罩**：桌面端抽屉弹出时，默认包含半透明遮罩层 `rgba(0,0,0, 0.1)` 盖住主聊天区，点击遮罩层立即关闭抽屉。

---

## 5. UI 验收负面清单 (Negative Constraints)

1. **圆角禁令**：禁止任何组件使用超过 `12px` 的大圆角（IM 应显得专业、稳重，标准为 `4px-8px` 圆角）。
2. **移动端痕迹禁令**：禁止在桌面端出现底部 Tab 栏、汉堡包收起菜单、全屏大弹窗。
3. **颜色滥用禁令**：除了未读数 `#FF4D4F`、高亮 `#277EFB`、提醒 `#FF7D00`（`--dt-warning-color`），界面 90% 的面积必须是中性灰白体系。
4. **假动画禁令**：禁止在发送消息、切换会话时使用超过 `200ms` 的缓慢渐变动画。企业软件追求"硬切"和"瞬间响应"。

- **背景/阴影**：使用 `transition: background 200ms, box-shadow 200ms`
- **禁止项**：严禁对 `color` 属性使用超过 `200ms` 的过渡

## 7. 弹窗与抽屉规范 (Dialog & Drawer)

### 7.1 弹窗 (Dialog)

- **尺寸**：宽度 `520px`（标准）、`720px`（大）、`320px`（小）
- **内边距**：顶部 `20px`，左右 `24px`，底部 `16px`
- **圆角**：`8px`
- **遮罩**：`rgba(0, 0, 0, 0.5)`
- **动画**：淡入 + 缩放，时长 `250ms`，缓动 `ease-out`

### 7.2 抽屉 (Drawer)

- **尺寸**：宽度 `320px`（标准）、`480px`（大）
- **内边距**：`20px`
- **圆角**：左侧 `8px`，右侧无
- **遮罩**：`rgba(0, 0, 0, 0.1)`
- **动画**：从右向左滑入，时长 `250ms`，缓动 `ease-out`

### 7.3 工具提示 (Tooltip)

- **背景**：`rgba(0, 0, 0, 0.75)`
- **文字**：`#FFFFFF`，字号 `12px`
- **内边距**：`8px 12px`
- **圆角**：`4px`
- **最大宽度**：`280px`
- **显示延迟**：`300ms`
- **隐藏延迟**：`100ms`
