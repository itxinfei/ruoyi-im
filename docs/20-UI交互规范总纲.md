# 20 UI交互规范总纲

> **定位**：锁定项目的“钉钉感”，严禁生成任何平铺式、移动端化的劣质 UI。所有交互必须遵循触发式展示、三段式布局以及统一的视觉 Token。

---

## 1. 空间逻辑指令 (Spatial Rules)

### 1.1 触发式展示 (Trigger‑Only)
- **硬性红线**：禁止将次要信息（群成员、个人资料、文件详情）直接显示在右侧。
- **必须实现**：详情类内容必须使用 `el‑drawer`（侧边抽屉）或 `el‑dialog`（弹窗）承载，且必须由用户主动点击图标/名称触发。

### 1.2 三段式稳定布局
- **L1 (Primary Nav)**：宽度 72px，暗色侧边栏，固定在左侧。
- **L2 (Secondary Nav)**：宽度 240‑320px，会话列表区，支持垂直滚动。
- **L3 (Content)**：聊天/主功能区，占据剩余宽度，禁止出现不必要的滚动条嵌套。
- **响应式规则**：当视口宽度 < 1024px 时，L2 自动折叠为抽屉模式；L1 保持固定宽度；L3 全宽展示。

---

## 2. 视觉 Token 指令 (Visual Tokens)

### 2.1 颜色锁
- **品牌蓝**：`var(--im-primary-color)`（默认 `#1677FF`），所有交互高亮使用该变量。
- **中性灰**：`#F7F7F7`（会话栏背景），`#F0F2F5`（页面整体背景）。
- **文字色**：标题 `#1F1F1F`，正文 `#8C8C8C`，在暗色模式下分别映射为 `--im-text-primary-dark`、`--im-text-secondary-dark`。

### 2.2 排版与字距
- **字体栈**：`PingFang SC, Microsoft YaHei, system-ui`，不引入外部 Google Fonts。
- **字号规范**：标题 16‑20px，正文 14px，辅助说明 12px。
- **行高**：正文 1.5，标题 1.4，确保可读性。

### 2.3 气泡规范
- **对齐**：头像顶部与气泡顶部基线对齐。
- **气泡颜色**：发送者使用 `var(--im-bubble-me)`（品牌蓝），接收者使用 `var(--im-bubble-other)`（浅灰）。
- **圆角**：统一 `8px`，在暗色模式下保持一致。
- **Hover 效果**：消息操作条（复制、撤回）仅在 Hover 时出现，使用 `opacity` 过渡 200ms。

---

## 3. 资源气密性指令 (Air‑gap Assets)

### 3.1 图标
- **本地化**：严禁使用远程 `cdn.iconfont.cn` 或 `iconify`，统一使用 `@element-plus/icons-vue` 或 `src/assets/icons/` 下的 SVG。
- **调用方式**：`<svg-icon name="icon‑msg" />`，图标统一尺寸 `16px`，颜色使用 `currentColor`。

### 3.2 字体图标与图片
- **SVG 优先**：所有装饰性图标均使用 SVG，确保在高 DPI 屏幕上清晰。
- **图片规范**：业务图片（头像、文件预览）统一使用 `webp`，并在 `src/assets/images/` 中提供 `2x`、`3x` 版本。

---

## 4. 动效与反馈 (Animation & Feedback)

### 4.1 页面切换动效
- **淡入淡出**：模块切换使用 200ms `fade` 过渡，使用 CSS `transition: opacity 0.2s` 实现。
- **侧边抽屉**：打开/关闭使用 `transform: translateX` 动画，时长 250ms，配合 `ease-out` 曲线。

### 4.2 交互反馈
- **按钮状态**：所有交互按钮必须提供 `loading`、`disabled` 两种视觉状态，使用 Element Plus `el-button` 的 `loading` 属性实现。
- **错误提示**：统一使用 `ElMessage`，错误信息颜色 `var(--im-error-color)`，并在 3 秒后自动消失。

---

## 5. 可访问性 (Accessibility)

- **键盘导航**：所有可交互组件必须支持 `Tab` 键聚焦，焦点样式使用 `outline: 2px solid var(--im-primary-color)`。
- **ARIA 标识**：弹窗、抽屉必须提供 `role="dialog"` 并设置 `aria‑label`。
- **颜色对比**：文本与背景的对比度必须 ≥ 4.5:1，尤其在暗色模式下需额外检查。

---

## 6. 主题化与暗色模式

- **CSS 变量**：所有颜色、间距、圆角均通过 `:root` 中的变量定义，暗色模式在 `[data-theme="dark"]` 下覆盖对应变量。
- **切换机制**：通过 Pinia 状态 `themeStore.isDark` 控制 `document.documentElement.dataset.theme`，实现即时切换。
- **组件适配**：Element Plus 组件默认使用 `--el-color-primary`，在项目入口统一映射到 `--im-primary-color`，确保统一品牌色。

---

> **业务背景**：本规范旨在确保 IM 系统在企业内部使用时，既保持钉钉的熟悉感，又符合现代 Web 可维护性与可扩展性要求。所有开发者在实现新功能时必须遵循本规范，否则视为违规。
