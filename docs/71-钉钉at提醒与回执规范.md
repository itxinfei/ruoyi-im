# 71 钉钉 @提醒与已读回执规范 (Mention & Read Receipt Spec)

> **用途**：定义 @ 提醒和已读回执的视觉与交互标准。这是钉钉桌面端区别于普通 IM 的核心交互之一。
> **权威依据**：docs/00-钉钉8.2像素级权威基线.md §2.6 / 视觉资产 #16-mention-list.png #17-mention-message.png
> **优先级**：⭐⭐⭐ 最高

---

## 1. @ 提醒（Mention）

### 1.1 三种 @ 场景

| 场景 | 表现位置 | 视觉特征 |
|-----|---------|---------|
| **会话列表前缀** | 会话列表项的消息预览前 | "[有人@我]" 橙色加粗 |
| **消息气泡内 @文字** | 消息正文中的 @某人 | 橙色文字 + 浅橙底色 + 不可换行 |
| **@ 选择面板** | 输入区上方浮层 | 滑入 popover、最大 6 行 |

### 1.2 @ 文字色与背景（核心钉钉味）

CSS 实现：
- color: var(--dt-mention-color) = #FF7D00 橙色
- background: var(--dt-mention-bg) = rgba(255, 125, 0, 0.08) 浅橙底
- padding: 0 4px
- border-radius: var(--dt-radius-sm) = 4px
- font-weight: var(--dt-font-weight-medium) = 500
- white-space: nowrap (不换行)
- hover: 底色加深至 rgba(255, 125, 0, 0.16)

### 1.3 会话列表前缀枚举

| 前缀 | 颜色 | 字重 | 出现时机 |
|-----|------|-----|---------|
| [有人@我] | #FF7D00 | 600 | 群聊有 @ 提醒 |
| [草稿] | #F53F3F | 600 | 输入框有未发送内容 |
| [图片] | #8A9099 | 400 | 最后一条是图片 |
| [文件] | #8A9099 | 400 | 最后一条是文件 |
| [语音通话] | #8A9099 | 400 | 最后一条是通话记录 |
| [视频通话] | #8A9099 | 400 | 同上 |
| [红包] | #F53F3F | 500 | 最后一条是红包 |
| [DING] | #FF4D4F | 600 | 收到 DING 消息 |

### 1.4 消息气泡中的 @ 渲染

由于消息内容是字符串，需要在渲染时把 "@张三 " 替换为带样式的 span。

实现要点：
1. 使用 v-html 渲染，但必须先转义 HTML 防 XSS
2. 然后替换 @姓名 标记为 .mention-tag span
3. mentions 数组结构：[{ userId, userName }]

伪代码：
```
renderedContent = computed:
  html = escapeHtml(content)
  for m in mentions:
    tag = "@" + m.userName
    html = html.replaceAll(escapeHtml(tag), span_with_class)
  return html
```

### 1.5 @ 选择面板（Mention Picker）

触发条件：用户在输入框输入 @ 字符（仅限群聊）。

视觉规范（ASCII 骨架）：

```
┌───────────────────────────┐
│  🔍 搜索成员              │
├───────────────────────────┤
│  ◯ 张三   后端工程师      │
│  ◯ 李四   前端工程师      │
│  ◯ 王五   产品经理        │
│  ──────────────           │
│  📢 @所有人              │
└───────────────────────────┘
```

| 元素 | 规范 |
|-----|------|
| 容器 | position: fixed; z-index: var(--dt-z-popover); width: 280px; max-height: 280px |
| 背景 | var(--dt-bg-card); 圆角 var(--dt-radius-lg); 阴影 var(--dt-shadow-3) |
| 搜索框 | 高 36px，padding 0 12px |
| 列表项 | 高 40px，gap 12px |
| 头像 | 24px × 24px，圆角 var(--dt-radius-sm) (4px) |
| 姓名 | 13px primary，部门/职位 11px tertiary |
| @所有人 | 上方加 1px 分隔线，icon 容器使用 var(--dt-mention-bg) |

### 1.6 交互规则

| 触发 | 行为 |
|-----|------|
| 输入 @ | 立刻显示选择面板（< 100ms） |
| 输入 @关键词 | 实时过滤成员（按姓名 / 拼音首字母） |
| ↑ / ↓ 键 | 切换选中项 |
| Enter | 确认插入 "@姓名 " (末尾必须空格) |
| Esc | 关闭面板，保留已输入字符 |
| 点击空白 | 关闭面板 |
| 选择 @所有人 | 仅群主 / 管理员可见 |

## 2. 已读回执（Read Receipt）

### 2.1 钉钉的回执视觉特点

```
单聊已读回执：
  [我发的消息气泡]
                    ↳ ✓✓ 已读 / ✓ 未读 / ⏱ 发送中
                       (仅发送方自己可见)

群聊已读回执：
  [我发的消息气泡]
                    ↳ 12人已读 (点击查看详情)
```

### 2.2 单聊回执规范

| 状态 | 视觉 | CSS Token |
|-----|-----|-----------|
| 发送中（0-3s） | ⏱ 转圈 icon，灰色 | --dt-text-tertiary |
| 已发送未读 | ✓ 单勾，灰色 | --dt-delivered-check-color |
| 对方已读 | ✓✓ 双勾，蓝色 | --dt-read-check-color |
| 发送失败 | ⚠ 红色感叹号 + 重试按钮 | --dt-error-color |

实现要点：
1. 仅发送方自己可见 (v-if isOwn)
2. 双勾必须重叠 8px（margin-left: -8px）制造钉钉感
3. 发送中状态使用 rotate 动画
4. 失败状态可点击触发 retry 事件

### 2.3 群聊回执规范

显示 "X人已读" 或 "全部已读" 链接，点击弹出详情抽屉。

| 状态 | 文本 | 颜色 |
|-----|------|------|
| 部分已读 | "12人已读" | --dt-text-tertiary |
| 全部已读 | "全部已读" | --dt-success-color |
| Hover | 同上 | --dt-brand-color + 下划线 |

### 2.4 已读详情抽屉

点击 "12人已读" 时弹出右侧抽屉：

```
┌─────────────────────────┐
│  消息已读详情      ✕    │
├─────────────────────────┤
│  [Tab] 已读 (12)        │
│  [Tab] 未读 (4)         │
├─────────────────────────┤
│  ◯ 张三   2分钟前        │
│  ◯ 李四   5分钟前        │
│  ◯ 王五   10分钟前       │
│  ...                    │
└─────────────────────────┘
```

| 元素 | 规范 |
|-----|------|
| 抽屉宽度 | 320px (var(--dt-drawer-width)) |
| Tab 高度 | 40px |
| 列表项高度 | 56px |
| 头像 | 36px 圆角矩形 (var(--dt-radius-sm)) |
| 时间字号 | 12px / var(--dt-text-tertiary) |

## 3. 验收清单 (AC)

### 3.1 @ 提醒
- [ ] @ 文字必须使用 #FF7D00 橙色，禁止用蓝色或其他颜色
- [ ] 会话列表前缀 [有人@我] 必须橙色加粗
- [ ] 输入 @ 立刻显示选择面板（< 100ms）
- [ ] 面板支持 ↑↓Enter 键盘导航
- [ ] 选择后自动插入 "@姓名 " 末尾带空格

### 3.2 已读回执
- [ ] 单聊使用 ✓ / ✓✓ 双勾，禁止用文字"已读"
- [ ] 双勾必须重叠 8px（不是并排）
- [ ] 群聊显示 "X人已读" 链接，点击弹出详情抽屉
- [ ] 仅发送方自己可见回执，禁止接收方看到
- [ ] 失败状态可点击重试

### 3.3 颜色一致性
- [ ] 全应用范围 #FF7D00 橙色仅用于：① @ 相关 ② DING 相关
- [ ] 已读双勾仅用 #277EFB 品牌蓝
- [ ] 未读单勾仅用 #8A9099 中性灰

## 4. 反例（禁止）

| 反例 | 错误 | 正确 |
|-----|------|------|
| @ 文字用蓝色 | color: var(--dt-brand-color) | color: var(--dt-mention-color) |
| 已读用文字 | <span>已读</span> | <Check /> × 2 (双勾 icon) |
| 双勾并排 | gap: 4px | margin-left: -8px (重叠) |
| 接收方可见回执 | v-if="message" | v-if="isOwn" |
| @ 选择面板用 fixed 直接定位 | 全局浮层位置错乱 | 使用 Teleport + 计算光标位置 |
