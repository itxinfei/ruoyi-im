# 21 IM 桌面端页面规范

> 用途：把消息模块的桌面端页面规范写成更细的执行标准。
> 说明：本文件用于约束"像钉钉 Windows 客户端那样工作"，不是要求品牌素材逐像素复刻。

## 1. 适用范围

本规范适用于消息模块相关页面与组件：

1. `MainPage`
2. `SessionPanel`
3. `ChatPanel`
4. `ChatHeader`
5. `MessageList`
6. `MessageBubble`
7. `MessageInput`
8. `UserDetailDrawer`
9. `GroupDetailDrawer`
10. 搜索相关页面

## 2. 设计目标

1. 视觉、布局、交互默认参考钉钉 Windows 客户端。
2. 优先桌面端信息密度，不采用偏移动端的卡片堆叠和大圆角胶囊风格。
3. 优先解决结构清晰、操作顺手、状态明确，而不是做装饰性设计。
4. IM 页面是平台主工作区，必须稳定、克制、可长期使用。

## 3. 页面总布局

### 3.1 主框架

消息主页面采用四段结构：

1. 左侧一级导航窄栏
2. 二级会话栏
3. 聊天主区域
4. 右侧详情区域

### 3.2 推荐尺寸

1. 一级导航宽度：`72px`
2. 二级会话栏宽度：`240px`
3. 右侧详情区域宽度：`320px`
4. 聊天头部高度：`60px`
5. 输入区最小高度：`96px`
6. 输入区最大高度：`300px`

### 3.3 布局要求

1. 二级会话栏与聊天区之间必须有清晰分隔。
2. 聊天区为页面主视觉中心，不允许被次要信息抢占。
3. 右侧详情区只展示当前会话上下文，不承载其他无关模块。
4. 页面横向结构必须稳定，不因切换会话频繁抖动。

## 4. 一级导航规范

### 4.1 信息架构

一级导航应面向“完整企业协同平台”，而不是单独聊天系统。

应覆盖：

1. 消息
2. 搜索
3. 通讯录
4. 工作台
5. 云盘/文档
6. 日历
7. 待办
8. 审批
9. 邮箱
10. AI 助手
11. 设置
12. 个人资料
13. 管理入口

### 4.2 视觉要求

1. 图标风格统一，线性或简洁面性，不混用多套风格。
2. 当前选中模块有明确高亮。
3. 悬停、选中、未读、提醒状态不能混淆。
4. 不使用偏移动端的大面积胶囊式高亮。

## 5. 二级会话栏规格 (Session Panel)

### 5.1 搜索与筛选 (Search & Filter)
- **搜索框**：固定高度 `32px`，圆角 `4px`，背景色 `var(--dt-bg-hover)`。
- **交互感**：Focus 时背景变白，增加 `1px` 蓝边投影。
- **筛选行**：仅显示"全部"与"未读"两个 Text Tab，字号 `12px`。

### 5.2 会话项细节 (Session Item Details)
- **未读数 (Badge)**：红色圆形（#FF4D4F），白色字，字号 `10px`。位于头像右上角，**不要**遮挡头像核心区域。
- **状态图标 (Flags)**：
  - **草稿**：摘要首位红色 `[草稿]` 字样。
  - **免打扰**：静音图标位于时间左侧，灰色展示。
  - **置顶**：项背景色略微加深（#F9FAFB），不使用大图标。

## 6. 聊天头部规范

### 6.1 结构

聊天头部应分为左右两区：

1. 左侧：会话名称、会话副信息
2. 右侧：搜索、通话、详情等高频操作

### 6.2 设计要求

1. 操作按钮数量必须克制，只保留高频动作。
2. 图标按钮尺寸和间距紧凑，符合桌面端效率习惯。
3. 不允许存在点击无效的壳按钮。

## 7. 消息列表规范

### 7.1 整体要求

1. 默认滚动到底部显示最新消息。
2. 上滑加载历史消息时滚动位置保持稳定。
3. 新消息到达时自动定位行为要可预期。

### 7.2 时间分割线

1. 时间分割线居中。
2. 文字弱化，不抢主消息内容。
3. 分割线前后留白明确，但不造成双重空白。

### 7.3 消息分组

1. 同一侧连续消息应有分组感。
2. 同侧消息间距小于异侧消息间距。
3. 分组不能破坏消息操作区域的可用性。

## 8. 消息气泡与主区域规格 (Chat Panel)

### 8.1 消息操作悬浮条 (Action Bar)
- **触发**：鼠标 Hover 气泡时，在气泡上方弹出。
- **样式**：白色背景，`1px` 边框，图标 `16x16`，间距 `12px`。包含：回复、转发、复制、收藏、更多。

### 8.2 系统级通知 (System Notice)
- **场景**：时间线、撤回提示。
- **样式**：居中展示，文字色 `var(--dt-text-desc)`，字号 `12px`。撤回后的"重新编辑"应为蓝色链接。

### 8.3 链接卡片规格 (Link Card)
- **布局**：最大宽度 `360px`，白色背景，`var(--dt-radius-s)` 圆角。内边距 `12px`，标题 `14px` 加粗。

### 8.4 发送与接收气泡
- **发送方**：背景 `var(--dt-bubble-me)`，文字白色。气泡在前，头像在后。
- **接收方**：背景 `var(--dt-bubble-other)`，文字 `var(--dt-text-main)`。头像在前，气泡在后。
- **头像规格**：固定 `36x36`，圆角 `4px`。

## 9. 输入区规范

### 9.1 基础结构

输入区应包含：

1. 工具栏
2. 文本输入区域
3. 回复态或引用态展示区
4. 发送按钮

### 9.2 行为要求

1. Enter 发送
2. Ctrl+Enter 或 Cmd+Enter 换行
3. 图片、文件、@ 入口必须真实可用
4. 粘贴图片或文件时必须进入明确上传流程

### 9.3 禁止项

1. 展示未闭环的录音入口
2. 展示不可点击的工具按钮
3. 用复杂富文本样式破坏办公输入效率

## 10. 业务详情抽屉 (Business Drawer)

### 10.1 核心结构 (Hierarchy)
- **Section 1: 基础/成员**：单聊展示详细资料卡；群聊展示 **"成员九宫格"**（前 8 名头像 + 更多按钮），头像间距 `8px`。
- **Section 2: 功能矩阵**：列表项形式：群文件、群公告、查找聊天内容。右侧带 `>` 弱指引。
- **Section 3: 设置区**：置顶、免打扰、保存到通讯录等开关，统一使用 **Small Switch**。

### 10.2 间距与高度
- **Section 垂直间距**：固定 `24px`。
- **List Item 高度**：固定 `44px`，文字字号 `14px`。
- **内边距**：容器内边距固定为 `20px`。

## 11. 搜索页面规范

### 11.1 搜索范围

至少支持：

1. 消息
2. 联系人
3. 群组
4. 文件

### 11.2 搜索行为

1. 会话栏搜索用于本地过滤会话。
2. 独立搜索页用于跨模块聚合搜索。
3. 会话内搜索应能定位到具体消息。

### 11.3 结果要求

1. 结果必须可跳转。
2. 无结果时显示清晰空状态。
3. 搜索失败时显示明确反馈。

## 12. 禁止偏差

以下情况视为明显偏离目标形态：

1. 明显移动端风格的胶囊式导航和大面积圆角卡片
2. 消息区存在错位、头像与气泡不在同一基线
3. 会话列表信息过载或标签过多
4. 详情区塞入与当前会话无关内容
5. 页面存在“开发中”死路入口
6. UI 有入口但实际事件未接入

## 14. 8px 黄金比例尺 (Visual Rhythm)

为了确保界面具备工业级的严整感，所有组件开发必须遵循 **8px 栅格步进**：

### 14.1 间距步进 (Spacing)
所有 `margin` 和 `padding` 必须从以下数列中选择：
- **4px (XS)**：极紧凑，如气泡内小图标间距。
- **8px (S)**：标准紧凑，如列表项内边距。
- **16px (M)**：标准间距，如卡片内边距、气泡间距。
- **24px (L)**：大间距，如页面区块分隔。
- **32px (XL)**：超大间距，如页面顶部留白。

### 14.2 圆角规格 (Radius)
禁止随意定义圆角，统一为：
- **4px**：小按钮、标签。
- **8px**：消息气泡、输入框、下拉面板。
- **12px**：卡片、抽屉容器、大头像。

### 14.3 层级阴影 (Elevation)
- **Level 1 (静止)**：`0 2px 8px rgba(0,0,0,0.05)` - 消息操作栏。
- **Level 2 (悬浮/激活)**：`0 4px 16px rgba(0,0,0,0.1)` - 气泡 Hover、右键菜单。
- **Level 3 (顶层)**：`0 8px 32px rgba(0,0,0,0.15)` - 抽屉、模态弹窗。

## 16. 键盘优先与快捷键规范 (Efficiency)

为了对齐桌面端效率体验，所有核心交互必须支持以下快捷键映射：

### 16.1 全局快捷键
- **Ctrl + F**：焦点自动定位至二级会话栏搜索框。
- **Esc**：关闭当前打开的详情抽屉 (Detail Drawer) 或 弹窗。
- **Alt + S** (可选)：快速拉起截图（若具备插件能力）。

### 16.2 聊天区快捷键
- **Enter**：发送消息。
- **Ctrl + Enter**：在输入框内换行。
- **↑ / ↓ (在会话栏焦点时)**：上下切换选中会话。
- **Ctrl + / **：快速打开常用表情面板。

## 17. 交付验收清单

1. 页面结构是否稳定且接近钉钉 Windows 客户端
2. 会话栏是否简洁、清晰、可快速扫读
3. 聊天头部操作是否克制且真实可用
4. 消息气泡排列是否符合桌面 IM 习惯
5. 输入区工具栏是否只保留真实可用能力
6. 详情面板是否与当前会话上下文一致
7. 搜索是否可跳转、可定位、可反馈异常
