# 补丁设计草案：MessageBubble 文本渲染增强

目标：提升对链接边界的鲁棒性，以及为未来扩展简单文本格式（粗体/斜体）做好准备。

实现要点（不含具体实现代码）：
- 将 parsedTextParts 拓展为多类型分块：text、link、bold、italic。
- 链接识别改为更健壮的正则，忽略结尾的标点符号（如句号、逗号）属于文本的一部分。
- 增加对简单文本格式的识别标记：如文本中出现 **text** 转为 bold，_text_ 转为 italic（初步处理，避免直接 using v-html）。
- 渲染逻辑改为基于 part.type 动态渲染，而非仅 text/link。
- 引入简单的缓存/记忆，以提升大文本渲染时的性能。

验收要点：
- 链接边界正确，不把末尾标点误作链接的一部分。
- 支持简单的 Bold/Italic 渲染（示例文本通过测试用例验证）。
- 仍然保持无 XSS 风险（使用文本节点渲染，不用 v-html）。
- 渲染性能在大文本时无明显下降。
